# 设计系统：Supabase

## 1. 视觉主题与氛围

Supabase 的网站是一个原生暗色模式的开发者平台，传递着高级代码编辑器的美学 — 深黑色背景（`#0f0f0f`、`#171717`）搭配翡翠绿点缀色（`#3ecf8e`、`#00c573`），呼应了品牌开源、PostgreSQL 绿色的身份认同。这个设计系统感觉像是从终端窗口中诞生，然后演变为一个成熟的营销界面，却没有失去它的开发者灵魂。

字体排版基于 "Circular" — 一款带有圆角末端的几何无衬线字体，柔化了技术感的锐利边缘。在 72px 字号和 1.00 行高下，英雄区文字被压缩到绝对最小的垂直空间，创造出紧凑、有力的陈述，毫不浪费。等宽字体（Source Code Pro）少量用于带有 1.2px 字间距的大写技术标签，创造出将营销网站与产品体验连接起来的"开发者控制台"标记。

让 Supabase 独具特色的是其成熟的基于 HSL 的颜色令牌系统。Supabase 没有使用扁平的十六进制值，而是几乎为每种颜色使用带有 alpha 通道的 HSL（`--colors-crimson4`、`--colors-purple5`、`--colors-slateA12`），实现了一个细腻的分层系统，颜色通过透明度相互作用。这通过半透明创造了深度 — 边框使用 `rgba(46, 46, 46)`，表面使用 `rgba(41, 41, 41, 0.84)`，点缀色使用部分不透明度，所有这些都与暗色背景融合，从最少的颜色成分中创造出丰富、有层次的调色板。

绿色点缀色（`#3ecf8e`）有选择性地出现 — 在 Supabase 标志中、在链接颜色（`#00c573`）中、在边框高亮（`rgba(62, 207, 142, 0.3)`）中 — 总是作为"这就是 Supabase"的信号，而非装饰元素。用于主要 CTA 的药丸形按钮（9999px 圆角）与用于次要元素的标准 6px 圆角形成对比，创造出清晰的重要性视觉层级。

**关键特征：**
- 原生暗色模式：近乎黑色的背景（`#0f0f0f`、`#171717`）— 绝非纯黑
- 翡翠绿品牌点缀色（`#3ecf8e`、`#00c573`）作为身份标记，谨慎使用
- Circular 字体 — 带有圆角末端的几何无衬线字体
- Source Code Pro 用于大写技术标签（1.2px 字间距）
- 基于 HSL 的颜色令牌系统，带有 alpha 通道用于半透明分层
- 主要 CTA 使用药丸形按钮（9999px），次要元素使用 6px 圆角
- 中性灰色梯度从 `#171717` 经过 `#898989` 到 `#fafafa`
- 边框系统使用深灰色（`#2e2e2e`、`#363636`、`#393939`）
- 最小化阴影 — 通过边框对比和透明度创造深度
- Radix 颜色原语（crimson、purple、violet、indigo、yellow、tomato、orange、slate）

## 2. 调色板与角色

### 品牌
- **Supabase Green**（`#3ecf8e`）：主要品牌色、标志、点缀边框
- **Green Link**（`#00c573`）：用于链接和操作的交互绿色
- **Green Border**（`rgba(62, 207, 142, 0.3)`）：微妙的绿色边框点缀

### 中性色阶（暗色模式）
- **Near Black**（`#0f0f0f`）：主要按钮背景、最深的表面
- **Dark**（`#171717`）：页面背景、主要画布
- **Dark Border**（`#242424`）：水平分割线、区块分隔符
- **Border Dark**（`#2e2e2e`）：卡片边框、标签页边框
- **Mid Border**（`#363636`）：按钮边框、分隔符
- **Border Light**（`#393939`）：次要边框
- **Charcoal**（`#434343`）：第三级边框、深色点缀
- **Dark Gray**（`#4d4d4d`）：重度次要文本
- **Mid Gray**（`#898989`）：弱化文本、链接颜色
- **Light Gray**（`#b4b4b4`）：次要链接文本
- **Near White**（`#efefef`）：浅色边框、微妙表面
- **Off White**（`#fafafa`）：主要文本、按钮文本

### Radix 颜色令牌（基于 HSL）
- **Slate Scale**：`--colors-slate5` 至 `--colors-slateA12` — 中性渐变
- **Purple**：`--colors-purple4`、`--colors-purple5`、`--colors-purpleA7` — 点缀光谱
- **Violet**：`--colors-violet10`（`hsl(251, 63.2%, 63.2%)`）— 鲜艳点缀
- **Crimson**：`--colors-crimson4`、`--colors-crimsonA9` — 暖色点缀/警示
- **Indigo**：`--colors-indigoA2` — 微妙的蓝色渲染
- **Yellow**：`--colors-yellowA7` — 注意/警告
- **Tomato**：`--colors-tomatoA4` — 错误点缀
- **Orange**：`--colors-orange6` — 暖色点缀

### 表面与覆盖层
- **Glass Dark**（`rgba(41, 41, 41, 0.84)`）：半透明暗色覆盖层
- **Slate Alpha**（`hsla(210, 87.8%, 16.1%, 0.031)`）：超微妙蓝色渲染
- **Fixed Scale Alpha**（`hsla(200, 90.3%, 93.4%, 0.109)`）：浅色霜状覆盖层

### 阴影
- Supabase 在其暗色主题中**几乎不使用阴影**。深度通过边框对比和表面颜色差异来创造，而非 box-shadows。聚焦状态使用 `rgba(0, 0, 0, 0.1) 0px 4px 12px` — 极简、功能性。

## 3. 字体排版规则

### 字体家族
- **主要**：`Circular`，回退字体：`custom-font, Helvetica Neue, Helvetica, Arial`
- **等宽**：`Source Code Pro`，回退字体：`Office Code Pro, Menlo`

### 层级

| 角色 | 字体 | 字号 | 字重 | 行高 | 字间距 | 备注 |
|------|------|------|------|------|--------|------|
| 展示英雄区 | Circular | 72px (4.50rem) | 400 | 1.00（紧凑）| normal | 最大密度，零浪费 |
| 区块标题 | Circular | 36px (2.25rem) | 400 | 1.25（紧凑）| normal | 功能区块标题 |
| 卡片标题 | Circular | 24px (1.50rem) | 400 | 1.33 | -0.16px | 轻微负字距 |
| 副标题 | Circular | 18px (1.13rem) | 400 | 1.56 | normal | 次要标题 |
| 正文 | Circular | 16px (1.00rem) | 400 | 1.50 | normal | 标准正文 |
| 导航链接 | Circular | 14px (0.88rem) | 500 | 1.00–1.43 | normal | 导航项 |
| 按钮 | Circular | 14px (0.88rem) | 500 | 1.14（紧凑）| normal | 按钮标签 |
| 说明文字 | Circular | 14px (0.88rem) | 400–500 | 1.43 | normal | 元数据、标签 |
| 小字 | Circular | 12px (0.75rem) | 400 | 1.33 | normal | 附注、页脚链接 |
| 代码标签 | Source Code Pro | 12px (0.75rem) | 400 | 1.33 | 1.2px | `text-transform: uppercase` |

### 原则
- **字重克制**：几乎所有文本使用字重 400（常规/书籍）。字重 500 仅用于导航链接和按钮标签。检测到的系统中没有粗体（700）— 层级通过字号而非字重创造。
- **1.00 英雄区行高**：英雄区文字被压缩到绝对的零行距。这是决定性的字体排版姿态 — 文字感觉像终端命令：紧凑、高效，没有浪费的垂直空间。
- **卡片负字距**：卡片标题使用 -0.16px 字间距，一种微妙的收紧，在不显眼的情况下将其与正文区分开来。
- **等宽字体作为仪式**：Source Code Pro 以大写形式配合 1.2px 字间距是"开发者控制台"的声音 — 少量用于连接产品体验的技术标签。
- **几何个性**：Circular 的圆角末端在原本可能冰冷的、技术感的界面中创造了温度。字体是人性化元素。

## 4. 组件样式

### 按钮

**主要药丸形（暗色）**
- 背景：`#0f0f0f`
- 文本：`#fafafa`
- 内边距：8px 32px
- 圆角：9999px（完整药丸形）
- 边框：`1px solid #fafafa`（暗色上的白色边框）
- 聚焦阴影：`rgba(0, 0, 0, 0.1) 0px 4px 12px`
- 用途：主要 CTA（"Start your project"）

**次要药丸形（暗色、弱化）**
- 背景：`#0f0f0f`
- 文本：`#fafafa`
- 内边距：8px 32px
- 圆角：9999px
- 边框：`1px solid #2e2e2e`（暗色边框）
- 不透明度：0.8
- 用途：与主要按钮并列的次要 CTA

**幽灵按钮**
- 背景：透明
- 文本：`#fafafa`
- 内边距：8px
- 圆角：6px
- 边框：`1px solid transparent`
- 用途：第三级操作、图标按钮

### 卡片与容器
- 背景：暗色表面（`#171717` 或稍浅）
- 边框：`1px solid #2e2e2e` 或 `#363636`
- 圆角：8px–16px
- 无可见阴影 — 边框定义边缘
- 内部内边距：16px–24px

### 标签页
- 边框：`1px solid #2e2e2e`
- 圆角：9999px（药丸形标签页）
- 激活状态：绿色点缀或较浅表面
- 非激活状态：暗色、弱化

### 链接
- **绿色**：`#00c573` — Supabase 品牌链接
- **主要浅色**：`#fafafa` — 暗色上的标准链接
- **次要**：`#b4b4b4` — 弱化链接
- **弱化**：`#898989` — 第三级链接、页脚

### 导航
- 与页面匹配的暗色背景（`#171717`）
- 带有绿色图标的 Supabase 标志
- Circular 14px 字重 500 用于导航链接
- 简洁的水平布局，带有产品下拉菜单
- 绿色 "Start your project" CTA 药丸形按钮
- 粘性头部行为

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 比例：1px、4px、6px、8px、12px、16px、20px、24px、32px、40px、48px、90px、96px、128px
- 显著的大跨度：48px → 90px → 96px → 128px 用于主要区块间距

### 网格与容器
- 居中内容，带有充裕的最大宽度
- 全宽暗色区块，内容约束在内部
- 功能网格：基于图标的网格，卡片尺寸一致
- 标志网格用于"Trusted by"区块
- 页脚：暗色背景上的多列布局

### 断点
| 名称 | 宽度 | 关键变化 |
|------|------|----------|
| Mobile | <600px | 单列、堆叠布局 |
| Desktop | >600px | 多列网格、扩展布局 |

*注：Supabase 使用了一个极简的断点系统 — 主要只有一个 600px 断点，表明采用移动优先的方法配合渐进增强。*

### 留白哲学
- **戏剧性的区块间距**：主要区块之间 90px–128px 的间距创造了电影般的节奏 — 每个区块都是暗色虚空中独立的场景。
- **密集的内容块**：在区块内部，间距紧凑（16px–24px），创造出浓缩的信息集群。
- **边框定义空间**：Supabase 不用留白 + 阴影来分隔，而是在暗色背景上使用细边框 — 通过线条而非间隙来分隔。

### 边框圆角比例
- 标准（6px）：幽灵按钮、小元素
- 舒适（8px）：卡片、容器
- 中等（11px–12px）：中等尺寸面板
- 大（16px）：功能卡片、主要容器
- 药丸形（9999px）：主要按钮、标签页指示器

## 6. 深度与层级

| 级别 | 处理方式 | 用途 |
|------|----------|------|
| 扁平（Level 0） | 无阴影，边框 `#2e2e2e` | 默认状态，大多数表面 |
| 微妙边框（Level 1） | 边框 `#363636` 或 `#393939` | 交互元素、悬停状态 |
| 聚焦（Level 2） | `rgba(0, 0, 0, 0.1) 0px 4px 12px` | 仅聚焦状态 |
| 绿色点缀（Level 3） | 边框 `rgba(62, 207, 142, 0.3)` | 品牌高亮元素 |

**阴影哲学**：Supabase 有意避免阴影。在原生暗色模式设计中，阴影几乎不可见且毫无作用。相反，深度通过精细的边框层级来传达 — 从 `#242424`（几乎不可见）经过 `#2e2e2e`（标准）到 `#393939`（突出）。30% 不透明度的绿色点缀边框（`rgba(62, 207, 142, 0.3)`）是"提升"状态 — 品牌色本身成为深度信号。

## 7. 最佳实践

### 应该
- 使用近乎黑色的背景（`#0f0f0f`、`#171717`）— 深度来自灰色边框层级
- 谨慎使用 Supabase 绿色（`#3ecf8e`、`#00c573`）— 它是身份标记，不是装饰
- 几乎所有内容都使用字重 400 的 Circular — 500 仅用于按钮和导航
- 将英雄区文字设置为 1.00 行高 — 零行距是字体排版的标志性特征
- 通过边框颜色差异创造深度（`#242424` → `#2e2e2e` → `#363636`）
- 药丸形（9999px）仅用于主要 CTA 和标签页
- 使用带有 alpha 通道的基于 HSL 的颜色实现半透明分层效果
- 使用 Source Code Pro 大写标签作为开发者上下文标记

### 不应该
- 不要添加 box-shadows — 它们在暗色背景上不可见，会破坏边框定义的深度系统
- 不要使用粗体（700）字重 — 系统仅使用 400 和 500
- 不要将绿色应用于背景或大表面 — 它用于边框、链接和小点缀
- 不要将暖色（crimson、orange）作为主要设计元素 — 它们作为语义令牌用于状态
- 不要将英雄区行高增加到 1.00 以上 — 密度是有意为之
- 不要在按钮上使用大圆角（16px 以上）— 药丸形（9999px）或标准（6px），不要中间值
- 不要将主要表面的背景调亮到 `#171717` 以上 — 暗色是结构性的
- 不要忘记半透明边框 — `rgba` 边框颜色是分层机制

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 关键变化 |
|------|------|----------|
| Mobile | <600px | 单列、堆叠功能、精简导航 |
| Desktop | >600px | 多列网格、完整导航、扩展区块 |

### 折叠策略
- 英雄区：72px → 按比例缩小
- 功能网格：多列 → 单列堆叠
- 标志行：水平 → 换行网格
- 导航：完整 → 汉堡菜单
- 区块间距：90–128px → 48–64px
- 按钮：内联 → 全宽堆叠
