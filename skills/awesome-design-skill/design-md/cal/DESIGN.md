# 设计系统：Cal.com

## 1. 视觉主题与氛围

Cal.com 的网站是单色克制的杰作——一个灰度世界，大胆来自纯粹自信的黑色文本在白色空间上，而非色彩。灵感来自 Uber 的极简美学，调色板刻意剥离色相：近黑色标题（`#242424`）、中灰次要文本（`#898989`）和纯白表面。颜色被视为外来物质——当它出现时（罕见的蓝色链接、绿色信任徽章），感觉像黑白照片中的受控强调。

Cal Sans 是品牌定制的几何展示字体，由 Mark Davis 设计，是视觉中心。字母在大尺寸时刻意间距极近，创造密集、建筑般的标题，感觉像是刻入页面。在 64px 和 48px，Cal Sans 标题使用字重 600 和紧凑的 1.10 行高——自信、压缩、立即识别。正文文本系统切换到 Inter，提供"坚如磐石"的可读性，补充 Cal Sans 的展示个性。排版配对创造清晰分工：Cal Sans 发言，Inter 解释。

提升系统对于极简网站来说非常复杂——11 个阴影定义使用多层阴影创造细腻的深度层级，结合环边框（`0px 0px 0px 1px`）、柔和漫射阴影和内嵌高光。这种以阴影为先的深度方法（而非以边框为先）给表面带来微妙的三维感，感觉现代和精致。基于 Framer 构建，border-radius 刻度从 2px 到 9999px（胶囊），Cal.com 平衡了几何精度与柔和、圆润的交互元素。

**关键特征：**
- 纯灰度品牌调色板——无品牌色，通过单色实现大胆
- Cal Sans 定制几何展示字体，默认字间距极紧
- 多层阴影系统（11 个定义）配环边框 + 漫射阴影 + 内嵌高光
- 标题使用 Cal Sans，正文使用 Inter——清晰的排版分工
- 宽广 border-radius 刻度从 2px 到 9999px（胶囊）——多变的圆角
- 白色画布配近黑（#242424）文本——最大对比，零装饰
- 产品截图作为主要视觉内容——日程安排 UI 自我推销
- 基于 Framer 平台构建

## 2. 色彩调色板与角色

### 主色
- **Charcoal**（`#242424`）：主要标题和按钮文本——Cal.com 标志性的近黑色，比纯黑温暖
- **Midnight**（`#111111`）：最深文本/覆盖颜色——50% 不透明度用于微妙覆盖
- **White**（`#ffffff`）：主要背景和表面——主导画布

### 次要与强调色
- **Link Blue**（`#0099ff`）：带下划线装饰的行内链接——系统中唯一的蓝色，严格保留给超链接
- **Focus Ring**（`#3b82f6` 50% 不透明度）：键盘聚焦指示器——仅用于无障碍，正常交互中不可见
- **Default Link**（`#0000ee`）：某些元素的浏览器默认链接颜色——未修改，表明开放性

### 表面与背景
- **Pure White**（`#ffffff`）：主页面背景和卡片表面
- **Light Gray**（约 `#f5f5f5`）：微妙的章节区分——几乎不可见的色调
- **Mid Gray**（`#898989`）：次要文本，描述，柔和标签

### 中性色与文本
- **Charcoal**（`#242424`）：标题，按钮，主 UI 文本
- **Midnight**（`#111111`）：高对比度链接和导航文本的深黑
- **Mid Gray**（`#898989`）：描述，次要标签，柔和内容
- **Pure Black**（`#000000`）：某些链接文本元素
- **Border Gray**（约 `rgba(34, 42, 53, 0.08–0.10)`）：使用环阴影代替 CSS 边框的基于阴影的边框

### 语义与强调色
- Cal.com 品牌元素刻意无色——"灰度品牌以强调大胆和专业"
- 产品 UI 截图显示颜色（日程安排界面中的蓝色、绿色），但营销网站本身保持单色
- 理念反映 Uber 方法：让内容承载颜色，框架保持中性

### 渐变系统
- 营销网站上无渐变——设计完全平面和单色
- 深度完全通过阴影实现，而非颜色过渡

## 3. 排版规则

### 字体家族
- **Display**：`Cal Sans`——Mark Davis 设计的定制几何无衬线。开源，可在 Google Fonts 和 GitHub 上获得。默认字间距极紧，专为大标题设计。有 6 个字符变体（Cc, j, t, u, 0, 1）
- **正文**：`Inter`——"坚如磐石"的标准正文字体。后备：`Inter Placeholder`
- **UI Light**：`Cal Sans UI Variable Light`——轻字重变体（300），-0.2px 字间距，用于柔和 UI 文本
- **UI Medium**：`Cal Sans UI Medium`——中等字重变体（500），用于强调标题
- **Mono**：`Roboto Mono`——用于代码块和技术内容
- **第三级**：`Matter Regular` / `Matter SemiBold` / `Matter Medium`——特定上下文的额外正文字体

### 层级

| 角色 | 字体 | 大小 | 字重 | 行高 | 字间距 | 备注 |
|------|------|------|--------|-------------|----------------|-------|
| Display Hero | Cal Sans | 64px | 600 | 1.10 | 0px | 最大冲击，紧凑默认间距 |
| Section Heading | Cal Sans | 48px | 600 | 1.10 | 0px | 大章节标题 |
| Feature Heading | Cal Sans | 24px | 600 | 1.30 | 0px | 特性块标题 |
| Sub-heading | Cal Sans | 20px | 600 | 1.20 | +0.2px | 较小尺寸可读性的正间距 |
| Sub-heading Alt | Cal Sans | 20px | 600 | 1.50 | 0px | 宽松行高变体 |
| Card Title | Cal Sans | 16px | 600 | 1.10 | 0px | 最小 Cal Sans 用法 |
| Caption Label | Cal Sans | 12px | 600 | 1.50 | 0px | Cal Sans 小标签 |
| Body Light | Cal Sans UI Light | 18px | 300 | 1.30 | -0.2px | 轻字重正文导言文本 |
| Body Light Standard | Cal Sans UI Light | 16px | 300 | 1.50 | -0.2px | 轻字重正文 |
| Caption Light | Cal Sans UI Light | 14px | 300 | 1.40–1.50 | -0.2 to -0.28px | 轻标题和描述 |
| UI Label | Inter | 16px | 600 | 1.00 | 0px | UI 按钮和导航标签 |
| Caption Inter | Inter | 14px | 500 | 1.14 | 0px | 小 UI 文本 |
| Micro | Inter | 12px | 500 | 1.00 | 0px | 最小 Inter 文本 |
| Code | Roboto Mono | 14px | 600 | 1.00 | 0px | 代码片段，技术文本 |
| Body Matter | Matter Regular | 14px | 400 | 1.14 | 0px | 备选正文（产品 UI） |

### 原则
- **大尺寸 Cal Sans，小尺寸 Inter**：Cal Sans 专门用于标题和展示——从不用于正文。系统严格执行此分工
- **默认紧凑，小时放宽**：Cal Sans 字母"故意间距极近"在大尺寸。在 20px 及以下，必须应用正字间距（+0.2px）以防止拥挤
- **字重 300 正文变体**：Cal Sans UI Variable Light 字重 300 创造优雅、通透的正文，与密集的 600 字重标题形成对比
- **字重 600 占主导**：几乎所有 Cal Sans 用法都在字重 600（半粗）——字体设计为此字重表现
- **轻文本负追踪**：Cal Sans UI Light 使用 -0.2px 至 -0.28px 字间距，微妙地收紧已经紧凑的字形

## 4. 组件样式

### 按钮
- **深色主要**：`#242424`（或 `#1e1f23`）背景，白色文本，6–8px 圆角。悬停：不透明度降至 0.7。标志性 CTA——白色上最深
- **白色/幽灵**：白色背景配阴影环边框，深色文本。使用多层阴影系统进行微妙提升
- **胶囊**：9999px 圆角用于圆形胶囊状动作和徽章
- **紧凑**：4px 内边距，小文本——产品 UI 内的实用动作
- **内嵌高光**：某些按钮具有 `rgba(255, 255, 255, 0.15) 0px 2px 0px inset`——微妙的内顶高光创造 3D 按压效果

### 卡片与容器
- **阴影卡片**：白色背景，多层阴影——`rgba(19, 19, 22, 0.7) 0px 1px 5px -4px, rgba(34, 42, 53, 0.08) 0px 0px 0px 1px, rgba(34, 42, 53, 0.05) 0px 4px 8px 0px`。环阴影（0px 0px 0px 1px）充当阴影边框
- **产品 UI 卡片**：日程安排界面截图显示在带阴影提升的卡片容器中
- **圆角**：标准卡片 8px，较大容器 12px，突出章节 16px
- **悬停**：可能微妙阴影加深或 scale 变换

### 输入框与表单
- **选择下拉**：白色背景，`#000000` 文本，1px solid `rgb(118, 118, 118)` 边框
- **聚焦**：使用 Framer 聚焦轮廓系统（`--framer-focus-outline`）
- **文本输入**：8px 圆角，标准边框处理
- **最小表单存在**：营销网站优先 CTA 按钮而非复杂表单

### 导航
- **顶部导航**：白色/透明背景，Cal Sans 链接近黑色
- **导航文本**：`#111111`（Midnight）用于主链接，`#000000` 用于强调
- **CTA 按钮**：导航中的深色主要——高对比度行动号召
- **移动端**：折叠为汉堡配简化导航
- **粘性**：滚动时固定

### 图片处理
- **产品截图**：大日程安排 UI 截图——产品是主要视觉
- **信任 logo**：水平信任条中的灰度公司 logo
- **纵横比**：产品 UI 截图使用宽横向
- **无装饰图像**：无插图、照片或抽象图形——纯粹产品 + 排版

## 5. 布局原则

### 间距系统
- **基础单位**：8px
- **刻度**：1px, 2px, 3px, 4px, 6px, 8px, 12px, 16px, 20px, 24px, 28px, 80px, 96px
- **章节内边距**：主要章节间 80px–96px 垂直（慷慨）
- **卡片内边距**：内部 12px–24px
- **组件间隙**：相关元素间 4px–8px
- **显著跳跃**：从 28px 到 80px——刻意差距强调章节级间距层次

### 网格与容器
- **最大宽度**：约 1200px 内容容器，居中
- **列模式**：全宽英雄，居中文本块，2-3 列特性网格
- **特性展示**：产品截图两侧配描述文本
- **断点**：98px, 640px, 768px, 810px, 1024px, 1199px——Framer 生成

### 留白哲学
- **慷慨章节间距**：章节间 80px–96px 创造透气、高端感觉
- **产品优先内容**：截图主导视觉空间——最小周围装饰
- **居中标题**：Cal Sans 标题居中，上下边距慷慨

### Border Radius 刻度
- **2px**：行内元素微妙圆角
- **4px**：小 UI 组件
- **6px–7px**：按钮，小卡片，图片
- **8px**：标准交互元素——按钮，输入框，图片
- **12px**：中等容器——链接，较大卡片，图片
- **16px**：大型章节容器
- **29px**：特殊圆角元素
- **100px**：大圆角——小元素上几乎圆形
- **1000px**：极大圆角
- **9999px**：完整胶囊形状——徽章，链接

## 6. 深度与提升

| 级别 | 处理方式 | 用途 |
|-------|-----------|-----|
| Level 0 (Flat) | 无阴影 | 页面画布，基本文本容器 |
| Level 1 (Inset) | `rgba(0,0,0,0.16) 0px 1px 1.9px 0px inset` | 按压/凹陷元素，输入槽 |
| Level 2 (Ring + Soft) | `rgba(19,19,22,0.7) 0px 1px 5px -4px, rgba(34,42,53,0.08) 0px 0px 0px 1px, rgba(34,42,53,0.05) 0px 4px 8px` | 卡片，容器——工作马阴影 |
| Level 3 (Ring + Soft Alt) | `rgba(36,36,36,0.7) 0px 1px 5px -4px, rgba(36,36,36,0.05) 0px 4px 8px` | 无环边框的备选卡片提升 |
| Level 4 (Inset Highlight) | `rgba(255,255,255,0.15) 0px 2px 0px inset` 或 `rgb(255,255,255) 0px 2px 0px inset` | 按钮内高光——3D 按压效果 |
| Level 5 (Soft Only) | `rgba(34,42,53,0.05) 0px 4px 8px` | 微妙环境阴影 |

### 阴影哲学
Cal.com 的阴影系统是设计中最复杂的元素——11 个阴影定义使用多层合成技术：
- **环边框**：`0px 0px 0px 1px` 阴影充当边框，完全避免 CSS `border`。这创造细线围合而不影响布局
- **漫射柔和阴影**：`0px 4px 8px` 5% 不透明度添加温和环境深度
- **锐利接触阴影**：`0px 1px 5px -4px` 70% 不透明度创造紧密的底边阴影用于固定
- **内嵌高光**：按钮顶部的白色内嵌阴影创造微妙 3D 斜面
- 阴影以逗号分隔堆栈组合——每个表面获得 2-3 个层叠阴影定义协同工作

### 装饰性深度
- 无渐变或发光效果
- 所有深度来自复杂的阴影合成系统
- 整体效果微妙但精准——表面感觉像放在桌子上的实体卡片

## 7. 宜与忌

### 宜
- 标题（24px+）专门使用 Cal Sans，正文永不使用——它是紧凑默认间距的展示字体
- 24px 以下使用 Cal Sans 时应用正字间距（+0.2px）——字体在小尺寸会拥挤
- 保持灰度调色板——大胆来自对比，而非颜色
- 卡片提升使用多层阴影系统——环阴影 + 漫射阴影 + 接触阴影
- 保持背景纯白——单色哲学需要干净画布
- 所有正文使用 Inter 字重 300–600——它是 Cal Sans 展示个性的可靠搭档
- 让产品截图成为视觉内容——无插图，无装饰图形
- 应用慷慨章节间距（80px–96px）——呼吸空间对高端感觉至关重要

### 忌
- 正文文本或 16px 以下文本使用 Cal Sans——它不为长阅读设计
- 添加品牌颜色——Cal.com 刻意灰度，颜色仅保留给链接和 UI 状态
- 当阴影可以实现相同围合时使用 CSS 边框——环阴影技术是系统方法
- 在小尺寸对 Cal Sans 应用负字间距——24px 以下需要正间距（+0.2px）
- 创造重、深阴影——Cal.com 的阴影微妙（5% 不透明度漫射）配锐利接触边缘
- 使用插图、抽象图形或装饰元素——视觉语言仅排版 + 产品 UI
- 混合 Cal Sans 字重——字体为字重 600 设计，其他字重破坏预期字符
- 章节间距减至 48px 以下——慷慨留白是高端单色美学的核心

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 关键变化 |
|------|-------|-------------|
| Mobile | <640px | 单列，英雄文本约 36px，堆叠特性，汉堡导航 |
| Tablet Small | 640px–768px | 某些元素开始 2 列 |
| Tablet | 768px–810px | 布局调整，更完整网格 |
| Tablet Large | 810px–1024px | 多列特性网格 |
| Desktop | 1024px–1199px | 完整布局，扩展导航 |
| Large Desktop | >1199px | 最大宽度容器，居中内容 |

### 触控目标
- 按钮：8px 圆角配舒适内边距（垂直 10px+）
- 导航链接：深色文本，间距充足
- 移动 CTA：全宽深色按钮，便于拇指访问
- 胶囊徽章：9999px 圆角创造大、可点击目标

### 折叠策略
- **导航**：完整水平导航 → 移动端汉堡
- **英雄**：64px Cal Sans 展示 → 移动端约 36px
- **特性网格**：多列 → 2 列 → 单堆叠列
- **产品截图**：在容器内按比例缩放，保持纵横比
- **章节间距**：从 80px–96px 减至移动端约 48px

### 图片行为
- 产品截图响应式缩放
- 信任 logo 在移动端重排为多行网格
- 无艺术指导变化——所有尺寸相同构图
- 图片使用 7px–12px border-radius 保持一致圆角

### 迭代指南
用此设计系统完善现有屏幕时：
1. 验证标题使用 Cal Sans 字重 600，正文使用 Inter——永不混合
2. 检查调色板纯灰度——如果看到品牌色，移除它们
3. 确保卡片提升使用多层阴影堆栈，而非 CSS 边框
4. 确认章节间距慷慨（80px+）——如果章节感觉拥挤，添加更多空间
5. 整体基调应感觉像干净、专业的日程安排工具——单色自信，无任何装饰修饰
