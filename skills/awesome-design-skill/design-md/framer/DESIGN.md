# 设计系统：Framer

## 1. 视觉主题与氛围

Framer 的网站是一个电影般的、专注于工具的深色画布，散发着由崇尚工艺的设计师所构建的设计工具的自信。整个体验沉浸在纯黑色中——不是温暖的炭灰或舒适的深灰，而是绝对的虚空（`#000000`），让每个元素、每张截图、每个排版装饰都感觉像漂浮在深空中。这个网站将其产品 UI 视为主角艺术，将高保真截图和交互式演示直接嵌入叙事流程中。

排版是标志性动作：GT Walsheim 配合极端紧凑的字间距（在 110px 展示文本上达到 -5.5px 的极端值）创造出压缩的、动感的、几乎像弹簧般紧绷的标题——就像被压缩的文字随时可能展开。过渡到 Inter 用于正文是无缝的，广泛使用 OpenType 特性（`cv01`、`cv05`、`cv09`、`cv11`、`ss03`、`ss07`），即使小字体也有精致的定制感。Framer Blue（`#0099ff`）被谨慎但果断地使用——作为链接颜色、边框点缀和微妙的环阴影——在无温度的黑色背景上创造出冷峻、带电的贯穿线。

整体效果是一个网页设计师的夜店：黑暗、精确、诱人，毫不掩饰地以产品为先。每个部分都为了展示工具能做什么而存在，网站本身就是概念验证。

**核心特征：**
- 纯黑（`#000000`）虚空画布——绝对的黑暗，不是温暖或灰调的
- GT Walsheim 展示字体配合极端负字间距（110px 时 -5.5px）
- Framer Blue（`#0099ff`）作为唯一强调色——冷峻、带电、精确
- 胶囊形按钮（40px–100px 圆角）——交互元素没有尖锐的角落
- 产品截图作为主角艺术——工具本身就是营销
- 毛玻璃按钮变体在深色表面使用 `rgba(255, 255, 255, 0.1)`
- Inter 上广泛使用 OpenType 特性以实现精致的微排版

## 2. 色彩系统与角色

### 主色
- **Pure Black**（`#000000`）：主背景，定义 Framer 深色优先身份的虚空画布
- **Pure White**（`#ffffff`）：深色表面上的主要文本颜色，强调背景上的按钮文本
- **Framer Blue**（`#0099ff`）：主要强调色——链接、边框、环阴影、交互高亮

### 辅助色与强调色
- **Muted Silver**（`#a6a6a6`）：次要文本，柔和标签，深色表面上变暗的描述
- **Near Black**（`#090909`）：提升的深色表面，阴影环颜色用于微妙的深度分离

### 表面与背景
- **Void Black**（`#000000`）：页面背景，主画布
- **Frosted White**（`rgba(255, 255, 255, 0.1)`）：半透明按钮背景，深色上的毛玻璃效果表面
- **Subtle White**（`rgba(255, 255, 255, 0.5)`）：稍不透明的毛玻璃元素用于悬停状态

### 中性色与文本
- **Pure White**（`#ffffff`）：标题文本，高强调正文
- **Muted Silver**（`#a6a6a6`）：正文，描述，次要信息
- **Ghost White**（`rgba(255, 255, 255, 0.6)`）：三级文本，深色表面上的占位符

### 语义与强调
- **Framer Blue**（`#0099ff`）：链接，交互边框，焦点环
- **Blue Glow**（`rgba(0, 153, 255, 0.15)`）：焦点环阴影，交互元素周围的微妙蓝色光晕
- **Default Link Blue**（`#0000ee`）：标准浏览器链接颜色（在内容区域谨慎使用）

### 渐变系统
- 没有明显的渐变使用——Framer 依赖纯平坦黑色表面，偶尔使用蓝色调光晕营造深度
- 产品截图后面使用极低透明度的 Framer Blue 微妙径向光晕效果

## 3. 排版规则

### 字体家族
- **Display**：`GT Walsheim Framer Medium` / `GT Walsheim Medium`——定制几何无衬线体，字重 500。后备字体：`GT Walsheim Framer Medium Placeholder`，系统无衬线体
- **Body/UI**：`Inter Variable` / `Inter`——可变无衬线体，具有广泛的 OpenType 特性。后备字体：`Inter Placeholder`、`-apple-system`、`system-ui`
- **Accent**：`Mona Sans`——GitHub 的开源字体，用于特定元素的超细字重（100）
- **Monospace**：`Azeret Mono`——配套等宽字体用于代码和技术标签
- **Rounded**：`Open Runde`——小型圆角配套字体用于微型标签

### 原则
- **压缩即个性**：GT Walsheim 的极端负字间距（110px 时 -5.5px）是定义性的排版姿态——标题感觉弹簧般紧绷、紧迫、几乎喘不过气
- **OpenType 极致主义**：Inter 同时部署 6+ 个 OpenType 特性（`cv01`、`cv05`、`cv09`、`cv11`、`ss03`、`ss07`），即使正文字号也能创造微妙的定制感
- **展示字体字重克制**：所有 GT Walsheim 使用都是字重 500（medium）——从不加粗，从不常规。这创造了自信但不咄咄逼人的展示调性
- **超紧行高**：展示文本行高 0.85 意味着字母几乎垂直重叠——有意的密度，适合手臂距离阅读

## 4. 组件样式

### 按钮
- **Frosted Pill**：`rgba(255, 255, 255, 0.1)` 背景，黑色文本（`#000000`），胶囊形状（40px 圆角）。毛玻璃效果按钮存在于深色表面——半透明、环境感、微妙
- **Solid White Pill**：`rgb(255, 255, 255)` 背景，黑色文本（`#000000`），完整胶囊形状（100px 圆角），内边距 `10px 15px`。主要 CTA——干净、深色上高对比度、不容错过
- **Ghost**：无可见背景，白色文本，仅依靠文本样式。悬停显示微妙的毛玻璃背景
- **Transition**：基于缩放的动画（矩阵变换，0.85 缩放因子），透明度过渡用于显示效果

### 卡片与容器
- **Dark Surface Card**：黑色或近黑（`#090909`）背景，`rgba(0, 153, 255, 0.15) 0px 0px 0px 1px` 蓝色环阴影边框，圆角（10px–15px 圆角）
- **Elevated Card**：多层阴影——`rgba(255, 255, 255, 0.1) 0px 0.5px 0px 0.5px`（微妙顶部高光）+ `rgba(0, 0, 0, 0.25) 0px 10px 30px`（深度环境阴影）
- **Product Screenshots**：全宽或在深色容器内带内边距，8px–12px 边框圆角用于软件 UI 预览
- **Hover**：Framer Blue 环阴影的微妙光晕增强，或毛玻璃表面的亮度变化

### 输入与表单
- 营销网站上表单存在感最小
- 输入框遵循深色主题：深色背景，微妙边框，白色文本
- 焦点状态：Framer Blue（`#0099ff`）环边框，`1px solid #0099ff`
- 占位符文本使用 `rgba(255, 255, 255, 0.4)`

### 导航
- **Dark floating nav bar**：黑色背景配毛玻璃效果，白色文本链接
- **Nav links**：Inter 15px，字重 400，白色文本配微妙悬停透明度变化
- **CTA button**：胶囊形状，白色或毛玻璃，位于导航右侧末端
- **Mobile**：折叠为汉堡菜单，保持深色主题
- **Sticky behavior**：滚动时导航固定在顶部

### 图片处理
- **Product screenshots as hero art**：全宽嵌入式 UI 截图配圆角（8px–12px）
- **Dark-on-dark composition**：截图放置在黑色背景上配微妙阴影以分离深度
- **16:9 和自定义宽高比**：产品演示填满其容器
- **无装饰性图像**：所有图像都是功能性的——展示工具、输出或工作流程

### 信任与社会证明
- 客户标志和推荐在深色表面使用柔和灰色
- 最小装饰——产品截图本身就是信任信号

## 5. 布局原则

### 间距系统
- **基础单位**：8px
- **比例**：1px, 2px, 3px, 4px, 5px, 6px, 8px, 10px, 12px, 15px, 20px, 30px, 35px
- **区块内边距**：大垂直间距（区块间 80px–120px）
- **卡片内边距**：内部内边距 15px–30px
- **组件间距**：相关元素间 8px–20px

### 网格与容器
- **最大宽度**：约 1200px 容器，居中
- **列模式**：全宽主视觉，双列特性区块，单列产品展示
- **非对称布局**：特性区块通常配对文本（40%）和截图（60%）

### 留白哲学
- **通过黑暗呼吸**：区块间慷慨的垂直间距——黑色背景意味着留白表现为虚空，在内容块之间创造戏剧性停顿
- **内部紧凑，外部宽敞**：单个组件紧密组合（紧行高、压缩文本）但漂浮在慷慨的周围空间中
- **产品优先密度**：截图区域允许密集和信息丰富，与稀疏的营销文本形成对比

### 边框圆角比例
- **1px**：微型元素，几乎方形的精确边缘
- **5px–7px**：小型 UI 元素，图像缩略图——微妙柔化
- **8px**：标准组件圆角——代码块、按钮、交互元素
- **10px–12px**：卡片，产品截图——舒适圆润
- **15px–20px**：大型容器，特性卡片——慷慨圆润
- **30px–40px**：导航胶囊，分页——明显圆润
- **100px**：完整胶囊形状——主要 CTA，标签元素

## 6. 深度与层级

| 层级 | 处理 | 用途 |
|------|------|------|
| Level 0（Flat） | 无阴影，纯黑表面 | 页面背景，空白区域 |
| Level 1（Ring） | `rgba(0, 153, 255, 0.15) 0px 0px 0px 1px` | 卡片边框，交互元素轮廓——Framer Blue 光晕环 |
| Level 2（Contained） | `rgb(9, 9, 9) 0px 0px 0px 2px` | 近黑环用于深色表面上微妙的包容 |
| Level 3（Floating） | `rgba(255, 255, 255, 0.1) 0px 0.5px 0px 0.5px, rgba(0, 0, 0, 0.25) 0px 10px 30px` | 提升卡片，浮动元素——微妙白色顶边高光 + 深度环境阴影 |

### 阴影哲学
Framer 的层级系统与传统浅色主题设计相反。不是在浅色背景上使用深色阴影，Framer 使用：
- **蓝色调环阴影**在极低透明度（0.15）用于包容——这是标志性动作，微妙地为每个有边框的元素打上品牌印记
- **白色边缘高光**（0.5px）在提升元素的顶边——模拟光线照射顶面
- **深度环境阴影**用于真正的浮动元素——`rgba(0, 0, 0, 0.25)` 大扩散（30px）

### 装饰性深度
- **Blue glow auras**：关键交互区域后面的微妙 Framer Blue（`#0099ff`）径向渐变
- **无背景模糊/玻璃态**：尽管有毛玻璃按钮效果，没有大量玻璃模糊使用——半透明通过简单的 rgba 透明度实现

## 7. 宜与忌

### 宜
- 使用纯黑（`#000000`）作为主背景——不是深灰，不是炭灰
- 在 GT Walsheim 展示文本上应用极端负字间距（-3px 到 -5.5px）
- 保持所有按钮胶囊形状（40px+ 圆角）——从不使用方形或微圆按钮
- 专门使用 Framer Blue（`#0099ff`）用于交互强调——链接、边框、焦点状态
- 在深色背景上部署 `rgba(255, 255, 255, 0.1)` 用于毛玻璃表面
- GT Walsheim 仅保持字重 500——medium 字重就是品牌
- 在 Inter 文本上使用广泛的 OpenType 特性（cv01, cv05, cv09, cv11, ss03, ss07）
- 让产品截图成为视觉焦点——工具自我营销
- 应用蓝色环阴影（`rgba(0, 153, 255, 0.15) 0px 0px 0px 1px`）用于卡片包容

### 忌
- 使用温暖的深色背景（不要 `#1a1a1a`、`#2d2d2d` 或棕调黑色）
- 对 GT Walsheim 展示文本应用加粗（700+）字重——仅限 medium 500
- 引入 Framer Blue 之外的额外强调色——这是一个单强调色系统
- 在非交互元素上使用大圆角（卡片使用 10px–15px，只有按钮使用 40px+）
- 添加装饰性图像、插图或图标——产品本身就是插图
- 在标题上使用正字间距——一切都是压缩的，负字间距
- 创建厚重的投影——深度通过微妙的环和最小环境传达
- 在内容区块后面放置浅色/白色背景——虚空是神圣的
- 使用衬线或展示字重字体——系统仅限几何无衬线体

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 关键变化 |
|------|------|----------|
| Mobile | <809px | 单列，堆叠特性区块，缩小主视觉文本（62px→40px），汉堡导航 |
| Tablet | 809px–1199px | 双列特性开始，导航链接部分可见，截图缩小 |
| Desktop | >1199px | 完整布局，扩展导航显示所有链接 + CTA，110px 展示主视觉，并排特性 |

### 触摸目标
- 胶囊按钮：最小 40px 高度配 10px 垂直内边距——超过 44px WCAG 最低要求
- 导航链接：15px 文本配慷慨内边距用于触摸可访问性
- 移动端 CTA 按钮：移动端全宽胶囊便于拇指触达

### 折叠策略
- **Navigation**：完整水平导航 → 移动端断点的汉堡菜单
- **Hero text**：110px 展示 → 85px → 62px → ~40px 跨断点，保持极端负字间距比例
- **Feature sections**：并排（文本 + 截图）→ 移动端垂直堆叠
- **Product screenshots**：在容器内响应式缩放，保持宽高比
- **Section spacing**：按比例减少——桌面 120px → 移动端 60px

### 图片行为
- 产品截图是响应式的，在容器边界内缩放
- 无艺术指导变化——跨断点使用相同裁剪
- 深色背景确保截图在任何尺寸下保持视觉冲击力
- 截图在用户滚动到视图时延迟加载

### 迭代指南
在使用此设计系统优化现有屏幕时：
1. 每次专注于一个组件——深色画布使每个元素都珍贵
2. 始终验证 GT Walsheim 标题的字间距——极端负字间距是不可协商的
3. 检查 Framer Blue 仅出现在交互元素上——从不作为装饰性背景或非链接文本颜色
4. 确保所有按钮都是胶囊形状——任何方角都会立即破坏 Framer 美学
5. 测试毛玻璃表面，检查它们是否精确使用 `rgba(255, 255, 255, 0.1)`——太不透明看起来像 bug，太透明会消失
