# 设计系统：xAI

## 1. 视觉主题与氛围

xAI 的网站是深色优先、等宽字体驱动的极简主义设计典范——这是一个由工程师构建的设计系统，他们深知克制是精致的终极形式。整个体验锚定在近乎黑色的背景（`#1f2228`）与纯白文字（`#ffffff`）之上，创造出高对比度、受终端启发的美学风格，彰显深厚的技术可信度。没有渐变、没有装饰性插图、没有争夺注意力的色彩点缀。这是一个通过留白来传达信息的网站。

字体系统分为两种精心挑选的字体。`GeistMono`（Vercel 的等宽字体）负责展示级标题，以 320px 的超大尺寸和 300 的字重呈现，同时也作为按钮字体使用，采用大写字母并配合 1.4px 的字间距。`universalSans` 负责所有正文和次要标题文本，提供干净、几何感的无衬线字体的声音。选择等宽字体作为展示字体是决定性的美学决策——它将 xAI 定位为基础设施而非消费产品，由那些在终端中生活的人构建。

间距系统基于 8px 基准网格，数值集中在较小端（4px、8px、24px、48px），反映出紧凑、信息聚焦的布局理念。圆角极小——网站几乎不进行任何圆角处理，保持锐利、建筑感的边缘。没有装饰性阴影、没有渐变、没有分层的高度感。深度纯粹通过对比和留白来传达。

**核心特征：**
- 纯深色主题：`#1f2228` 背景配 `#ffffff` 文字——没有灰色中间地带
- GeistMono 用于极端展示尺寸（320px，字重 300）——等宽字体作为奢华元素
- 大写等宽字体按钮，1.4px 字间距——技术感、权威感
- universalSans 用于正文（16px/1.5）和标题（30px/1.2）——干净的对比
- 零装饰元素：无阴影、无渐变、无彩色点缀
- 8px 间距网格，稀疏而精心设计的比例
- Heroicons SVG 图标系统——极简、功能性
- Tailwind CSS 配合任意值——优先实用工具的工程方法

## 2. 色彩调色板与角色

### 主色
- **Pure White**（`#ffffff`）：唯一的文字颜色、链接颜色和所有前景元素。在 xAI 的系统中，白色不是背景——它是声音。
- **Dark Background**（`#1f2228`）：画布。一种带有微妙蓝色基调的暖调近黑色（非纯黑，非中性灰）。这种特定色调避免了 `#000000` 的刺眼，同时保持深沉的黑暗感。

### 交互色
- **White Default**（`#ffffff`）：默认状态的链接和交互元素颜色。
- **White Muted**（`rgba(255, 255, 255, 0.5)`）：链接悬停状态——刻意变暗而非变亮，这既不寻常又独具特色。
- **White Subtle**（`rgba(255, 255, 255, 0.2)`）：边框、分隔线和细微的表面处理。
- **Ring Blue**（`rgb(59, 130, 246) / 0.5`）：Tailwind 的默认焦点环颜色（`--tw-ring-color`），用于键盘可访问性焦点状态。

### 表面与边框
- **Surface Elevated**（`rgba(255, 255, 255, 0.05)`）：微妙的卡片背景和悬停表面——几乎不可见的提升感。
- **Surface Hover**（`rgba(255, 255, 255, 0.08)`）：交互容器更明显的悬停状态。
- **Border Default**（`rgba(255, 255, 255, 0.1)`）：卡片、分隔线和容器的标准边框。
- **Border Strong**（`rgba(255, 255, 255, 0.2)`）：活动状态和按钮轮廓的强调边框。

### 功能色
- **Text Primary**（`#ffffff`）：所有标题、正文、标签。
- **Text Secondary**（`rgba(255, 255, 255, 0.7)`）：描述、说明文字、辅助文本。
- **Text Tertiary**（`rgba(255, 255, 255, 0.5)`）：弱化标签、占位符文本、时间戳。
- **Text Quaternary**（`rgba(255, 255, 255, 0.3)`）：禁用文本、非常细微的注释。

## 3. 字体规则

### 字体族
- **展示 / 按钮**：`GeistMono`，回退字体：`ui-monospace, SFMono-Regular, Roboto Mono, Menlo, Monaco, Liberation Mono, DejaVu Sans Mono, Courier New`
- **正文 / 标题**：`universalSans`，回退字体：`universalSans Fallback`

### 层级结构

| 角色 | 字体 | 尺寸 | 字重 | 行高 | 字间距 | 转换 | 备注 |
|------|------|------|------|------|--------|------|------|
| 展示级主标题 | GeistMono | 320px (20rem) | 300 | 1.50 | normal | none | 极端尺寸，等宽字体奢华感 |
| 章节标题 | universalSans | 30px (1.88rem) | 400 | 1.20（紧凑） | normal | none | 干净的无衬线对比 |
| 正文 | universalSans | 16px (1rem) | 400 | 1.50 | normal | none | 标准阅读文本 |
| 按钮 | GeistMono | 14px (0.88rem) | 400 | 1.43 | 1.4px | uppercase | 加宽字间距的等宽字体，权威感 |
| 标签 / 说明 | universalSans | 14px (0.88rem) | 400 | 1.50 | normal | none | 辅助文本 |
| 小号 / 元信息 | universalSans | 12px (0.75rem) | 400 | 1.50 | normal | none | 时间戳、脚注 |

### 原则
- **等宽字体作为展示**：GeistMono 在 320px 下不是噱头——它是品牌宣言。固定宽度字符在极端尺寸下创造出一种有节奏、建筑感的品质，这是任何比例字体都无法实现的。
- **大尺寸使用轻字重**：320px 标题使用 300 字重，防止等宽字体在极端尺寸下显得厚重或粗糙。它给人精确而非压迫的感觉。
- **大写按钮**：所有按钮文本都是大写 GeistMono，配合 1.4px 字间距。这为交互元素创造出独特的技术感，近乎命令行的美学风格。
- **无衬线字体用于阅读**：universalSans 在 16px/1.5 下为正文内容提供出色的可读性，与等宽字体展示元素形成干净的对比。
- **双字体清晰度**：系统精确使用两种字体，角色分明——等宽字体用于冲击力和交互，无衬线字体用于信息和阅读。没有重叠，没有模糊。

**幽灵 / 轮廓按钮**
- 背景：透明
- 文字：`#ffffff`
- 内边距：12px 24px
- 圆角：0px
- 边框：`1px solid rgba(255, 255, 255, 0.2)`
- 字体：GeistMono 14px 字重 400，大写，字间距 1.4px
- 悬停：`rgba(255, 255, 255, 0.05)` 背景

**文字链接**

- 移动端：汉堡菜单切换

### 徽章 / 标签
**等宽字体标签**
- 背景：透明
- 文字：`#ffffff`

- 边框：`1px solid rgba(255, 255, 255, 0.2)`
- 圆角：0px

### 输入框与表单

- 文字：`#ffffff`
- 占位符：`rgba(255, 255, 255, 0.3)`
- 标签：`rgba(255, 255, 255, 0.7)`，universalSans 14px

## 5. 布局原则

### 间距系统
- 基准单位：8px
- 比例：4px、8px、24px、48px
- 比例刻意稀疏——xAI 避免细微的间距区分，偏好通过留白创造清晰视觉层级的较大跳跃

### 网格与容器
- 最大内容宽度：约 1200px
- 主视觉区：全视口高度，配合巨大的居中等宽字体标题
- 功能区块：简单的垂直堆叠，配合充足的区块内边距（48px-96px）
- 桌面端功能描述采用双栏布局
- 全宽深色区块保持整体单一的深色背景

### 留白理念
- **极端慷慨**：xAI 使用大量留白。320px 标题配合 48px+ 的周围内边距创造出一种空旷感，这本身就是一种设计宣言——内容如此重要，需要呼吸空间。
- **垂直韵律优先于水平密度**：内容垂直堆叠，区块间有较大间隔，而非水平方向紧密排列。这创造出一种滚动驱动的体验，感觉深思熟虑且具有电影感。
- **无视觉噪音**：缺少装饰元素、区块间边框和色彩变化意味着留白是主要的结构工具。

### 断点
- 2000px、1536px、1280px、1024px、1000px、768px、640px
- Tailwind 响应式修饰符驱动断点行为

### 圆角比例
- 锐角（0px）：按钮、卡片、输入框的主要处理方式——默认值
- 微妙（4px）：偶尔用于次要容器的柔和处理
- 近零圆角理念是品牌极简主义身份的核心

## 6. 深度与高度

| 层级 | 处理方式 | 用途 |
|------|----------|------|
| 平面（Level 0） | 无阴影，无边框 | 页面背景、正文内容 |
| 表面（Level 1） | `rgba(255,255,255,0.03)` 背景 | 微妙的卡片表面 |
| 边框（Level 2） | `1px solid rgba(255,255,255,0.1)` 边框 | 卡片、容器、分隔线 |
| 活动（Level 3） | `1px solid rgba(255,255,255,0.2)` 边框 | 悬停状态、活动元素 |
| 焦点（可访问性） | 使用 `rgb(59,130,246)/0.5` 的环 | 键盘焦点指示器 |

**高度理念**：xAI 完全拒绝传统的基于阴影的高度系统。网站上没有任何 box-shadow。相反，深度通过三种机制传达：（1）基于不透明度的边框在交互时变亮，创造出元素"激活"而非"抬升"的感觉；（2）极其细微的背景不透明度变化（`0.03` 到 `0.08`），创造出几乎感知不到的表面差异；（3）320px 展示字体与 16px 正文文本之间的巨大尺寸对比，创造出字体深度。这是通过对比和不透明度实现的高度，而非通过模拟的光影。

## 7. 宜与忌

### 宜
- 使用 `#1f2228` 作为通用背景——绝不使用纯黑 `#000000`
- 所有展示标题和按钮文字使用 GeistMono——等宽字体即品牌
- 所有按钮标签应用大写 + 1.4px 字间距
- 巨大展示标题（320px）使用 300 字重
- 边框保持在 `rgba(255, 255, 255, 0.1)`——几乎不可见，但非不存在
- 悬停时将交互元素变暗至 `rgba(255, 255, 255, 0.5)`——与常规相反
- 保持锐角（0px 圆角）作为默认——极简主义精度
- 所有正文和阅读文本使用 universalSans，16px/1.5

### 忌
- 不要使用 box-shadow——xAI 没有任何阴影高度
- 不要引入白色和深色背景之外的颜色点缀——单色调色板是神圣的
- 不要使用大圆角（8px+、药丸形）——锐角边缘是有意为之
- 不要为标题使用粗字重（600-700）——仅使用 300-400 字重
- 不要在悬停时变亮元素——xAI 反而变暗至 0.5 不透明度
- 不要添加装饰性渐变、插图或色块
- 不要为按钮使用比例字体——GeistMono 大写是强制性的
- 不要使用彩色状态指示器，除非绝对必要——将一切保持在白/深色光谱中

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 关键变化 |
|------|------|----------|
| 移动端 | <640px | 单栏，主标题尺寸急剧缩小 |
| 小平板 | 640-768px | 内边距略微增加 |
| 平板 | 768-1024px | 双栏布局开始，标题尺寸增加 |
| 桌面 | 1024-1280px | 完整布局，充足留白 |
| 大屏 | 1280-1536px | 更宽的容器，更多呼吸空间 |
| 超大屏 | 1536-2000px | 最大内容宽度，居中 |
| 超宽屏 | >2000px | 内容保持居中，极端边距 |

### 触摸目标
- 按钮使用 12px 24px 内边距以舒适触摸
- 导航链接间隔 24px
- 最小点击目标：44px 高度
- 移动端：全宽按钮便于拇指触及

### 折叠策略
- 主视觉区：320px 等宽字体标题急剧缩小（移动端约 48px-64px）
- 导航：水平链接折叠为汉堡菜单
- 功能区块：双栏变为单栏堆叠
- 区块内边距：96px -> 48px -> 24px 跨断点变化
- 巨大展示字体是首先调整尺寸的元素——必须保持冲击力但不能溢出

### 图片行为
- 最少图片——网站依赖字体和留白
- 任何产品截图保持锐角
- 全宽媒体按视口比例缩放

### 迭代指南
1. 始终以 `#1f2228` 背景开始——绝不使用纯黑或灰色背景
2. GeistMono 用于展示和按钮，universalSans 用于其他一切——绝不混淆这些角色
3. 所有按钮必须是 GeistMono 大写配合 1.4px 字间距——这是不可协商的
4. 绝不使用阴影，永远——深度仅来自边框不透明度和背景不透明度
5. 边框始终是低不透明度的白色（0.1 默认，0.2 用于强调）
6. 悬停行为变暗至 0.5 不透明度而非变亮——与大多数系统相反
7. 默认使用锐角（0px）——仅在特定次要容器使用 4px
8. 正文文字使用 16px universalSans 配合 1.5 行高以舒适阅读
9. 充足的区块内边距（48px-96px）——让内容在黑暗中呼吸
10. 单色白/深色调色板是绝对的——除非功能关键，否则抵制添加颜色
