# 设计系统：Vercel

## 1. 视觉主题与氛围

Vercel 的网站是将开发者基础设施隐形化的视觉宣言 —— 一个极度克制的设计系统，近乎哲学境界。页面以纯白色（`#ffffff`）为主，搭配近乎黑色（`#171717`）的文字，营造出画廊般的空灵感，每个元素都必须证明其存在的价值。这不是作为装饰的极简主义，而是作为工程原则的极简主义。Geist 设计系统将界面视作编译器对待代码 —— 每一个不必要的 token 都被剥离，直到只剩下结构本身。

定制的 Geist 字体家族是皇冠上的明珠。Geist Sans 在展示尺寸下使用激进的负字间距（-2.4px 至 -2.88px），创造出紧凑、紧迫且经过工程化处理的标题感 —— 就像被压缩用于生产环境的代码。在正文尺寸下，字距会适当放松，但几何精确性依然保持。Geist Mono 作为等宽字体伴侣，用于代码、终端输出和技术标签。两种字体都全局启用 OpenType `"liga"`（连字），增添了一层精致的字排层次，回馈仔细阅读的读者。

Vercel 与其他单色设计系统的区别在于其"阴影即边框"的设计哲学。Vercel 不使用传统的 CSS 边框，而是使用 `box-shadow: 0px 0px 0px 1px rgba(0,0,0,0.08)` —— 一种零偏移、零模糊、1px 扩展的阴影，创造出类似边框的线条，却不受盒模型限制。这种技术使边框存在于阴影层，能够实现更平滑的过渡、圆角不被裁切，以及比传统边框更柔和的视觉重量。整个深度系统建立在分层的多值阴影堆栈之上，每一层都有特定用途：一层用于边框，一层用于柔和提升，一层用于环境深度。

**核心特征：**
- Geist Sans 采用极端负字间距（展示尺寸 -2.4px 至 -2.88px）—— 文字如同压缩的基础设施
- Geist Mono 用于代码和技术标签，全局启用 OpenType `"liga"`
- 阴影即边框技术：`box-shadow 0px 0px 0px 1px` 全面取代传统边框
- 多层阴影堆栈营造细腻深度（单条声明中包含边框 + 提升层 + 环境层）
- 近纯白画布配合 `#171717` 文字 —— 不是纯黑，创造微对比度的柔和感
- 工作流专属强调色：Ship Red（`#ff5b4f`）、Preview Pink（`#de1d8d`）、Develop Blue（`#0a72ef`）
- Focus ring 系统使用 `hsla(212, 100%, 48%, 1)` —— 高饱和度蓝色确保无障碍访问
- 药丸形徽章（9999px 圆角）配合着色背景作为状态指示器

## 2. 色彩系统与角色

### 主色
- **Vercel Black**（`#171717`）：主要文字、标题、深色表面背景。不是纯黑 —— 微暖色调避免刺眼。
- **Pure White**（`#ffffff`）：页面背景、卡片表面、深色按钮上的文字。
- **True Black**（`#000000`）：辅助使用，`--geist-console-text-color-default`，用于特定控制台/代码场景。

### 工作流强调色
- **Ship Red**（`#ff5b4f`）：`--ship-text`，"部署到生产环境"工作流步骤 —— 温暖、紧迫的珊瑚红。
- **Preview Pink**（`#de1d8d`）：`--preview-text`，预览部署工作流 —— 鲜艳的品红粉。
- **Develop Blue**（`#0a72ef`）：`--develop-text`，开发工作流 —— 明亮、专注的蓝色。

### 控制台/代码颜色
- **Console Blue**（`#0070f3`）：`--geist-console-text-color-blue`，语法高亮蓝色。
- **Console Purple**（`#7928ca`）：`--geist-console-text-color-purple`，语法高亮紫色。
- **Console Pink**（`#eb367f`）：`--geist-console-text-color-pink`，语法高亮粉色。

### 交互色
- **Link Blue**（`#0072f5`）：主要链接颜色，带下划线装饰。
- **Focus Blue**（`hsla(212, 100%, 48%, 1)`）：`--ds-focus-color`，交互元素的 focus ring。
- **Ring Blue**（`rgba(147, 197, 253, 0.5)`）：`--tw-ring-color`，Tailwind ring 工具类。

### 中性色阶
- **Gray 900**（`#171717`）：主要文字、标题、导航文字。
- **Gray 600**（`#4d4d4d`）：次要文字、描述性文案。
- **Gray 500**（`#666666`）：三级文字、弱化链接。
- **Gray 400**（`#808080`）：占位符文字、禁用状态。
- **Gray 100**（`#ebebeb`）：边框、卡片轮廓、分隔线。
- **Gray 50**（`#fafafa`）：微妙表面着色、内阴影高光。

### 表面与遮罩
- **Overlay Backdrop**（`hsla(0, 0%, 98%, 1)`）：`--ds-overlay-backdrop-color`，模态框/对话框背景。
- **Selection Text**（`hsla(0, 0%, 95%, 1)`）：`--geist-selection-text-color`，文字选区高亮。
- **Badge Blue Bg**（`#ebf5ff`）：药丸徽章背景，着蓝色表面。
- **Badge Blue Text**（`#0068d6`）：药丸徽章文字，深蓝色确保可读性。

### 阴影与深度
- **Border Shadow**（`rgba(0, 0, 0, 0.08) 0px 0px 0px 1px`）：标志性阴影 —— 取代传统边框。
- **Subtle Elevation**（`rgba(0, 0, 0, 0.04) 0px 2px 2px`）：卡片的最小提升。
- **Card Stack**（`rgba(0,0,0,0.08) 0px 0px 0px 1px, rgba(0,0,0,0.04) 0px 2px 2px, rgba(0,0,0,0.04) 0px 8px 8px -8px, #fafafa 0px 0px 0px 1px`）：完整多层卡片阴影。
- **Ring Border**（`rgb(235, 235, 235) 0px 0px 0px 1px`）：标签页和图片的浅灰色环边框。

## 3. 字体规则

### 字体家族
- **主字体**：`Geist`，回退字体：`Arial, Apple Color Emoji, Segoe UI Emoji, Segoe UI Symbol`
- **等宽字体**：`Geist Mono`，回退字体：`ui-monospace, SFMono-Regular, Roboto Mono, Menlo, Monaco, Liberation Mono, DejaVu Sans Mono, Courier New`
- **OpenType 特性**：所有 Geist 文本全局启用 `"liga"`；特定标题使用 `"tnum"` 用于表格数字。

### 设计原则
- **压缩即身份**：Geist Sans 在展示尺寸使用 -2.4px 至 -2.88px 的字间距 —— 这是所有主流设计系统中最为激进的负字距。这创造出一种被"压缩"的文本感，就像为生产环境优化过的代码。字距随尺寸递减逐渐放松：32px 时 -1.28px，24px 时 -0.96px，16px 时 -0.32px，14px 时恢复正常。
- **连字无处不在**：每个 Geist 文本元素都启用 OpenType `"liga"`。连字不是装饰 —— 它们是结构性的，创造更紧密、更高效的字形组合。
- **三种字重，严格分工**：400（正文/阅读），500（UI/交互），600（标题/强调）。不使用粗体（700），仅微型徽章除外。这种狭窄的字重范围通过尺寸和字距创造层级，而非字重。
- **等宽字体作为身份标识**：Geist Mono 大写形式配合 `"tnum"` 或 `"liga"` 作为"开发者控制台"的声音 —— 紧凑的技术标签将营销网站与产品连接起来。

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 尺度：1px, 2px, 3px, 4px, 5px, 6px, 8px, 10px, 12px, 14px, 16px, 32px, 36px, 40px
- 值得注意：从 16px 直接跳到 32px —— 主尺度中没有 20px 或 24px

### 网格与容器
- 最大内容宽度：约 1200px
- Hero：居中单列，顶部留有充足内边距
- 功能区块：2–3 列网格布局用于卡片
- 全宽分隔线使用 `border-bottom: 1px solid #171717`
- 代码/仪表板截图全宽展示或带边框容器

### 留白哲学
- **画廊式空灵**：区块间巨大的垂直内边距（80px–120px+）。留白本身就是设计 —— 它传达出 Vercel 无需证明什么，也无需隐藏什么。
- **压缩文字，舒展空间**：标题上激进的负字间距被周围慷慨的留白所平衡。文字是密集的；周围的空间是广阔的。
- **区块节奏**：白色区块与白色区块交替 —— 区块间没有颜色变化。分隔仅来自边框（阴影边框）和间距。

### 圆角尺度
- 微型（2px）：行内代码片段、小跨度
- 微妙（4px）：小容器
- 标准（6px）：按钮、链接、功能元素
- 舒适（8px）：卡片、列表项
- 图片（12px）：特色卡片、图片容器（顶部圆角）
- 大型（64px）：标签导航药丸
- 特大（100px）：大型导航链接
- 完整药丸（9999px）：徽章、状态药丸、标签
- 圆形（50%）：菜单切换、头像容器

## 6. 深度与提升

| 层级 | 处理方式 | 用途 |
|------|----------|------|
| 扁平（Level 0） | 无阴影 | 页面背景、文本块 |
| 环形（Level 1） | `rgba(0,0,0,0.08) 0px 0px 0px 1px` | 大多数元素的阴影即边框 |
| 浅环（Level 1b） | `rgb(235,235,235) 0px 0px 0px 1px` | 标签页、图片的较浅环形 |
| 微妙卡片（Level 2） | 环形 + `rgba(0,0,0,0.04) 0px 2px 2px` | 标准卡片，最小提升 |
| 完整卡片（Level 3） | 环形 + 微妙层 + `rgba(0,0,0,0.04) 0px 8px 8px -8px` + 内层 `#fafafa` 环 | 特色卡片、高亮面板 |
| 焦点（无障碍） | `2px solid hsla(212, 100%, 48%, 1)` 轮廓 | 所有交互元素的键盘焦点 |

**阴影哲学**：Vercel 拥有当代 Web 设计中最为精妙的阴影系统。与传统 Material Design 意义上用阴影表示提升不同，Vercel 使用多值阴影堆栈，每一层都有明确的建筑学目的：一层创造"边框"（0px 扩展，1px），另一层添加环境柔和感（2px 模糊），另一层处理远距离深度（8px 模糊配合负扩展），内层环（`#fafafa`）创造出使卡片从内部"发光"的微妙高光。这种分层方法让卡片感觉是被构建的，而非漂浮的。

### 装饰性深度
- Hero 渐变：柔和、淡彩多彩渐变冲洗置于 hero 内容之后（几乎不可见，营造氛围）
- 区块边框：主要区块间 `1px solid #171717`（完整深色线条）
- 无背景色变化 —— 深度完全来自阴影分层和边框对比

## 7. 做与不做

### 做
- 在展示尺寸使用带激进负字间距的 Geist Sans（48px 时 -2.4px 至 -2.88px）
- 使用阴影即边框（`0px 0px 0px 1px rgba(0,0,0,0.08)`）替代传统 CSS 边框
- 在所有 Geist 文本上启用 `"liga"` —— 连字是结构性的，非可选项
- 使用三字重系统：400（正文），500（UI），600（标题）
- 仅在工作流上下文中应用工作流强调色（Red/Pink/Blue）
- 为卡片使用多层阴影堆栈（边框 + 提升层 + 环境层 + 内层高光）
- 保持调色板非彩色 —— 从 `#171717` 到 `#ffffff` 的灰色构成系统
- 使用 `#171717` 而非 `#000000` 作为主要文字 —— 微暖色调很重要

### 不做
- 不要在 Geist Sans 上使用正字间距 —— 始终为负或零
- 不要在正文上使用 700 字重（粗体）—— 600 是上限，仅用于标题
- 不要在卡片上使用传统 CSS `border` —— 使用阴影边框技术
- 不要在 UI 界面中引入暖色（橙色、黄色、绿色）
- 不要装饰性地使用工作流强调色（Ship Red、Preview Pink、Develop Blue）
- 不要使用重阴影（> 0.1 不透明度）—— 阴影系统是耳语级别的
- 不要增加正文字间距 —— Geist 设计为紧凑运行
- 不要在主要操作按钮上使用药丸圆角（9999px）—— 药丸仅用于徽章/标签
- 不要在卡片阴影中跳过内层 `#fafafa` 环 —— 它是让系统运转的发光层

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 关键变化 |
|------|------|----------|
| 小手机 | <400px | 紧凑单列，最小内边距 |
| 手机 | 400–600px | 标准移动端，堆叠布局 |
| 小平板 | 600–768px | 2 列网格开始出现 |
| 平板 | 768–1024px | 完整卡片网格，扩展内边距 |
| 小桌面 | 1024–1200px | 标准桌面布局 |
| 桌面 | 1200–1400px | 完整布局，最大内容宽度 |
| 大桌面 | >1400px | 居中，慷慨边距 |

### 触摸目标
- 按钮使用舒适的内边距（垂直 8px–16px）

### 迭代指南
1. 始终使用阴影即边框替代 CSS 边框 —— `0px 0px 0px 1px rgba(0,0,0,0.08)` 是基础
2. 字间距随字体尺寸缩放：48px 时 -2.4px，32px 时 -1.28px，24px 时 -0.96px，14px 时正常
