# 设计系统：Notion

## 1. 视觉主题与氛围

Notion 的网站体现了工具本身的哲学：一块不会妨碍你的空白画布。设计系统建立在温暖的灰色调而非冷灰色之上，营造出一种独特的亲和极简主义，感觉像优质纸张而非冰冷的玻璃。页面画布是纯白色（`#ffffff`），但文字并非纯黑色——而是一种温暖的近黑色（`rgba(0,0,0,0.95)`），不经意地柔化了阅读体验。暖灰色阶（`#f6f5f4`、`#31302e`、`#615d59`、`#a39e98`）带有微妙的黄棕底色，赋予界面一种可触摸的、近乎模拟的温暖感。

定制的 NotionInter 字体（修改版 Inter）是系统的支柱。在展示尺寸（64px）下，它使用强烈的负字间距（-2.125px），创造出紧凑而精确的标题感。字重范围比典型系统更广：400 用于正文，500 用于 UI 元素，600 用于半粗体标签，700 用于展示标题。OpenType 特性 `"lnum"`（ lining numerals）和 `"locl"`（localized forms）在较大文字上启用，增添了排版的精致感，值得细细品读。

让 Notion 视觉语言独特的是其边框哲学。Notion 不使用粗边框或阴影，而是采用超薄的 `1px solid rgba(0,0,0,0.1)` 边框——这些边框如同低语，几乎难以察觉的分隔线，创造出没有重量感的结构。阴影系统同样克制：多层堆叠的累计透明度从未超过 0.05，创造出能被感知而非被看见的深度。

**核心特征：**
- NotionInter（修改版 Inter）在展示尺寸使用负字间距（64px 时为 -2.125px）
- 温暖中性色板：灰色带有黄棕底色（`#f6f5f4` 暖白，`#31302e` 暖黑）
- 通过 `rgba(0,0,0,0.95)` 实现的近黑色文字——非纯黑，创造微妙温暖感
- 超薄边框：全站使用 `1px solid rgba(0,0,0,0.1)`——轻如低语的分隔
- 多层阴影堆叠，透明度低于 0.05，营造若有若无的深度
- Notion Blue（`#0075de`）作为 CTA 和交互元素的唯一强调色
- 胶囊徽章（9999px 圆角）配有淡蓝色背景，用于状态指示
- 8px 基础间距单位，采用有机、非刚性的比例

## 2. 色彩系统与角色

### 主色
- **Notion Black**（`rgba(0,0,0,0.95)` / `#000000f2`）：主要文字、标题、正文。95% 的透明度柔化了纯黑，同时不影响可读性。
- **Pure White**（`#ffffff`）：页面背景、卡片表面、蓝色按钮上的文字。
- **Notion Blue**（`#0075de`）：主要 CTA、链接颜色、交互强调色——核心 UI 界面中唯一的饱和色。

### 品牌次要色
- **Deep Navy**（`#213183`）：次要品牌色，少量用于强调和深色功能区块。
- **Active Blue**（`#005bab`）：按钮激活/按下状态——Notion Blue 的深色变体。

### 暖灰色阶
- **Warm White**（`#f6f5f4`）：背景表面淡色、区块交替、微妙卡片填充。黄色底色是关键。
- **Warm Dark**（`#31302e`）：深色表面背景、深色区块文字。比标准灰色更温暖。
- **Warm Gray 500**（`#615d59`）：次要文字、描述、弱化标签。
- **Warm Gray 300**（`#a39e98`）：占位文字、禁用状态、说明文字。

### 语义强调色
- **Teal**（`#2a9d99`）：成功状态、正面指示。
- **Green**（`#1aae39`）：确认、完成徽章。
- **Orange**（`#dd5b00`）：警告状态、注意指示。
- **Pink**（`#ff64c8`）：装饰性强调、功能亮点。
- **Purple**（`#391c57`）：高级功能、深沉强调。
- **Brown**（`#523410`）：大地色强调、温暖功能区块。

### 交互色
- **Link Blue**（`#0075de`）：主要链接颜色，悬停时显示下划线。
- **Link Light Blue**（`#62aef0`）：深色背景上的浅色链接变体。
- **Focus Blue**（`#097fe8`）：交互元素的焦点环。
- **Badge Blue Bg**（`#f2f9ff`）：胶囊徽章背景、淡蓝色表面。
- **Badge Blue Text**（`#097fe8`）：胶囊徽章文字，深蓝色以确保可读性。

### 阴影与深度
- **Card Shadow**（`rgba(0,0,0,0.04) 0px 4px 18px, rgba(0,0,0,0.027) 0px 2.025px 7.84688px, rgba(0,0,0,0.02) 0px 0.8px 2.925px, rgba(0,0,0,0.01) 0px 0.175px 1.04062px`）：多层卡片阴影。
- **Deep Shadow**（`rgba(0,0,0,0.01) 0px 1px 3px, rgba(0,0,0,0.02) 0px 3px 7px, rgba(0,0,0,0.02) 0px 7px 15px, rgba(0,0,0,0.04) 0px 14px 28px, rgba(0,0,0,0.05) 0px 23px 52px`）：五层深度阴影，用于模态框和特色内容。
- **Whisper Border**（`1px solid rgba(0,0,0,0.1)`）：标准分隔边框——卡片、分隔线、区块。

## 3. 排版规则

### 字体家族
- **主字体**：`NotionInter`，回退字体：`Inter, -apple-system, system-ui, Segoe UI, Helvetica, Apple Color Emoji, Arial, Segoe UI Emoji, Segoe UI Symbol`
- **OpenType 特性**：展示和标题文字启用 `"lnum"`（lining numerals）和 `"locl"`（localized forms）。

### 原则
- **大尺寸压缩**：NotionInter 在展示尺寸 64px 时使用 -2.125px 字间距，逐渐放松至 26px 时的 -0.625px 和 16px 时的 normal。压缩在标题处创造密度，同时在正文尺寸保持可读性。
- **四字重系统**：400（正文/阅读），500（UI/交互），600（强调/导航），700（标题/展示）。相比大多数系统更广的字重范围允许更细腻的层级。
- **温暖缩放**：行高随尺寸增加而收紧——正文（16px）为 1.50，副标题为 1.23-1.27，展示为 1.00-1.04。这创造了更紧凑、更有冲击力的标题。
- **徽章微调**：12px 徽章文字使用正字间距（0.125px）——系统中唯一的正字间距，创造更宽、更易读的小号文字。

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 比例：2px、3px、4px、5px、6px、7px、8px、11px、12px、14px、16px、24px、32px
- 非刚性有机比例，包含分数值（5.6px、6.4px）用于微调

### 网格与容器
- 最大内容宽度：约 1200px
- 英雄区：居中单列，顶部留有充足内边距（80-120px）
- 功能区块：2-3 列网格用于卡片
- 全宽暖白色（`#f6f5f4`）区块背景用于交替
- 代码/仪表板截图作为容器，配低语边框

### 留白哲学
- **充足的垂直节奏**：主要区块之间 64-120px。Notion 以巨大的垂直内边距让内容呼吸。
- **温暖交替**：白色区块与暖白色（`#f6f5f4`）区块交替，创造柔和的视觉节奏，没有刺眼的色彩断层。
- **内容优先密度**：正文文本块紧凑（行高 1.50），但周围有充足的边距，在留白的海洋中创造可阅读的内容岛屿。

### 圆角比例
- 微型（4px）：按钮、输入框、功能性交互元素
- 微妙（5px）：链接、列表项、菜单项
- 标准（8px）：小卡片、容器、行内元素
- 舒适（12px）：标准卡片、功能容器、图片顶部
- 大型（16px）：英雄卡片、特色内容、推广区块
- 完整胶囊（9999px）：徽章、胶囊、状态指示器
- 圆形（100%）：标签指示器、头像

## 6. 深度与层次

| 层级 | 处理方式 | 用途 |
|------|----------|------|
| 扁平（Level 0） | 无阴影，无边框 | 页面背景、文本块 |
| 低语（Level 1） | `1px solid rgba(0,0,0,0.1)` | 标准边框、卡片轮廓、分隔线 |
| 柔和卡片（Level 2） | 4 层阴影堆叠（最大透明度 0.04） | 内容卡片、功能块 |
| 深度卡片（Level 3） | 5 层阴影堆叠（最大透明度 0.05，52px 模糊） | 模态框、特色面板、英雄元素 |
| 焦点（无障碍） | `2px solid var(--focus-color)` 轮廓 | 所有交互元素的键盘焦点 |

**阴影哲学**：Notion 的阴影系统使用多层极低透明度（0.01 到 0.05）的阴影层，累积成柔和、自然的立体感。4 层卡片阴影从 1.04px 到 18px 模糊，创造出深度的渐变而非单一硬阴影。5 层深度阴影延伸至 52px 模糊，透明度 0.05，产生环境光遮蔽效果，感觉像自然光而非计算机生成的深度。这种分层方法让元素感觉嵌入页面而非漂浮其上。

### 装饰性深度
- 英雄区：装饰性角色插图（俏皮、手绘风格）
- 区块交替：白色到暖白色（`#f6f5f4`）背景切换
- 无硬性区块边框——分隔来自背景色变化和间距

## 7. 响应式行为

### 断点
| 名称 | 宽度 | 关键变化 |
|------|------|----------|
| 小型移动端 | <400px | 紧凑单列，最小内边距 |
| 移动端 | 400-600px | 标准移动端，堆叠布局 |
| 小型平板 | 600-768px | 开始 2 列网格 |
| 平板 | 768-1080px | 完整卡片网格，扩展内边距 |
| 小型桌面 | 1080-1200px | 标准桌面布局 |
| 桌面 | 1200-1440px | 完整布局，最大内容宽度 |
| 大型桌面 | >1440px | 居中，充足边距 |

### 触控目标
- 按钮使用舒适的内边距（垂直 8px-16px）
- 导航链接 15px，间距充足
- 胶囊徽章有 8px 水平内边距用于触控目标
- 移动菜单切换使用标准汉堡按钮

### 折叠策略
- 英雄区：64px 展示 -> 缩放至 40px -> 移动端 26px，保持比例字间距
- 导航：水平链接 + 蓝色 CTA -> 汉堡菜单
- 功能卡片：3 列 -> 2 列 -> 单列堆叠
- 产品截图：保持宽高比，响应式图片
- 信任栏标志：网格 -> 移动端水平滚动
- 页脚：多列 -> 堆叠单列
- 区块间距：80px+ -> 移动端 48px

### 图片行为
- 工作区截图在所有尺寸保持低语边框
- 英雄插图按比例缩放
- 产品截图使用响应式图片，保持一致圆角
- 全宽暖白色区块保持边缘到边缘处理

## 8. 无障碍与状态

### 焦点系统
- 所有交互元素都有可见的焦点指示器
- 焦点轮廓：`2px solid` 配焦点色 + 阴影级别 200
- 所有交互组件支持 Tab 导航
- 高对比度文字：白底近黑色超过 WCAG AAA（>14:1 比率）

### 交互状态
- **默认**：标准外观，配低语边框
- **悬停**：文字颜色变化，按钮 scale(1.05)，链接下划线
- **激活/按下**：scale(0.9) 变换，更深的背景变体
- **焦点**：蓝色轮廓环，配阴影增强
- **禁用**：暖灰色（`#a39e98`）文字，降低透明度

### 颜色对比度
- 主要文字（rgba(0,0,0,0.95)）在白色上：~18:1 比率
- 次要文字（#615d59）在白色上：~5.5:1 比率（WCAG AA）
- 蓝色 CTA（#0075de）在白色上：~4.6:1 比率（大文字 WCAG AA）
- 徽章文字（#097fe8）在徽章背景（#f2f9ff）上：~4.5:1 比率（大文字 WCAG AA）

### 迭代指南
1. 始终使用暖灰色——Notion 的灰色带有黄棕底色（#f6f5f4、#31302e、#615d59、#a39e98），从非蓝灰色
2. 字间距随字号缩放：64px 时 -2.125px，54px 时 -1.875px，26px 时 -0.625px，16px 时 normal
3. 四种字重：400（阅读），500（交互），600（强调），700（宣告）
4. 边框是低语：1px solid rgba(0,0,0,0.1)——从不过重
5. 阴影使用 4-5 层，单独透明度从不超过 0.05
6. 暖白色（#f6f5f4）区块背景对视觉节奏至关重要
7. 胶囊徽章（9999px）用于状态/标签，4px 圆角用于按钮和输入框
8. Notion Blue（#0075de）是核心 UI 中唯一的饱和色——谨慎用于 CTA 和链接
