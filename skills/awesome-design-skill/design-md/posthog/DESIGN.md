# 设计系统：PostHog

## 1. 视觉主题与氛围

PostHog 的网站给人的感觉就像是一份逃逸到野外的初创公司内部 wiki —— 温暖、不拘一格、刻意反传统。背景不是开发者工具常见的纯净白色或黑暗空洞，而是一种带有 sage 色调的温暖奶油色（`#fdfdf8`），让每个表面都具有手工制作的纸张质感。颜色偏向土色橄榄绿和柔和的 sage，而非 SaaS 世界常见的蓝色和紫色。就像有人在一个舒适的小棚屋里设计了一个开发者分析平台。

个性是主角：手绘的刺猬插图、古怪的动作人偶和俏皮的图像取代了 B2B SaaS 典型的库存照片和抽象渐变。IBM Plex Sans Variable 作为字体基础 —— 这是一款具有真正技术可信度的字体（由 IBM 创建，广泛用于开发者场景），在这里用于标题的粗体字重（700、800）以及正文的大行高。字体表达着"我们是认真的工程师"，而周围的一切却说"但我们不会太把自己当回事"。

交互设计承载着同样的精神：悬停状态会闪烁 PostHog Orange（`#F54E00`）文本 —— 一种隐藏的品牌色，静止时不会出现，但在交互时会带来惊喜。深色近黑色按钮（`#1e1f23`）在悬停时使用透明度降低而非颜色变化，激活状态会略微缩放。边框系统使用带有 sage 色调的灰色（`#bfc1b7`），与橄榄色文本调色板和谐统一。基于 Tailwind CSS、Radix UI 和 shadcn/ui 原语构建，技术基础现代且组件驱动，但视觉输出却固执地独特。

**核心特征：**
- 温暖的 sage/橄榄色调色板，而非传统蓝色 —— 土色调且亲切
- IBM Plex Sans Variable 字体，用于标题的粗体字重（700/800），行高 1.50+
- 隐藏的品牌橙色（`#F54E00`）仅在悬停交互时出现 —— 一个令人愉悦的惊喜
- 手绘刺猬插图和俏皮图像 —— 刻意反传统
- Sage 色调边框（`#bfc1b7`）和背景（`#eeefe9`）创造统一的温暖绿色系统
- 深色近黑色 CTA（`#1e1f23`）带有基于透明度的悬停状态
- 内容密集的编辑式布局 —— 网站读起来像杂志，而非典型的落地页
- Tailwind CSS + Radix UI + shadcn/ui 组件架构

## 2. 调色板与角色

### 主色
- **Olive Ink**（`#4d4f46`）：主要文本颜色 —— 一种独特的橄榄灰色，赋予所有文本温暖的土色调
- **Deep Olive**（`#23251d`）：链接文本和高强调标题 —— 带有绿色底色的近黑色
- **PostHog Orange**（`#F54E00`）：隐藏的品牌强调色 —— 仅出现在悬停状态，一种充满活力的橙色惊喜

### 次要与强调色
- **Amber Gold**（`#F7A501`）：深色按钮上的次要悬停强调色 —— 与橙色搭配的温暖金色
- **Gold Border**（`#b17816`）：特殊按钮边框 —— 用于特色 CTA 的琥珀金色
- **Focus Blue**（`#3b82f6`）：焦点环颜色（Tailwind 默认） —— 系统中唯一的蓝色，保留用于无障碍访问

### 表面与背景
- **Warm Parchment**（`#fdfdf8`）：主要页面背景 —— 带有黄绿色底色的温暖近白色
- **Sage Cream**（`#eeefe9`）：输入框背景、次要表面 —— 淡 sage 色调
- **Light Sage**（`#e5e7e0`）：按钮背景、三级表面 —— 柔和的 sage 绿色
- **Warm Tan**（`#d4c9b8`）：特色按钮背景 —— 用于强调的温暖 tan/卡其色
- **Hover White**（`#f4f4f4`）：通用悬停背景状态

### 中性色与文本
- **Olive Ink**（`#4d4f46`）：主要正文和 UI 文本
- **Muted Olive**（`#65675e`）：次要文本、浅色背景上的按钮标签
- **Sage Placeholder**（`#9ea096`）：占位符文本、禁用状态 —— 温暖的 sage 绿色
- **Sage Border**（`#bfc1b7`）：主要边框颜色 —— 用于所有边框的橄榄色调灰色
- **Light Border**（`#b6b7af`）：次要边框、工具栏边框 —— 稍深的 sage 色

### 语义与强调
- **PostHog Orange**（`#F54E00`）：悬停文本强调色 —— 表示交互性和品牌个性
- **Amber Gold**（`#F7A501`）：深色按钮悬停强调色 —— 温暖信号
- **Focus Blue**（`#3b82f6` 50% 透明度）：键盘焦点环 —— 仅用于无障碍的颜色
- **Dark Text**（`#111827`）：高对比度链接文本 —— 用于重要链接的近黑色

### 渐变系统
- 营销网站无渐变 —— PostHog 的视觉语言刻意保持扁平温暖
- 深度通过分层表面和边框围合实现，而非颜色过渡

## 3. 排版规则

### 字体族
- **展示与正文**：`IBM Plex Sans Variable` —— 可变字体（100–700+ 字重范围）。备选：`IBM Plex Sans, -apple-system, system-ui, Avenir Next, Avenir, Segoe UI, Helvetica Neue, Helvetica, Ubuntu, Roboto, Noto, Arial`
- **等宽字体**：`ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, Liberation Mono, Courier New` —— 系统等宽字体栈
- **代码展示**：`Source Code Pro` —— 备选：`Menlo, Consolas, Monaco`

### 层级

### 原则
- **粗体标题主导**：标题使用 700–800 字重 —— PostHog 的排版自信而坚定，不细声细语
- **宽松正文行高**：正文文本 1.50–1.71 行高创造极其舒适的阅读体验 —— 网站内容密集，为长时间阅读优化
- **分数尺寸**：多个尺寸（21.4px、19.3px、13.7px）表明是一个流动/缩放的排版系统而非固定档位 —— 可能是 Tailwind 的 rem 尺度在非标准基数下计算得出
- **大写作为分类信号**：粗体大写标签（18px–20px 字重 700）用于产品分类标题 —— 一种杂志编辑惯例
- **选择性负字间距**：字母间距在展示文本上收紧（30px 时 -0.75px），但在正文上放松为 0px —— 标题压缩，正文呼吸

## 4. 组件样式

### 按钮
- **深色主要按钮**：`#1e1f23` 背景、白色文本、6px 圆角、`10px 12px` 内边距。悬停：透明度 0.7 配 Amber Gold 文本。激活：透明度 0.8 配轻微缩放变换。主要 CTA —— 深色且自信
- **Sage 浅色按钮**：`#e5e7e0` 背景、Olive Ink（`#4d4f46`）文本、4px 圆角、`4px` 内边距。悬停：`#f4f4f4` 背景配 PostHog Orange 文本。紧凑实用按钮
- **Warm Tan 特色按钮**：`#d4c9b8` 背景、黑色文本、无可见圆角。悬停：同样的橙色文本闪烁。特色/高级操作
- **输入框样式**：`#eeefe9` 背景、Sage Placeholder（`#9ea096`）文本、4px 圆角、1px `#b6b7af` 边框。外观像搜索/筛选控件
- **近白色幽灵按钮**：`#fdfdf8` 背景、Olive Ink 文本、4px 圆角、透明 1px 边框。最小存在感
- **悬停模式**：所有按钮在悬停时闪烁 PostHog Orange（`#F54E00`）或 Amber Gold（`#F7A501`）文本 —— 品牌标志性的交互惊喜

### 卡片与容器
- **有边框卡片**：Warm Parchment（`#fdfdf8`）或白色背景、1px `#bfc1b7` 边框、4px–6px 圆角 —— 简洁简约
- **Sage 表面卡片**：`#eeefe9` 背景用于次要内容容器
- **阴影卡片**：`0px 25px 50px -12px rgba(0, 0, 0, 0.25)` —— 单个深阴影用于提升内容（模态框、下拉菜单）
- **悬停**：交互卡片上的橙色文本闪烁 —— 与按钮行为一致

### 输入框与表单
- **默认**：`#eeefe9` 背景、`#9ea096` 占位符文本、1px `#b6b7af` 边框、4px 圆角、`2px 0px 2px 8px` 内边距
- **焦点**：`#3b82f6` 环以 50% 透明度（Tailwind 蓝色焦点环）
- **文本颜色**：`#374151` 用于输入值 —— 比主要文本更深以保证可读性
- **边框变体**：多种边框模式 —— 部分输入框使用复合边框（仅上、左、下）

### 导航
- **顶部导航**：温暖背景、IBM Plex Sans 15px 字重 600
- **下拉菜单**：丰富的 mega-menu 结构与产品分类
- **链接颜色**：Deep Olive（`#23251d`）用于导航链接，悬停时下划线
- **CTA**：导航中的深色主要按钮（#1e1f23）—— "Get started - free"
- **移动端**：折叠为汉堡菜单与简化菜单

### 图片处理
- **手绘插图**：刺猬吉祥物和古怪插图 —— 标志性视觉元素
- **产品截图**：嵌入设备框架或简洁容器中的 UI 截图
- **动作人偶**：刺猬人偶的俏皮产品摄影 —— 反传统
- **信任标志**：企业标志（Airbus、GOV.UK）以柔和的信任栏展示
- **宽高比**：混合 —— 插图不规则，截图为 16:9 或宽屏

### AI 聊天组件
- 浮动的 PostHog AI 助手配对话气泡 —— 嵌入营销网站的交互式产品演示

## 5. 布局原则

### 间距系统
- **基础单位**：8px
- **尺度**：2px、4px、6px、8px、10px、12px、16px、18px、24px、32px、34px
- **区块间距**：区块间垂直 32px–48px（对于内容密集的网站来说紧凑）
- **卡片内边距**：内部 4px–12px（明显紧凑）
- **组件间距**：相关元素间 4px–8px

### 网格与容器
- **最大宽度**：1536px（最大断点），内容容器可能为 1200px–1280px
- **列模式**：多样 —— 文本内容单列、功能卡片 2-3 列网格、产品演示非对称布局
- **断点**：定义 13 个 —— 1px、425px、482px、640px、768px、767px、800px、900px、1024px、1076px、1160px、1280px、1536px

### 留白哲学
- **设计上内容密集**：PostHog 的网站信息丰富 —— 留白是适度的，而非 lavish
- **编辑式节奏**：内容区块像杂志一样流动，多样的布局让眼睛持续移动
- **插图作为呼吸空间**：手绘刺猬艺术自然地打破密集的内容区块

### 圆角尺度
- **2px**：小型行内元素、标签（`span`）
- **4px**：主要 UI 组件 —— 按钮、输入框、下拉菜单、菜单项（`button`、`div`、`combobox`）
- **6px**：次要容器 —— 较大按钮、列表项、卡片变体（`button`、`div`、`li`）
- **9999px**：药丸形状 —— 徽章、状态指示器、圆角标签（`span`、`div`）

## 6. 深度与高度

| 层级 | 处理方式 | 用途 |
|------|----------|------|
| 层级 0（扁平） | 无阴影、温暖羊皮纸背景 | 页面画布、大多数表面 |
| 层级 1（边框） | `1px solid #bfc1b7`（Sage Border） | 卡片围合、输入边框、区块分隔线 |
| 层级 2（复合边框） | 不同侧边的多个 1px 边框 | 输入组、工具栏元素 |
| 层级 3（深阴影） | `0px 25px 50px -12px rgba(0, 0, 0, 0.25)` | 模态框、浮动元素、mega-menu 下拉菜单 |

### 阴影哲学
PostHog 的高度系统极其简约 —— 整个系统只存在一个阴影定义。深度通过以下方式传达：
- **边框围合**：sage 色调边框（`#bfc1b7`）1px 创造温和的温暖分隔
- **表面颜色变化**：从 `#fdfdf8` 到 `#eeefe9` 到 `#e5e7e0` 创造无阴影的分层深度
- **唯一的阴影**：定义的阴影（`0 25px 50px -12px`）保留给浮动元素 —— 模态框、下拉菜单、弹出框。这是一个深沉、戏剧性的阴影，在需要时创造清晰的分隔

### 装饰性深度
- **插图分层**：手绘刺猬艺术自然创造视觉深度
- **无渐变或发光**：扁平、温暖的表面系统完全依赖边框和表面颜色差异
- **无玻璃态**：全程完全不透明表面

## 7. 做与不做

### 做
- 使用 olive/sage 色系（#4d4f46、#23251d、#bfc1b7）用于文本和边框 —— 温暖的绿色底色对品牌至关重要
- 在悬停状态闪烁 PostHog Orange（#F54E00）—— 这是隐藏的品牌签名
- 在标题使用 IBM Plex Sans 粗体字重（700/800）—— 字体承载技术可信度
- 保持正文文本宽行高（1.50–1.71）—— 内容密集的网站需要可读性
- 维持温暖羊皮纸背景（#fdfdf8）—— 不是纯白，永不冷色
- 大多数 UI 元素使用 4px 圆角 —— 保持角落微妙且功能性
- 包含俏皮、手绘插图元素 —— 个性是差异化因素
- 在深色按钮上应用基于透明度的悬停状态（0.7 透明度）而非颜色变化

### 不做
- 使用蓝色、紫色或典型的科技 SaaS 颜色 —— PostHog 的调色板刻意是 olive/sage
- 添加重度阴影 —— 系统只对浮动元素使用一个阴影；其他一切使用边框
- 让设计看起来以传统意义上的"精致"或"高端" —— PostHog 的魅力在于其不羁、充满活力的能量
- 在正文文本使用紧行高 —— 宽松的 1.50+ 间距对内容密集的布局至关重要
- 在卡片上应用大圆角（12px+）—— PostHog 使用 4px–6px，保持紧凑功能性
- 移除橙色悬停闪烁 —— 这是核心交互模式，不是装饰
- 用库存照片替换插图 —— 手绘刺猬艺术就是品牌
- 使用纯白（#ffffff）作为页面背景 —— 温暖的 sage-cream（#fdfdf8）色调是基础性的

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 关键变化 |
|------|------|----------|
| 移动端小屏 | <425px | 单列、紧凑内边距、堆叠卡片 |
| 移动端 | 425px–640px | 轻微布局调整、更大触摸目标 |
| 平板 | 640px–768px | 开始 2 列网格、导航部分可见 |
| 平板大屏 | 768px–1024px | 多列布局、展开导航 |
| 桌面端 | 1024px–1280px | 完整布局、3 列功能网格、展开 mega-menu |
| 大桌面端 | 1280px–1536px | 最大宽度容器、宽松边距 |
| 超大桌面端 | >1536px | 居中容器于最大宽度 |

### 触摸目标
- 按钮：4px–6px 圆角配 `4px–12px` 内边距 —— 紧凑但可用
- 导航链接：15px 文本字重 600 配足够内边距
- 移动端：汉堡菜单配简化导航
- 输入框：宽松垂直内边距便于拇指友好表单

### 折叠策略
- **导航**：带下拉菜单的完整 mega-menu → 移动端汉堡菜单
- **功能网格**：3 列 → 2 列 → 单列堆叠
- **排版**：展示尺寸跨断点缩小（30px → 更小）
- **插图**：在容器内响应式缩放，部分可能在移动端隐藏以节省空间
- **区块间距**：比例缩小同时保持可读性

### 图片行为
- 插图在容器内响应式缩放
- 产品截图保持宽高比
- 信任标志在移动端重排为多行网格
- AI 聊天组件可能在小屏幕上重新定位或简化

### 迭代指南
使用此设计系统优化已生成的屏幕时：
1. 验证背景是温暖羊皮纸（#fdfdf8）而非纯白 —— sage-cream 的温暖至关重要
2. 检查所有文本使用 olive 色系（#4d4f46、#23251d）而非纯黑或中性灰
3. 确保悬停状态闪烁 PostHog Orange（#F54E00）—— 如果悬停感觉平淡，你就遗漏了这一点
4. 确认边框使用 sage 色调灰色（#bfc1b7）而非中性灰 —— 温暖贯穿每个元素
5. 整体基调应该感觉像一个有趣、充满活力的初创公司 wiki —— 永远不要企业化精致或死板
