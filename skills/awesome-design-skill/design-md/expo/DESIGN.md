# 设计系统：Expo

## 1. 视觉主题与氛围

Expo 的界面是一个发光、自信辐射的开发者平台，建立在这样的前提上：用于构建应用的工具应该像应用本身一样精致。整个体验存在于明亮、轻盈的画布上——一种带冷色调的米白色（`#f0f0f3`），赋予页面微妙的技术冷感，却无纯白的生硬。这是一个呼吸的网站：区块间巨大的垂直间距创造出画廊般的节奏，每个功能都有自己的"房间"。

设计语言果断地单色——纯黑（`#000000`）标题对着最浅的背景，用一系列冷蓝灰色（`#60646c`、`#b0b4ba`、`#555860`）处理所有次要传达。颜色几乎完全从界面本身消失；当它出现时，保留给产品截图、应用图标和 React 宇宙插图——使实际内容在中性画布上迸发生机。

让 Expo 独特的是其药丸形状几何。按钮、标签、视频容器甚至图片使用慷慨圆角或完整药丸形状角落（24px–9999px），创造有机、亲切的感觉，与典型尖角开发者工具美学形成对比。结合巨大标题上的紧凑字间距（64px 时 -1.6px 到 -3px），结果是既高端又友好的设计——就像为开发者重新想象的 Apple 产品页面。

**核心特征：**
- 发光的冷白色画布（`#f0f0f3`）配画廊般的垂直间距
- 严格单色：纯黑标题，冷蓝灰正文，无装饰色
- 到处都是药丸形状几何——按钮、标签、容器、图片（24px–9999px 圆角）
- 巨大展示标题（64px）配极端负字间距（-1.6px 到 -3px）
- Inter 作为唯一字体，使用 400–900 字重实现完整表现范围
- 轻如低语的阴影，几乎不将元素从表面提起
- 产品截图作为界面中唯一的颜色来源

## 2. 色彩调色板与角色

### 主色
- **Expo Black**（`#000000`）：绝对锚点——用于主要标题、CTA 按钮和品牌身份。冷白上的纯黑创造最大对比度而不显激进。
- **Near Black**（`#1c2024`）：正文内容的主要文字颜色——一种几乎无法察觉的蓝黑色，比纯 #000 柔和，适合长篇阅读。

### 次要与强调色
- **Link Cobalt**（`#0d74ce`）：标准链接颜色——一种可信赖、饱和的蓝色，表示交互性而不与单色层级竞争。
- **Legal Blue**（`#476cff`）：更亮、更饱和的蓝色用于法律/页脚链接——比 Link Cobalt 更引人注目。
- **Widget Sky**（`#47c2ff`）：一种轻盈、友好的青蓝色用于小部件品牌元素——系统中最亮的强调色。
- **Preview Purple**（`#8145b5`）：一种丰富的紫罗兰色用于"预览"或测试版功能指示器——与标准内容形成清晰视觉区分。

### 表面与背景
- **Cloud Gray**（`#f0f0f3`）：主要页面背景——一种带有最淡淡蓝紫色调的冷米白色。不暖，不育——精确地技术化。
- **Pure White**（`#ffffff`）：卡片表面、按钮背景和提升的内容容器。与 Cloud Gray 形成清晰的"提升"区分。
- **Widget Dark**（`#1a1a1a`）：深色主题小部件和覆盖元素的深色表面。
- **Banner Dark**（`#171717`）：最深的表面变体，用于促销横幅和高对比度容器。

### 中性色与文字
- **Slate Gray**（`#60646c`）：主力次要文字颜色。一种权威而不沉重的冷蓝灰色。
- **Mid Slate**（`#555860`）：比 Slate 略深，用于强调的次要文字。
- **Silver**（`#b0b4ba`）：第三级文字、占位符和去强调的元数据。舒适可读但明显后退。
- **Pewter**（`#999999`）：深色背景下的手风琴图标和深度去强调的 UI 元素。
- **Light Silver**（`#cccccc`）：深色背景下的箭头图标和装饰元素。
- **Dark Slate**（`#363a3f`）：深色表面上的边框、开关轨道和强调的容器。
- **Charcoal**（`#333333`）：深色模式开关背景和深层次要表面。

### 语义与强调色
- **Warning Amber**（`#ab6400`）：一种温暖、深沉的琥珀色用于警告状态——故意不是亮黄色，传达严肃性。
- **Destructive Rose**（`#eb8e90`）：一种柔和的粉珊瑚色用于禁用的破坏性操作——比典型红色柔和，减少警报疲劳。
- **Border Lavender**（`#e0e1e6`）：标准卡片/容器边框——一种可见而不沉重的冷薰衣草灰色。
- **Input Border**（`#d9d9e0`）：按钮和表单元素边框——比卡片边框略暖/深，用于交互元素。
- **Dark Focus Ring**（`#2547d0`）：深色主题背景下键盘聚焦指示器的深蓝色。

### 渐变系统
- 设计在界面层明显**无渐变**。视觉丰富度来自产品截图、React 宇宙插图和精心的阴影分层，而非颜色渐变。这种缺席就是设计决定——渐变会破坏临床精确性。

## 3. 排版规则

### 字体家族
- **主要**：`Inter`，回退：`-apple-system, system-ui`
- **等宽**：`JetBrains Mono`，回退：`ui-monospace`
- **系统回退**：`system-ui, Segoe UI, Roboto, Helvetica, Arial, Apple Color Emoji, Segoe UI Emoji`

### 层级

| 角色 | 字体 | 大小 | 字重 | 行高 | 字间距 | 备注 |
|------|------|------|--------|-------------|----------------|-------|
| 展示 / 英雄 | Inter | 64px (4rem) | 700–900 | 1.10 (紧凑) | -1.6px 到 -3px | 最大冲击，极端字距 |
| 章节标题 | Inter | 48px (3rem) | 600 | 1.10 (紧凑) | -2px | 功能区块锚点 |
| 副标题 | Inter | 20px (1.25rem) | 600 | 1.20 (紧凑) | -0.25px | 卡片标题，功能名 |
| 大正文 | Inter | 18px (1.13rem) | 400–500 | 1.40 | normal | 介绍段落，区块描述 |
| 正文 / 按钮 | Inter | 16px (1rem) | 400–700 | 1.25–1.40 | normal | 标准文字，导航链接，按钮 |
| 说明 / 标签 | Inter | 14px (0.88rem) | 400–600 | 1.00–1.40 | normal | 描述，元数据，徽章文字 |
| 标签 / 小字 | Inter | 12px (0.75rem) | 500 | 1.00–1.60 | normal | 最小无衬线文字，徽章 |
| 代码正文 | JetBrains Mono | 16px (1rem) | 400–600 | 1.40 | normal | 行内代码，终端命令 |
| 代码说明 | JetBrains Mono | 14px (0.88rem) | 400–600 | 1.40 | normal | 代码片段，技术标签 |
| 代码小字 | JetBrains Mono | 12px (0.75rem) | 400 | 1.60 | normal | 大写技术标签 |

### 原则
- **一种字体，完整表现**：Inter 是唯一的无衬线字体，从字重 400（常规）到 900（黑体）。这给设计一个统一的声音，同时在轻如低语的正文和雷鸣般的展示标题之间实现戏剧性对比。
- **极端负字距在较大尺寸**：64px 标题使用 -1.6px 到 -3px 字间距，创造超密集文字块，感觉像标志类型。这种激进压缩是标志性排版动作。
- **字重即层级**：700–900 用于展示，600 用于标题，500 用于强调，400 用于正文。跳跃是果断的——没有模棱两可的中间字重。
- **一致的 1.40 正文行高**：几乎所有正文和 UI 文字共享 1.40 行高，创造有节奏的垂直一致性。

## 4. 组件样式

### 按钮

**主要（边框白）**
- 背景：Pure White（`#ffffff`）
- 文字：Near Black（`#1c2024`）
- 内边距：0px 12px（紧凑，内容驱动高度）
- 边框：细实线 Input Border（`1px solid #d9d9e0`）
- 圆角：微圆（6px）
- 阴影：悬停时微妙组合阴影
- 低调默认——干净、专业、不英雄

**主要药丸**
- 同主要按钮但用药丸形状圆角（9999px）
- 用于英雄 CTA 和高强调操作
- 额外圆润表示"从这里开始"

**深色主要**
- 背景：Expo Black（`#000000`）
- 文字：Pure White（`#ffffff`）
- 药丸形状（9999px）或慷慨圆角（32–36px）
- 无边框（黑色就是边框）
- 最大强调 CTA——保留给主要转化操作

### 卡片与容器
- 背景：Pure White（`#ffffff`）——明显从 Cloud Gray 页面提升
- 边框：细实线 Border Lavender（`1px solid #e0e1e6`）用于标准卡片
- 圆角：舒适圆角（8px）用于标准卡片；慷慨圆角（16–24px）用于特色容器
- 阴影级别 1：低语（`rgba(0,0,0,0.08) 0px 3px 6px, rgba(0,0,0,0.07) 0px 2px 4px`）——几乎无法察觉的提升
- 阴影级别 2：标准（`rgba(0,0,0,0.1) 0px 10px 20px, rgba(0,0,0,0.05) 0px 3px 6px`）——清晰浮动高度
- 悬停：可能微妙的阴影加深或背景变化

### 输入与表单
- 背景：Pure White（`#ffffff`）
- 文字：Near Black（`#1c2024`）
- 边框：细实线 Input Border（`1px solid #d9d9e0`）
- 内边距：0px 12px（与按钮尺寸一致）
- 圆角：微圆（6px）
- 聚焦：通过 CSS 自定义属性的蓝色环阴影

### 导航
- 透明/模糊背景上的粘性顶部导航
- 标志：黑色 Expo 字标
- 链接：Near Black（`#1c2024`）或 Slate Gray（`#60646c`），14–16px Inter 字重 500
- CTA：右侧黑色药丸按钮（"Sign Up"）
- GitHub star 徽章作为社会证明
- 状态指示器（"All Systems Operational"）配绿点

### 图片处理
- 产品截图和设备模型是视觉英雄
- 视频和图片容器慷慨圆角（24px）
- 截图显示在真实设备框架中
- 深色 UI 截图在轻盈画布上提供对比
- 圆角容器内全出血

### 独特组件

**React 宇宙标志**
- 动画/插画的 React 标志作为视觉中心
- 将 Expo 身份连接到 React 生态系统
- 其他摄影页面上唯一的插画元素

**设备预览网格**
- 同时显示多种设备类型（手机、平板、网页）
- 视觉演示跨平台能力
- 每个设备使用真实设备外观

**状态徽章**
- 导航中的"All Systems Operational"药丸
- 绿点 + 文字——紧凑信任信号
- 药丸形状（36px 圆角）

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 刻度：1px、2px、4px、8px、12px、16px、24px、32px、40px、48px、64px、80px、96px、144px
- 按钮内边距：0px 12px（异常紧凑——高度由行高驱动）
- 卡片内部内边距：约 24–32px
- 区块垂直间距：巨大（估计主要区块间 96–144px）
- 组件间距：兄弟元素间 16–24px

### 网格与容器
- 最大容器宽度：约 1200–1400px，居中
- 英雄区：居中单列，巨大呼吸空间
- 功能区块：交替布局（图片左/右，全宽展示）
- 卡片网格：2–3 列用于功能亮点
- 全宽区块配包含的内部内容

### 留白哲学
- **画廊般的节奏**：每个区块感觉像自己的展览，被广阔空白包围。这创造高端、从容的浏览体验。
- **呼吸空间就是设计**：慷慨留白就是主要设计元素——它传达信心、质量，以及每个功能值得单独关注。
- **内容岛屿**：区块作为孤立的"岛屿"漂浮在白色空间中，通过滚动而非视觉延续连接。

### 边框圆角刻度
- 近方形（4px）：小行内元素，标签
- 微圆（6px）：按钮，表单输入，组合框——功能性交互圆角
- 舒适圆角（8px）：标准内容卡片，容器
- 慷慨圆角（16px）：功能标签，内容面板
- 很圆（24px）：按钮，视频/图片容器，标签面板——标志性柔和
- 高度圆角（32–36px）：英雄 CTA，状态徽章，导航按钮
- 药丸形状（9999px）：主要操作按钮，标签，头像——最大友好度

## 6. 深度与高度

| 级别 | 处理 | 用途 |
|-------|-----------|-----|
| 扁平（Level 0） | 无阴影 | Cloud Gray 页面背景，行内文字 |
| 表面（Level 1） | 白色背景，无阴影 | Cloud Gray 上的标准白色卡片 |
| 低语（Level 2） | `rgba(0,0,0,0.08) 0px 3px 6px` + `rgba(0,0,0,0.07) 0px 2px 4px` | 微妙卡片提升，悬停状态 |
| 悬浮（Level 3） | `rgba(0,0,0,0.1) 0px 10px 20px` + `rgba(0,0,0,0.05) 0px 3px 6px` | 功能展示，产品截图 |
| 模态（Level 4） | 深色覆盖（`--dialog-overlay-background-color`）+ 重阴影 | 对话框，覆盖层 |

**阴影哲学**：Expo 使用阴影作为温柔低语而非建筑声明。主要深度机制是**背景颜色对比**——白色卡片漂浮在 Cloud Gray 上——而非阴影投射。当阴影出现时，它们柔软、漫射且定向（向下），创造纸张在桌面上方毫米级悬浮的感觉。

## 7. 做与不做

### 做
- 使用 Cloud Gray（`#f0f0f3`）作为页面背景，Pure White（`#ffffff`）用于提升卡片——双色浅色系统是本质
- 展示标题保持极端负字间距（64px 时 -1.6px 到 -3px）用于标志性压缩外观
- 主要 CTA 按钮使用药丸形状（9999px）圆角——有机形状是身份核心
- 黑色（`#000000`）保留给标题和主要 CTA——它在轻盈画布上承载最大权威
- 次要文字使用 Slate Gray（`#60646c`）——它是可读与后退之间的精确平衡
- 区块间保持巨大垂直间距（96px+）——画廊节奏定义高端感
- 使用产品截图作为主要视觉内容——界面保持单色，产品带来颜色
- Inter 应用完整字重范围（400–900）——字重对比就是层级

### 不做
- 不要在界面装饰中引入颜色——单色调色板是有意的
- 交互元素不要使用尖角（border-radius < 6px）——药丸/圆润几何是签名
- 区块间距不要低于 64px——呼吸空间就是设计
- 不要使用重投影阴影——深度来自背景对比和低语阴影
- 不要混入额外字体——Inter 处理从展示到说明的一切
- 正文文字字间距不要宽于 -0.25px——极端字距保留给展示
- 边框不要重于 2px——容器通过背景颜色和温和边框实现微妙
- 界面不要添加渐变——视觉丰富度来自内容，而非装饰
- 语义语境外不要使用饱和颜色——调色板严格灰度 + 功能蓝

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 关键变化 |
|------|-------|-------------|
| 移动端 | <640px | 单列，汉堡导航，堆叠卡片，英雄文字缩放至 ~36px |
| 平板 | 640–1024px | 2 列网格，压缩导航，中等英雄文字 |
| 桌面 | >1024px | 完整多列布局，扩展导航，巨大英雄（64px） |

*只检测到一个显式断点（640px），表明使用流体、容器查询或 min()/clamp() 基础的响应式系统，而非固定断点切换。*

### 触摸目标
- 按钮使用慷慨圆角（24–36px），创造大、手指友好的表面
- 导航链接间距充足
- 状态徽章为触摸设计（36px 圆角）
- 建议最小：44x44px

### 折叠策略
- **导航**：配 CTA 的完整水平导航在移动端折叠为汉堡
- **功能区块**：多列 → 堆叠单列
- **英雄文字**：64px → ~36px 渐进缩放
- **设备预览**：网格 → 堆叠/轮播
- **卡片**：并排 → 垂直堆叠
- **间距**：按比例减少但保持慷慨节奏

### 图片行为
- 产品截图按比例缩放
- 移动端设备模型可能简化或显示更少设备
- 所有尺寸保持圆角
- 下方内容懒加载

### 迭代指南
1. 一次专注于一个组件
2. 引用具体颜色名称和十六进制代码——"使用 Slate Gray (#60646c)"而非"使其灰色"
3. 刻意使用圆角值——按钮 6px，卡片 8px，图片 24px，药丸 9999px
