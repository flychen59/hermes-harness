# 设计系统：Raycast

## 1. 视觉主题与氛围

Raycast 的营销网站就像精密仪器的深色内部——一个用黑曜石雕刻而成的瑞士手表表壳。背景不只是深色，而是带有蓝色调的近黑色（`#07080a`），营造出一种身处 macOS 原生应用而非网站的感觉。每一个表面、每一个边框、每一个阴影都经过精心校准，以唤起高性能桌面工具的感觉：快速、简约、可信赖。

标志性手法是借鉴 macOS 窗口边框的分层阴影系统：多层 box-shadow 配合内嵌高光，模拟物理深度，仿佛卡片和按钮是深色桌面上实际按压或凸起的玻璃元素。结合 Raycast Red（`#FF6363`）——几乎只用于 hero 区域标志性的斜条纹图案——调色板打造出一个"有个性强大工具"的品牌形象。红色不占主导地位；它只是点缀。

Inter 被用于所有地方——标题、正文、按钮、说明文字——配合广泛的 OpenType 特性（`calt`、`kern`、`liga`、`ss03`）创造一致、可读的排版风格。正字间距（正文 0.2px–0.4px）对于深色 UI 来说是不寻常的，赋予文字一种轻盈、通透的质感，平衡密集的深色表面。GeistMono 出现代码元素，强化开发者工具的身份认同。

**关键特征：**
- 带蓝色调的近黑色背景（`#07080a`）——不是纯黑，微妙地偏向蓝色
- macOS 原生阴影系统，配合多层内嵌高光模拟物理深度
- Raycast Red（`#FF6363`）作为点缀色——hero 条纹，而非遍布各处
- Inter 配合正字间距（0.2px），营造轻盈、可读的深色模式体验
- Radix UI 组件原语驱动交互层
- 微妙的 rgba 白色边框（0.06–0.1 不透明度）用于深色表面上的围合
- 键盘快捷键样式，配合渐变键帽和重度阴影

## 2. 调色板与角色

### 主色
- **Near-Black Blue**（`#07080a`）：主页面背景——带有微妙蓝色冷调底色的基础虚空
- **Pure White**（`#ffffff`）：主标题文本、高强调元素
- **Raycast Red**（`#FF6363` / `hsl(0, 100%, 69%)`）：品牌强调色——hero 条纹、危险状态、关键高亮

### 辅助色与强调色
- **Raycast Blue**（`hsl(202, 100%, 67%)` / ~`#55b3ff`）：交互强调色——链接、焦点状态、选中项
- **Raycast Green**（`hsl(151, 59%, 59%)` / ~`#5fc992`）：成功状态、正向指示器
- **Raycast Yellow**（`hsl(43, 100%, 60%)` / ~`#ffbc33`）：警告强调、高亮
- **Blue Transparent**（`hsla(202, 100%, 67%, 0.15)`）：蓝色调覆盖层，用于交互表面
- **Red Transparent**（`hsla(0, 100%, 69%, 0.15)`）：红色调覆盖层，用于危险/错误表面

### 表面与背景
- **Deep Background**（`#07080a`）：页面画布，最深的表面
- **Surface 100**（`#101111`）：凸起表面、卡片背景
- **Key Start**（`#121212`）：键盘按键渐变起点
- **Key End**（`#0d0d0d`）：键盘按键渐变终点
- **Card Surface**（`#1b1c1e`）：徽章背景、标签填充、凸起容器
- **Button Foreground**（`#18191a`）：浅色背景上按钮文本的深色表面

### 中性色与文本
- **Near White**（`#f9f9f9` / `hsl(240, 11%, 96%)`）：主正文文本、高强调内容
- **Light Gray**（`#cecece` / `#cdcdce`）：次要正文文本、描述
- **Silver**（`#c0c0c0`）：第三级文本、柔和标签
- **Medium Gray**（`#9c9c9d`）：链接默认颜色、次要导航
- **Dim Gray**（`#6a6b6c`）：禁用文本、低强调标签
- **Dark Gray**（`#434345`）：淡化边框、非活动导航链接
- **Border**（`hsl(195, 5%, 15%)` / ~`#252829`）：卡片和分隔线的标准边框颜色
- **Dark Border**（`#2f3031`）：分隔线、表格边框

### 语义色与强调色
- **Error Red**（`hsl(0, 100%, 69%)`）：错误状态、破坏性操作
- **Success Green**（`hsl(151, 59%, 59%)`）：成功确认、正向状态
- **Warning Yellow**（`hsl(43, 100%, 60%)`）：警告、需要注意的状态
- **Info Blue**（`hsl(202, 100%, 67%)`）：信息高亮、链接

### 渐变系统
- **Keyboard Key Gradient**：从 `#121212`（顶部）到 `#0d0d0d`（底部）的线性渐变——模拟物理按键深度
- **Warm Glow**：`rgba(215, 201, 175, 0.05)` 径向扩散——特色元素背后微妙的暖色环境光晕

## 3. 排版规则

### 字体族
- **主字体**：`Inter`——人文无衬线体，用于所有地方。后备字体：`Inter Fallback`、system sans-serif
- **系统字体**：`SF Pro Text`——Apple 系统字体，用于特定的 macOS 原生 UI 元素。后备字体：`SF Pro Icons`、`Inter`、`Inter Fallback`
- **等宽字体**：`GeistMono`——Vercel 的等宽字体，用于代码元素。后备字体：`ui-monospace`、`SFMono-Regular`、`Roboto Mono`、`Menlo`、`Monaco`
- **OpenType 特性**：全局启用 `calt`、`kern`、`liga`、`ss03`；展示文本使用 `ss02`、`ss08`；hero 标题禁用 `liga`（`"liga" 0`）

### 层级

| Section Heading | 24px | 500 | normal | 0.2px | OpenType: calt, kern, liga, ss03 |

| Body Tight | 16px | 400 | 1.15 | 0.1px | UI 标签、紧凑上下文 |

### 原则
- **深色背景上的正字间距**：与大多数使用紧凑或中性字间距的深色 UI 不同，Raycast 应用了 +0.2px 到 +0.4px——创造轻盈、可读的感觉，补偿深色背景
- **字重 500 作为基准**：大多数正文使用 medium 字重（500），而非 regular（400）——微妙的额外厚度提高深色表面上的可读性
- **展示克制度**：64px/600 的 hero 文本自信但不夸张——Raycast 避免排版奇观，追求功能优雅
- **全局 OpenType**：`ss03`（stylistic set 3）在 Inter 上全局启用，赋予字体更几何化、工具化的特质

## 4. 组件样式

### 按钮
- **Primary Pill**：透明背景、白色文本、胶囊形状（86px 圆角）、多层内嵌阴影（`rgba(255, 255, 255, 0.1) 0px 1px 0px 0px inset`）。悬停：不透明度 0.6
- **Secondary Button**：透明背景、白色文本、6px 圆角、`1px solid rgba(255, 255, 255, 0.1)` 边框、微妙的投影（`rgba(0, 0, 0, 0.03) 0px 7px 3px`）。悬停：不透明度 0.6
- **Ghost Button**：无背景或边框、灰色文本（`#6a6b6c`）、86px 圆角、相同的内嵌阴影。悬停：不透明度 0.6，文本变亮为白色
- **CTA (Download)**：半透明白色背景（`hsla(0, 0%, 100%, 0.815)`）、深色文本（`#18191a`）、胶囊形状。悬停：全白色背景（`hsl(0, 0%, 100%)`）
- **过渡**：所有按钮使用不透明度过渡进行悬停效果，而非背景色变化——这是 Raycast 标志性的交互模式

### 卡片与容器
- **Standard Card**：`#101111` 表面、`1px solid rgba(255, 255, 255, 0.06)` 边框、12px–16px 圆角
- **Elevated Card**：环形阴影 `rgb(27, 28, 30) 0px 0px 0px 1px` 外部 + `rgb(7, 8, 10) 0px 0px 0px 1px inset` 内部——创造双层围合
- **Feature Card**：16px–20px 圆角、hero 元素背后微妙的暖色光晕（`rgba(215, 201, 175, 0.05) 0px 0px 20px 5px`）
- **悬停**：卡片通过边框不透明度增加或微妙阴影增强略微变亮

### 输入框与表单
- 深色输入框，`#07080a` 背景、`1px solid rgba(255, 255, 255, 0.08)` 边框、8px 圆角
- 焦点状态：边框变亮，蓝色光晕（`hsla(202, 100%, 67%, 0.15)`）环出现
- 文本：`#f9f9f9` 输入颜色、`#6a6b6c` 占位符
- 标签：`#9c9c9d`，14px，字重 500

### 导航
- **顶部导航**：深色背景与页面融合，白色文本链接，16px 字重 500
- **导航链接**：灰色文本（`#9c9c9d`）→ 悬停时变白，悬停时显示下划线装饰
- **CTA 按钮**：导航末端半透明白色胶囊按钮
- **移动端**：折叠为汉堡菜单，保持深色主题
- **粘性**：导航固定在顶部，带有微妙的边框分隔线

### 图片处理
- **产品截图**：macOS 窗口边框风格——圆角（12px）、模拟悬浮窗口的深色阴影
- **全出血区块**：深色截图与深色背景无缝融合
- **Hero 插图**：Raycast Red 中的斜条纹图案——抽象、几何、定义品牌
- **App UI 嵌入**：展示实际的 Raycast 命令面板和扩展——产品即内容

### 键盘快捷键按键
- **键帽样式**：渐变背景（`#121212` → `#0d0d0d`）、重度多层阴影（`rgba(0, 0, 0, 0.4) 0px 1.5px 0.5px 2.5px` + 内嵌阴影），创造逼真的物理按键外观
- 圆角：单个按键 4px–6px

### 徽章与标签
- **Neutral badge**：`#1b1c1e` 背景、白色文本、6px 圆角、14px 字体字重 500、`0px 6px` 内边距
- 紧凑、胶囊样式的分类处理

## 5. 布局原则

### 间距系统
- **基础单位**：8px
- **比例**：1px、2px、3px、4px、8px、10px、12px、16px、20px、24px、32px、40px
- **区块内边距**：主要区块之间垂直 80px–120px
- **卡片内边距**：内部间距 16px–32px
- **组件间距**：相关元素之间 8px–16px

### 网格与容器
- **最大宽度**：约 1200px 容器（断点在 1204px），居中
- **列模式**：单列 hero、2–3 列特性网格、全宽展示区块
- **应用展示**：产品 UI 在居中窗口框架中呈现

### 留白哲学
- **戏剧性的负空间**：区块漂浮在广阔的深色虚空中，在特性之间创造电影般的节奏
- **产品密集、营销稀疏**：产品 UI 截图信息密集，但周围的营销文案使用最少的文本和充足的间距
- **垂直节奏**：区块内元素之间一致的 24px–32px 间距

### 圆角比例
- **2px–3px**：微型元素、代码片段、小型指示器
- **4px–5px**：键盘按键、小型交互元素
- **6px**：按钮、徽章、标签——主力圆角
- **8px**：输入框、内联组件
- **9px–11px**：图片、中型容器
- **12px**：标准卡片、产品截图
- **16px**：大型卡片、特性区块
- **20px**：Hero 卡片、突出容器
- **86px+**：胶囊按钮、导航 CTA——完整胶囊形状

## 6. 深度与层级

| 级别 | 处理方式 | 用途 |
|------|----------|------|
| Level 0 (Void) | 无阴影，`#07080a` 表面 | 页面背景 |
| Level 1 (Subtle) | `rgba(0, 0, 0, 0.28) 0px 1.189px 2.377px` | 最小凸起，内联元素 |
| Level 2 (Ring) | `rgb(27, 28, 30) 0px 0px 0px 1px` 外部 + `rgb(7, 8, 10) 0px 0px 0px 1px inset` 内部 | 卡片围合，双环技术 |
| Level 3 (Button) | `rgba(255, 255, 255, 0.05) 0px 1px 0px 0px inset` + `rgba(255, 255, 255, 0.25) 0px 0px 0px 1px` + `rgba(0, 0, 0, 0.2) 0px -1px 0px 0px inset` | macOS 原生按钮按压——顶部白色高光，底部深色内嵌 |
| Level 4 (Key) | 5 层阴影堆叠，配合内嵌按压效果 | 键盘快捷键键帽——物理 3D 外观 |
| Level 5 (Floating) | `rgba(0, 0, 0, 0.5) 0px 0px 0px 2px` + `rgba(255, 255, 255, 0.19) 0px 0px 14px` + 内嵌 | 命令面板、悬浮面板——重度深度配合光晕 |

### 阴影哲学
Raycast 的阴影系统是 Web 上最 macOS 原生的。多层阴影结合：
- **外环**用于围合（替代传统边框）
- **内嵌顶部高光**（`rgba(255, 255, 255, 0.05–0.25)`）模拟来自上方的光源
- **内嵌底部暗影**（`rgba(0, 0, 0, 0.2)`）模拟下方的阴影
- 效果是物理性的：元素感觉像玻璃或拉丝金属，而非扁平矩形

### 装饰性深度
- **Warm glow**：`rgba(215, 201, 175, 0.05) 0px 0px 20px 5px` 位于特色元素背后——冷色深色画布上微妙的暖色光晕
- **Blue info glow**：`rgba(0, 153, 255, 0.15)` 用于交互状态强调
- **Red danger glow**：`rgba(255, 99, 99, 0.15)` 用于错误/破坏性状态强调

## 7. 最佳实践

### 应该做
- 使用 `#07080a`（而非纯黑）作为背景——蓝色冷调对于 Raycast 的感觉至关重要
- 在正文文本上应用正字间距（+0.2px）——这是与大多数深色 UI 刻意不同的设计
- 对交互元素使用配合内嵌高光的多层阴影——macOS 原生深度是标志性特征
- 将 Raycast Red（`#FF6363`）保持为点缀，而非遍布——保留给 hero 时刻和错误状态
- 使用 `rgba(255, 255, 255, 0.06)` 边框用于卡片围合——几乎不可见，但在结构上至关重要
- 将字重 500 作为正文文本基准——medium 字重提高深色模式可读性
- 对主要 CTA 使用胶囊形状（86px+ 圆角），对次要操作使用矩形形状（6px–8px）
- 在所有 Inter 文本上启用 OpenType 特性 `calt`、`kern`、`liga`、`ss03`
- 使用不透明度过渡（悬停：不透明度 0.6）进行按钮交互，而非颜色变化

### 不应该做
- 使用纯黑（`#000000`）作为背景——蓝色调将 Raycast 与通用深色主题区分开来
- 在正文文本上应用负字间距——Raycast 刻意使用正间距以提高可读性
- 将 Raycast Blue 作为所有内容的主要强调色——蓝色用于交互/信息，红色是品牌色
- 创建单层扁平阴影——多层内嵌系统是 macOS 原生美学的核心
- 在有 500 可用时使用 regular 字重（400）作为正文文本——额外的字重防止深色模式文本显得单薄
- 混合暖色和冷色边框——坚持使用冷灰色（`hsl(195, 5%, 15%)`）边板调色板
- 应用没有内嵌配合的重度投影——阴影总是成对出现（外部 + 内嵌）
- 使用装饰元素、渐变或彩色背景——深色虚空是舞台，内容是表演者

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 关键变化 |
|------|------|----------|
| Mobile | <600px | 单列、堆叠卡片、汉堡导航、hero 文本缩小至约 40px |
| Small Tablet | 600px–768px | 2 列网格开始、导航部分可见 |
| Tablet | 768px–1024px | 2–3 列特性、导航扩展、截图缩放 |
| Desktop | 1024px–1200px | 完整布局、所有导航链接可见、64px hero 展示 |
| Large Desktop | >1200px | 最大宽度容器居中、充足的侧边距 |

### 触控目标
- 胶囊按钮：86px 圆角配合 20px 内边距——远超 44px 最小值
- 次要按钮：最小 8px 内边距，但边框提供视觉目标扩展
- 导航链接：16px 文本配合周围内边距以实现无障碍触控目标

### 折叠策略
- **导航**：完整水平导航 → 移动端汉堡菜单配合滑出菜单
- **Hero**：64px 展示 → 48px → 36px 跨断点变化
- **特性网格**：3 列 → 2 列 → 单列堆叠
- **产品截图**：在容器内缩放，保持 macOS 窗口边框比例
- **键盘快捷键显示**：在移动端简化或隐藏，因为键盘快捷键在此无关

### 图片行为
- 产品截图在固定比例容器内响应式缩放
- Hero 斜条纹图案按比例缩放
- macOS 窗口边框圆角在所有尺寸下保持
- 无懒加载伪影——图片对产品叙事至关重要

### 迭代指南
使用此设计系统优化现有屏幕时：
1. 检查背景是 `#07080a` 而非纯黑——蓝色调至关重要
2. 验证正文文本上的字间距是正值（+0.2px）——负间距会破坏 Raycast 美学
3. 确保阴影同时具有外部和内嵌层——单层阴影看起来扁平且错误
4. 确认 Inter 启用了 OpenType 特性 `calt`、`kern`、`liga`、`ss03`
5. 测试悬停状态使用不透明度过渡（0.6）而非颜色交换——这是核心交互模式
