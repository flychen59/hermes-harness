# 设计系统：Sanity

## 1. 视觉主题与氛围

Sanity 的网站是一个开发者内容平台，呈现为夜间指挥中心的风格——深沉、精准且结构分明。整个体验建立在近乎黑色的画布（`#0b0b0b`）之上，这不太像是"深色模式切换"，更像是为那些常驻终端的用户所构建工具的自然状态。大多数 CMS 营销页面倾向于使用柔和的粉彩色和温馨的插画，而 Sanity 则倾向于展现其产品本身的厚重感：结构化内容值得拥有一个结构化的舞台。

标志性的字体风格是 waldenburgNormal——一种独特、略带几何感的无衬线字体，采用紧凑的负字间距（展示尺寸下为 -0.32px 至 -4.48px），赋予标题一种压缩的、工程化的质感。在 112px 的 hero 尺寸配合 -4.48px 字距时，文字几乎呈现出机械加工的效果——就像精密切割的钢制字母。这与用于代码和技术标签的 IBM Plex Mono 形成了一种双轨声调：编辑权威性与开发者可信度的结合。

使 Sanity 独具特色的是其单色深色调色板与鲜明、饱和的强调色点缀之间的相互作用。中性色阶从纯黑延伸至一组精心控制的灰度梯度（`#0b0b0b` -> `#212121` -> `#353535` -> `#797979` -> `#b9b9b9` -> `#ededed` -> `#ffffff`），没有冷暖偏向——只有纯粹的、无彩色的精准度。在这片克制的背景下，霓虹绿强调色（display-p3 green）和电光蓝（`#0052ef`）如同暗室控制台中的信号灯般引人注目。橙红色的 CTA（`#f36458`）为这个以冷色调为主的系统提供了唯一的暖色调点缀。

**核心特征：**
- 近黑色画布（`#0b0b0b`）作为默认的自然环境——不是一种深色"模式"，而是核心身份
- waldenburgNormal 在展示尺寸下采用极端负字间距，创造出精密工程感的排版风格
- 纯无彩色灰度色阶——没有冷暖底色，纯粹的灰度纪律
- 鲜明的强调色点缀：霓虹绿、电光蓝（`#0052ef`）和珊瑚红（`#f36458`）点缀于深色背景之上
- 药丸形主按钮（99999px 圆角）与微妙的圆角矩形（3-6px）次要操作形成对比
- IBM Plex Mono 作为编辑展示字体的技术性对照
- 全出血深色区块，内容容纳于精确测量的最大宽度容器中
- 悬停状态在所有交互元素上统一转变为电光蓝（`#0052ef`）——一种一致的"激活"信号

## 2. 色彩调色板与角色

### 主品牌色
- **Sanity Black**（`#0b0b0b`）：主要画布和主导表面颜色。不是纯黑但足够接近，给人一种绝对感。整个视觉识别的基础。
- **纯黑**（`#000000`）：用于最大对比度的时刻、深层叠加层和特定边框强调。
- **Sanity Red**（`#f36458`）：主要的 CTA 和品牌强调色——一种温暖的珊瑚红，作为主要行动号召颜色。用于"开始使用"按钮和主要转化点。

### 强调色与交互色
- **电光蓝**（`#0052ef`）：整个系统中通用的悬停/激活状态颜色。按钮、链接和交互元素在悬停时都会转变为这种蓝色。也用作 `--color-blue-700` 用于焦点环和激活状态。
- **浅蓝**（`#55beff` / `#afe3ff`）：次要蓝色变体，用于强调背景、徽章和暗淡的蓝色表面。
- **霓虹绿**（`color(display-p3 .270588 1 0)`）：一种鲜艳的广色域绿色，用作 `--color-fg-accent-green` 表示成功状态和高级功能高亮。在 sRGB 中回退为 `#19d600`。
- **强调洋红**（`color(display-p3 .960784 0 1)`）：一种鲜艳的广色域洋红，用于特殊强调时刻。

### 表面与背景
- **近黑**（`#0b0b0b`）：默认页面背景和主要表面。
- **深灰**（`#212121`）：提升的表面颜色，用于卡片、次要容器、输入框背景和基础画布上方的微妙层次。
- **中深色**（`#353535`）：第三层表面和边框颜色，用于在深色层之间创造深度。
- **纯白**（`#ffffff`）：用于反转区块、深底浅字文本和特定按钮表面。
- **浅灰**（`#ededed`）：用于反转/浅色区块的浅色表面和微妙的背景色调。

### 中性色与文字
- **白色**（`#ffffff`）：深色表面上的主要文字颜色，最大可读性。
- **银色**（`#b9b9b9`）：次要文字、深色表面上的正文内容、柔和的描述和占位符文本。
- **中灰**（`#797979`）：第三层文字、元数据、时间戳和弱化内容。
- **炭灰**（`#212121`）：浅色/反转表面上的文字。
- **近黑文字**（`#0b0b0b`）：白色/浅色按钮表面上的主要文字。

### 语义色
- **错误红**（`#dd0000`）：破坏性操作、验证错误和严重警告——一种纯净、高饱和度的红色。
- **GPC 绿**（`#37cd84`）：隐私/合规指示器绿色。
- **焦点环蓝**（`#0052ef`）：无障碍焦点环颜色，与交互蓝相匹配。

### 边框系统
- **深色边框**（`#0b0b0b`）：深色容器上的主要边框——几乎不可见，保持最小化围合。
- **微妙边框**（`#212121`）：深色表面上输入框、文本区和卡片边缘的标准边框。
- **中等边框**（`#353535`）：更明显的边框，用于强调性围合和分隔线。
- **浅色边框**（`#ffffff`）：反转/浅色元素或需要对比分离的按钮边框。
- **橙色边框**（`color(display-p3 1 0.3333 0)`）：用于高亮/特色元素的特殊强调边框。

## 3. 排版规则

### 字体族
- **展示 / 标题**：`waldenburgNormal`，回退：`waldenburgNormal Fallback, ui-sans-serif, system-ui`
- **正文 / UI**：`waldenburgNormal`，回退：`waldenburgNormal Fallback, ui-sans-serif, system-ui`
- **代码 / 技术**：`IBM Plex Mono`，回退：`ibmPlexMono Fallback, ui-monospace`
- **回退 / CJK**：`Helvetica`，回退：`Arial, Hiragino Sans GB, STXihei, Microsoft YaHei, WenQuanYi Micro Hei`

*注：waldenburgNormal 是一款定制字体。对于外部实现，可使用 Inter 或 Space Grotesk 作为无衬线替代品（几何感、略压缩的感觉）。IBM Plex Mono 可在 Google Fonts 上获取。*

### 原则
- **大尺寸下的极端负字间距**：72px 以上的展示标题使用激进的负字间距（-2.88px 至 -4.48px），创造出紧凑、工程化的质感，使 Sanity 与较宽松的编辑排版区分开来。
- **单一字体，多种声调**：waldenburgNormal 同时处理编辑展示和功能性 UI 文本。字重范围较窄（大多为 400-425，仅微型标签使用 500-600），保持声音的一致性。
- **OpenType 特性控制**：排版使用精心设计的特性设置，包括展示尺寸下的 `"cv01", "cv11", "cv12", "cv13", "ss07"` 和正文文本的 `"calt" 0`，针对不同情境微调字符替换。
- **紧凑标题，宽松正文**：标题使用 1.00-1.24 的行高（极其紧凑），而正文文本在 1.50 处呼吸自如。这种对比创造了清晰的视觉层次。
- **大写技术标签**：IBM Plex Mono 说明文字和小标签经常使用 `text-transform: uppercase` 配合紧凑行高，为技术元数据创造"系统读出"的美学。

### 网格与容器
- 最大内容宽度：约 1440px（从断点推断）
- 页面边沟：桌面端 32px，移动端 16px
- 内容区块使用全出血背景配合居中、最大宽度内容
- 多列布局：桌面端 2-3 列，移动端单列
- 卡片网格：CSS Grid 配合一致间隙（16-24px）

### 留白哲学
Sanity 在区块之间使用激进的垂直间距（64-120px），在深色画布上创造呼吸空间。区块内部，间距较紧凑（16-32px），形成密集的信息集群被慷慨的空白分隔。这种节奏赋予页面一种"幻灯片"质感——每个区块都像是独立的聚焦画面。

## 6. 深度与提升

### 阴影系统

| 级别 | 值 | 用途 |
|------|-----|------|
| Level 0（扁平） | none | 大多数元素的默认状态——深色表面仅通过颜色创造深度 |
| Level 1（微妙） | 0px 0px 0px 1px `#212121` | 类边框阴影，用于最小化围合而无需可见边框 |
| Level 2（焦点） | 0 0 0 2px `var(--color-blue-500)` | 输入框和交互元素的焦点环 |
| Level 3（叠加） | 背景模糊 + 半透明深色 | 导航叠加，模态背景 |

### 深度哲学
Sanity 的深度系统几乎完全是**色度**而非基于阴影的。提升通过表面颜色变化来传达：`#0b0b0b`（底层）-> `#212121`（提升）-> `#353535`（突出）-> `#ffffff`（反转/最高）。这种方法是深色界面的原生特性，传统投影在此会不可见。少数存在的阴影是基于环形（0px 0px 0px Npx）或基于模糊（backdrop-filter）而非偏移阴影，保持了扁平、精密工程的美学。

基于边框的围合（1px 实线 `#212121` 或 `#353535`）作为主要的空间分隔，边框深度经过校准，既可见又不主导。系统避免"悬浮卡片"美学——所有元素都感觉像是安装于表面而非悬浮其上。

## 7. 设计准则

### 应该
- 使用无彩色灰度色阶作为基础——保持纯粹的灰度纪律，没有冷暖色调
- 在所有交互元素上一致地应用电光蓝（`#0052ef`）作为通用悬停/激活状态
- 在 48px 及以上的展示标题上使用极端负字间距（-2px 至 -4.48px）
- 保持主要 CTA 为完整药丸形（99999px 圆角）配合珊瑚红（`#f36458`）
- 使用 IBM Plex Mono 大写字母作为技术标签、标签和系统元数据
- 通过表面颜色（由深到浅）而非阴影来传达深度
- 在深色画布上保持慷慨的垂直区块间距（64-120px）
- 对展示排版使用 `"cv01", "cv11", "cv12", "cv13", "ss07"` OpenType 特性

### 不应该
- 不要在中性色阶中引入冷暖色调——Sanity 的灰色是纯粹的无彩色
- 不要使用投影来表示提升——深色界面需要色度深度
- 不要使用 13px 至 99998px 之间的圆角——系统从 12px（大卡片）直接跳至药丸形（99999px）
- 不要在同一元素中混合珊瑚红 CTA 和电光蓝交互色
- 不要使用粗字重（700+）——系统最高为 600，且仅用于 11px 大写标签
- 不要在浅色表面放置浅色文字或在深色表面放置深色文字而不检查灰对灰的对比度
- 不要使用传统的偏移盒阴影——仅使用环形阴影（0 0 0 Npx）或基于边框的围合
- 不要在标题上打破紧凑行高——1.00-1.24 是范围，展示文本永远不要超过 1.5+

## 8. 响应式行为

### 断点

| 名称 | 宽度 | 行为 |
|------|------|------|
| Desktop XL | >= 1640px | 完整布局，最大内容宽度 |
| Desktop | >= 1440px | 标准桌面布局 |
| Desktop Compact | >= 1200px | 略微紧凑的桌面布局 |
| Laptop | >= 1100px | 减小列宽 |
| Tablet Landscape | >= 960px | 2 列布局开始折叠 |
| Tablet | >= 768px | 过渡区，部分元素堆叠 |
| Mobile Large | >= 720px | 接近平板布局 |
| Mobile | >= 480px | 单列，堆叠布局 |
| Mobile Small | >= 376px | 最低支持宽度 |

### 折叠策略
- **导航**：水平链接在 768px 以下折叠为汉堡菜单
- **Hero 排版**：在各断点间从 112px 缩放至 72px -> 48px -> 38px，保持紧凑的字间距比例
- **网格布局**：约 960px 处从 3 列变为 2 列，768px 以下变为单列
- **卡片网格**：移动端水平滚动而非换行（保持卡片宽高比）
- **区块间距**：移动端垂直内边距减少约 40%（120px -> 64px -> 48px）
- **按钮尺寸**：CTA 药丸保持内边距但减小字体尺寸；幽灵按钮保持固定
- **代码块**：水平滚动，保留等宽格式

### 移动端特定调整
- 全出血区块延展至边缘，内部保留 16px 边沟
- 触控目标：所有交互元素最小 44px
- 移动端尺寸下标题字间距略微放松（减少激进的负字间距）
- 图片容器从固定宽高比切换为全宽配合自动高度

### 迭代指南
1. **从深色开始**：以 `#0b0b0b` 背景、`#ffffff` 主要文字、`#b9b9b9` 次要文字为基础
2. **添加结构**：使用 `#212121` 表面和 `#353535` 边框进行围合——不用阴影
3. **应用排版**：Inter（或 Space Grotesk）配标题紧凑字间距，正文 1.50 行高
4. **色彩点缀**：添加 `#f36458` 用于 CTA，`#0052ef` 用于所有悬停/交互状态
5. **精修间距**：8px 基础单位，区块内 24-32px，区块间 64-120px
6. **技术细节**：添加 IBM Plex Mono 大写标签用于标签和元数据
7. **打磨**：确保所有交互元素悬停变为 `#0052ef`，所有按钮为药丸形或微妙 5px 圆角，边框为极细线（1px）
