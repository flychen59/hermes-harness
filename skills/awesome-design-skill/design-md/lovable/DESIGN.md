# 设计系统：Lovable

## 1. 视觉主题与氛围

Lovable 的网站通过克制的表达传递温暖。整个页面采用奶油色、羊皮纸色调的背景（`#f7f4ed`），使其与大多数开发者工具网站的冷白色调形成鲜明对比。这不是为了极简而极简——而是一种精心设计的选择，让人感觉亲切、近乎模拟质感，就像一本精心制作的手账。近乎黑色的文字（`#1c1c1c`）与温暖的奶油色背景形成的对比度，既护眼又保持了清晰的阅读性。

定制字体 Camera Plain Variable 是系统的核心武器。与那些传递"科技公司"信号的几何无衬线字体不同，Camera Plain 具有人文主义的温暖——略带圆角的末端、有机的曲线和舒适的阅读节奏。在展示尺寸（48px–60px）下，字重 600 配合强烈的负字间距（-0.9px 至 -1.5px），将标题压缩成自信、具有编辑感的陈述。该字体使用 `ui-sans-serif, system-ui` 作为后备字体，承认定制字体承载了品牌个性。

使 Lovable 视觉系统独特的是其基于透明度的深度模型。系统不是使用传统的灰度，而是将 `#1c1c1c` 以不同的透明度（0.03、0.04、0.4、0.82–0.83）进行调制，从而创建统一的色调范围。页面上的每种灰色在技术上都来自同一色调——只是透明度不同。这创造了一种几乎无法通过任意十六进制值实现的视觉一致性。边框系统也遵循这一原则：浅色分隔使用 `1px solid #eceae4`，较强的交互边界使用 `1px solid rgba(28, 28, 28, 0.4)`。

**核心特征：**
- 温暖的羊皮纸背景（`#f7f4ed`）——不是白色，不是米色，而是精心挑选的奶油色
- Camera Plain Variable 字体，具有人文主义温暖，展示尺寸下采用编辑风格的字间距
- 基于透明度的颜色系统：所有灰色都源自 `#1c1c1c` 的不同透明度级别
- 按钮内阴影技术：`rgba(255,255,255,0.2) 0px 0.5px 0px 0px inset, rgba(0,0,0,0.2) 0px 0px 0px 0.5px inset`
- 温暖的中性边框色板：`#eceae4` 用于细微分隔，`rgba(28,28,28,0.4)` 用于交互元素
- 全圆角（`9999px`）广泛应用于操作按钮和图标容器
- 聚焦状态使用 `rgba(0,0,0,0.1) 0px 4px 12px` 阴影实现柔和、温暖的强调效果
- shadcn/ui + Radix UI 组件原语配合 Tailwind CSS 实用样式

## 2. 颜色色板与角色

### 主色
- **Cream**（`#f7f4ed`）：页面背景、卡片表面、按钮表面。基础色——温暖、如纸质、人性化。
- **Charcoal**（`#1c1c1c`）：主要文本、标题、深色按钮背景。不是纯黑——有机的温暖感。
- **Off-White**（`#fcfbf8`）：深色背景上的按钮文字、微妙高亮。几乎与纯白无法区分。

### 中性色阶（基于透明度）
- **Charcoal 100%**（`#1c1c1c`）：主要文本、标题、深色表面。
- **Charcoal 83%**（`rgba(28,28,28,0.83)`）：较强的次要文本。
- **Charcoal 82%**（`rgba(28,28,28,0.82)`）：正文副本。
- **Muted Gray**（`#5f5f5d`）：次要文本、描述、说明文字。
- **Charcoal 40%**（`rgba(28,28,28,0.4)`）：交互边框、按钮轮廓。
- **Charcoal 4%**（`rgba(28,28,28,0.04)`）：微妙的悬停背景、细微着色。
- **Charcoal 3%**（`rgba(28,28,28,0.03)`）：几乎不可见的覆盖层、背景深度。

### 表面与边框
- **Light Cream**（`#eceae4`）：卡片边框、分隔线、图片轮廓。温暖的分隔线。
- **Cream Surface**（`#f7f4ed`）：卡片背景、区块填充——与页面背景相同，实现无缝融合。

### 交互色
- **Ring Blue**（`#3b82f6` 50% 透明度）：`--tw-ring-color`，Tailwind 聚焦环。
- **Focus Shadow**（`rgba(0,0,0,0.1) 0px 4px 12px`）：聚焦和激活状态阴影——柔和、温暖、漫射。

### 内阴影
- **Button Inset**（`rgba(255,255,255,0.2) 0px 0.5px 0px 0px inset, rgba(0,0,0,0.2) 0px 0px 0px 0.5px inset, rgba(0,0,0,0.05) 0px 1px 2px 0px`）：深色按钮上标志性的多层内阴影。

## 3. 排版规则

### 字体族
- **主要字体**：`Camera Plain Variable`，后备字体：`ui-sans-serif, system-ui`
- **字重范围**：400（正文/阅读），480（特殊展示），600（标题/强调）
- **特性**：可变字体，具有连续字重轴——允许精细调节中间字重如 480。

### 层级结构

| 角色 | 字体 | 大小 | 字重 | 行高 | 字间距 | 备注 |
|------|------|------|--------|-------------|----------------|-------|
| 展示主标题 | Camera Plain Variable | 60px (3.75rem) | 600 | 1.00–1.10 (紧凑) | -1.5px | 最大冲击力，编辑风格 |
| 展示备选 | Camera Plain Variable | 60px (3.75rem) | 480 | 1.00 (紧凑) | normal | 较轻的主标题变体 |
| 区块标题 | Camera Plain Variable | 48px (3.00rem) | 600 | 1.00 (紧凑) | -1.2px | 功能区块标题 |
| 副标题 | Camera Plain Variable | 36px (2.25rem) | 600 | 1.10 (紧凑) | -0.9px | 子区块 |
| 卡片标题 | Camera Plain Variable | 20px (1.25rem) | 400 | 1.25 (紧凑) | normal | 卡片标题 |
| 大正文 | Camera Plain Variable | 18px (1.13rem) | 400 | 1.38 | normal | 引言文字 |
| 正文 | Camera Plain Variable | 16px (1.00rem) | 400 | 1.50 | normal | 标准阅读文本 |
| 按钮 | Camera Plain Variable | 16px (1.00rem) | 400 | 1.50 | normal | 按钮标签 |
| 小按钮 | Camera Plain Variable | 14px (0.88rem) | 400 | 1.50 | normal | 紧凑按钮 |
| 链接 | Camera Plain Variable | 16px (1.00rem) | 400 | 1.50 | normal | 下划线装饰 |
| 小链接 | Camera Plain Variable | 14px (0.88rem) | 400 | 1.50 | normal | 页脚链接 |
| 说明文字 | Camera Plain Variable | 14px (0.88rem) | 400 | 1.50 | normal | 元数据、小字 |

### 原则
- **温暖的人文主义语调**：Camera Plain Variable 赋予 Lovable 亲切的个性。略带圆角的末端和有机曲线与大多数开发者工具使用的锐利几何无衬线字体形成对比。
- **可变字重作为设计工具**：该字体支持连续字重值（如 480），能够实现超越标准字重档次的细腻层级。字重 480 在 60px 下创建的展示风格比 semibold 更轻，但比 regular 更强。
- **大尺寸压缩**：标题使用负字间距（-0.9px 至 -1.5px）实现编辑感的冲击力。正文保持正常字间距以确保舒适阅读。
- **两个字重，清晰分工**：400（正文/UI/链接/按钮）和 600（标题/强调）。狭窄的字重范围通过大小和间距而非字重变化来创建层级。

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 尺度：8px, 10px, 12px, 16px, 24px, 32px, 40px, 56px, 80px, 96px, 128px, 176px, 192px, 208px
- 尺度在高端大幅扩展——区块使用 80px–208px 的垂直间距以获得编辑感的呼吸空间

### 网格与容器
- 最大内容宽度：约 1200px（居中）
- Hero：居中单列，巨大的垂直内边距（96px+）
- 功能区块：2–3 列网格
- 全宽页脚，多列链接布局
- 展示区块采用居中卡片网格

### 留白哲学
- **编辑感的慷慨**：Lovable 在区块边界处的间距非常宽松（80px–208px）。温暖的奶油背景使这些大面积空间感觉温馨而非空旷。
- **内容驱动的节奏**：卡片内部紧凑的间距（12–24px）与宽阔的区块间隙形成对比，创造出一种阅读节奏，在专注内容与视觉休息之间交替。
- **区块分隔**：页脚使用 `1px solid #eceae4` 边框和 16px 圆角容器。区块通过宽敞的间距而非边框线来定义。

### 边框圆角尺度
- 微型（4px）：小按钮、交互元素
- 标准（6px）：按钮、输入框、导航菜单
- 舒适（8px）：紧凑卡片、div
- 卡片（12px）：标准卡片、图片容器、模板
- 容器（16px）：大型容器、页脚区块
- 全药丸（9999px）：操作药丸、图标按钮、切换开关

## 6. 深度与层级

| 级别 | 处理方式 | 用途 |
|-------|-----------|-----|
| 扁平（Level 0） | 无阴影，奶油背景 | 页面表面，大多数内容 |
| 边框（Level 1） | `1px solid #eceae4` | 卡片、图片、分隔线 |
| 内嵌（Level 2） | `rgba(255,255,255,0.2) 0px 0.5px 0px inset, rgba(0,0,0,0.2) 0px 0px 0px 0.5px inset, rgba(0,0,0,0.05) 0px 1px 2px` | 深色按钮、主要操作 |
| 聚焦（Level 3） | `rgba(0,0,0,0.1) 0px 4px 12px` | 激活/聚焦状态 |
| 环（无障碍） | `rgba(59,130,246,0.5)` 2px 环 | 输入框的键盘聚焦 |

**阴影哲学**：Lovable 的深度系统有意保持浅层。系统不使用带有戏剧性投影的悬浮卡片，而是依靠温暖的边框（`#eceae4`）在奶油色表面上创造温和的围合感。唯一值得注意的阴影模式是深色按钮上的内阴影——一种微妙的多层技术，白色高光线位于顶边，而深色环和柔和投影处理底部。这创造了一种触感的、压入表面的感觉，而非悬浮于表面的感觉。温暖的聚焦阴影（`rgba(0,0,0,0.1) 0px 4px 12px`）特意设计为漫射且大范围，创造柔和的光晕而非锐利的轮廓。

### 装饰性深度
- Hero：主内容后方柔和、温暖的多色渐变渲染（粉色、橙色、蓝色）——氛围感，几乎不可见
- 页脚：渐变背景，温暖色调向底部过渡
- 无生硬的区块分隔线——间距和背景的温暖感处理过渡

## 7. 应做与不应做

### 应做
- 使用温暖的奶油背景（`#f7f4ed`）作为页面基础——这是品牌标志性的温暖
- 在展示尺寸下使用 Camera Plain Variable，配合负字间距（-0.9px 至 -1.5px）
- 从 `#1c1c1c` 的不同透明度级别衍生所有灰色，以保持色调统一
- 在深色按钮上使用内阴影技术，获得触感深度
- 使用 `#eceae4` 边框而非阴影作为卡片围合
- 保持字重系统狭窄：400 用于正文/UI，600 用于标题
- 仅对操作药丸和图标按钮使用全药丸圆角（9999px）
- 在激活状态应用 0.8 透明度，实现响应式触感反馈

### 不应做
- 不要使用纯白（`#ffffff`）作为页面背景——奶油色是有意为之
- 不要为卡片使用厚重的盒阴影——边框是围合机制
- 不要引入饱和的强调色——色板有意保持温暖中性
- 不要使用字重 700（粗体）——600 是系统中的最大字重
- 不要在矩形按钮上应用 9999px 圆角——药丸用于图标/操作切换
- 不要使用锐利的聚焦轮廓——系统使用基于柔和阴影的聚焦指示器
- 不要混合边框样式——`#eceae4` 用于被动元素，`rgba(28,28,28,0.4)` 用于交互元素
- 不要增加标题的字间距——Camera Plain 设计为大尺寸时保持紧凑

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 关键变化 |
|------|-------|-------------|
| 小型移动端 | <600px | 紧凑单列，减少内边距 |
| 移动端 | 600–640px | 标准移动端布局 |
| 小型平板 | 640–700px | 开始出现 2 列网格 |
| 平板 | 700–768px | 卡片网格扩展 |
| 小型桌面 | 768–1024px | 多列布局 |
| 桌面 | 1024–1280px | 完整功能布局 |
| 大型桌面 | 1280–1536px | 最大内容宽度，宽敞边距 |

### 触摸目标
- 按钮：8px 16px 内边距（舒适的触摸）
- 导航：项目间足够间距
- 药丸按钮：9999px 圆角创建大型易点击目标
- 菜单切换：6px 圆角按钮，尺寸充足

### 折叠策略
- Hero：60px → 48px → 36px 标题缩放，配合比例字间距
- 导航：水平链接 → 768px 处显示汉堡菜单
- 功能卡片：3 列 → 2 列 → 单列堆叠
- 模板画廊：网格 → 垂直堆叠卡片
- 统计栏：水平 → 垂直堆叠
- 页脚：多列 → 单列堆叠
- 区块间距：128px+ → 移动端 64px

### 图片行为
- 模板截图在所有尺寸下保持 `1px solid #eceae4` 边框
- 所有断点保持 12px 边框圆角
- 画廊图片响应式，保持一致的宽高比
- Hero 渐变在移动端柔化/简化

### 迭代指南
1. 始终使用奶油色（`#f7f4ed`）作为基础——绝不用纯白
2. 从 `#1c1c1c` 的透明度级别衍生灰色，而非使用独立的十六进制值
3. 使用 `#eceae4` 边框进行围合，而非阴影
4. 字间距随尺寸缩放：60px 时 -1.5px，48px 时 -1.2px，36px 时 -0.9px，16px 时正常
5. 两个字重：400（除标题外的所有内容）和 600（标题）
6. 深色按钮上的内阴影是标志性细节——不要省略
7. Camera Plain Variable 字重 480 仅用于特殊展示时刻
