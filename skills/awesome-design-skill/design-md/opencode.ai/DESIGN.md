# 设计系统：OpenCode

## 1. 视觉主题与氛围

OpenCode 的网站体现了终端原生、等宽字体优先的美学风格，这反映了其作为开源 AI 编程代理的身份。整个视觉系统建立在鲜明的深色背景与浅色文字对比之上，使用近乎黑色的背景（`#201d1d`）搭配温暖的米白色文字（`#fdfcfc`）。这不是一个通用的深色主题——而是一种温暖的、略带红棕色的深色，感觉像一个精致的终端模拟器而非冰冷的 IDE。深色和浅色中的温暖底色（注意 `#201d1d` 中的微妙红色通道——rgb(32, 29, 29)）创造了一种协调的、有生活气息的质感。

Berkeley Mono 是唯一的字体，确立了鲜明的等宽字体身份。每个元素——标题、正文、按钮、导航——都共享这一字体系列，创造了统一的"一切皆代码"理念。标题采用 38px 粗体和 1.50 行高，宽大且易读；正文采用 16px、字重 500，提供了比正常稍重的阅读重量，增强了屏幕上的可读性。等宽网格自然地在整个布局中强制对齐和节奏。

颜色系统特意保持极简。主色调仅由三个功能性色调组成：温暖的近乎黑色（`#201d1d`）、中等暖灰色（`#9a9898`）和明亮的米白色（`#fdfcfc`）。语义颜色借用 Apple HIG 调色板——蓝色强调（`#007aff`）、红色危险（`#ff3b30`）、绿色成功（`#30d158`）、橙色警告（`#ff9f0a`）——为界面提供熟悉、可信的信号颜色，而不会增加品牌复杂性。边框使用微妙的温暖透明色（`rgba(15, 0, 0, 0.12)`），与整个系统的温暖底色相呼应。

**核心特征：**
- Berkeley Mono 作为唯一字体——全等宽字体，无 sans-serif 或 serif 声音
- 温暖的近乎黑色主色（`#201d1d`）带有红棕色底色，非纯黑
- 米白色文字（`#fdfcfc`）带有温暖色调，非纯白
- 全局 4px 最小边框圆角——锐利、实用的边角
- 8px 基础间距系统，最大扩展至 96px
- Apple HIG 启发的语义颜色（blue、red、green、orange）
- 使用 `rgba(15, 0, 0, 0.12)` 的透明暖色边框
- 邮箱输入框采用宽大的 20px 内边距和 6px 圆角——系统中圆角最大的组件
- 单一按钮变体：深色背景、浅色文字、紧凑的垂直内边距（4px 20px）
- 下划线链接作为默认链接样式，强化以文字为中心的身份

## 2. 色彩调色板与角色

### 主色
- **OpenCode Dark**（`#201d1d`）：主背景、按钮填充、链接文字。一种带有微妙红棕色温暖的近乎黑色——rgb(32, 29, 29)。
- **OpenCode Light**（`#fdfcfc`）：深色表面上的主要文字、按钮文字。一种略带温暖的米白色，避免临床感的纯白色。
- **Mid Gray**（`#9a9898`）：次要文字、柔和链接。一种中性的暖灰色，连接深色和浅色。

### 辅助色
- **Dark Surface**（`#302c2c`）：比主深色略浅，用于提升表面和微妙区分。
- **Border Gray**（`#646262`）：较强的边框、交互元素的轮廓环。
- **Light Surface**（`#f1eeee`）：浅色模式表面、微妙的背景变化。

### 强调色
- **Accent Blue**（`#007aff`）：主要强调色、链接、交互高亮。Apple 系统蓝。
- **Accent Blue Hover**（`#0056b3`）：悬停状态的深蓝色。
- **Accent Blue Active**（`#004085`）：按下/活动状态的最深蓝色。

### 语义色
- **Danger Red**（`#ff3b30`）：错误状态、破坏性操作。Apple 系统红。
- **Danger Hover**（`#d70015`）：危险元素悬停时的深红色。
- **Danger Active**（`#a50011`）：按下危险状态的最深红色。
- **Success Green**（`#30d158`）：成功状态、正面反馈。Apple 系统绿。
- **Warning Orange**（`#ff9f0a`）：警告状态、警示信号。Apple 系统橙。
- **Warning Hover**（`#cc7f08`）：警告元素悬停时的深橙色。
- **Warning Active**（`#995f06`）：按下警告状态的最深橙色。

### 文字色阶
- **Text Muted**（`#6e6e73`）：柔和标签、禁用文字、占位符内容。
- **Text Secondary**（`#424245`）：浅色背景上的次要文字、说明文字。

### 边框色
- **Border Warm**（`rgba(15, 0, 0, 0.12)`）：主边框颜色，带有红色调的透明暖黑色。
- **Border Tab**（`#9a9898`）：Tab 下划线边框，2px 实线底部。
- **Border Outline**（`#646262`）：容器的 1px 实线轮廓边框。

## 3. 字体规则

### 字体系列
- **通用**：`Berkeley Mono`，回退字体：`IBM Plex Mono, ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, Liberation Mono, Courier New, monospace`

### 层级结构

| 角色 | 大小 | 字重 | 行高 | 备注 |
|------|------|------|------|------|
| Heading 1 | 38px (2.38rem) | 700 | 1.50 | Hero 标题、页面标题 |
| Heading 2 | 16px (1.00rem) | 700 | 1.50 | 章节标题、粗体强调 |
| Body | 16px (1.00rem) | 400 | 1.50 | 标准正文、段落 |
| Body Medium | 16px (1.00rem) | 500 | 1.50 | 链接、按钮文字、导航项 |
| Body Tight | 16px (1.00rem) | 500 | 1.00（紧凑） | 紧凑标签、Tab 项 |
| Caption | 14px (0.88rem) | 400 | 2.00（宽松） | 脚注、元数据、小标签 |

### 原则
- **一种字体，一种声音**：Berkeley Mono 被独占使用。展示、正文和代码之间没有字体变化——一切都在相同的等宽字体规范中表达。层次仅通过大小和字重实现。
- **字重即层级**：700 用于标题，500 用于交互/中等强调，400 用于正文。三个字重级别创造整个层次结构。
- **宽大行高**：1.50 作为标准行高，给等宽网格中的文字呼吸空间。Caption 上宽松的 2.00 行高创造清晰的视觉分隔。
- **紧凑用于交互**：交互元素（Tab、紧凑标签）使用 1.00 行高，创造密集的、可点击目标。

## 4. 组件样式

### 按钮

**主按钮（深色填充）**
- 背景：`#201d1d`（OpenCode Dark）
- 文字：`#fdfcfc`（OpenCode Light）
- 内边距：4px 20px
- 圆角：4px
- 字体：16px Berkeley Mono，字重 500，行高 2.00（宽松）
- 轮廓：`rgb(253, 252, 252) none 0px`
- 用途：主要 CTA、主要操作

### 输入框

**邮箱输入**
- 背景：`#f8f7f7`（浅色中性）
- 文字：`#201d1d`
- 边框：`1px solid rgba(15, 0, 0, 0.12)`
- 内边距：20px
- 圆角：6px
- 字体：Berkeley Mono，标准大小
- 用途：表单字段、邮箱收集

### 链接

**默认链接**
- 颜色：`#201d1d`
- 装饰：下划线 1px
- 字重：500
- 用途：正文内容中的主要文字链接

**浅色链接**
- 颜色：`#fdfcfc`
- 装饰：无
- 用途：深色背景上的链接、导航

**柔和链接**
- 颜色：`#9a9898`
- 装饰：无
- 用途：页脚链接、次要导航

### Tab

**Tab 导航**
- 底部边框：`2px solid #9a9898`（活动 Tab 指示器）
- 字体：16px，字重 500，行高 1.00
- 用途：章节切换、内容筛选

### 导航
- 使用 Berkeley Mono 的简洁水平布局
- 品牌标志字左对齐，等宽字体
- 链接 16px 字重 500，带下划线装饰
- 深色背景与页面背景匹配
- 无背景模糊或透明——仅纯色表面

### 图片处理
- 终端/代码截图作为 Hero 图片
- 深色终端美学搭配等宽字体
- 最小化边框，内容自身表达

### 特色组件

**Terminal Hero**
- 全宽深色终端窗口作为 Hero 元素
- 终端框架内的 ASCII 艺术/风格化 Logo
- 带语法高亮的等宽字体命令示例
- 强化产品的 CLI-first 身份

**功能列表**
- 带 Berkeley Mono 文字的项目符号功能项
- 字重 500 用于功能名称，400 用于描述
- 项目间紧凑的垂直间距
- 无卡片或边框——纯文字布局

**邮箱收集**
- 浅色背景输入框（`#f8f7f7`）与深色页面形成对比
- 宽大的 20px 内边距便于舒适输入
- 6px 圆角——系统中圆角最大的元素
- Newsletter/waitlist 模式

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 精细尺度：1px、2px、4px（用于边框和微调的 8px 以下）
- 标准尺度：8px、12px、16px、20px、24px
- 扩展尺度：32px、40px、48px、64px、80px、96px
- 系统遵循干净的 4/8px 网格，持续倍增

### 网格与容器
- 最大内容宽度：约 800-900px（窄，阅读优化）
- 单列布局作为主要模式
- 居中内容，宽大的水平边距
- Hero 章节：全宽深色终端元素
- 功能章节：单列文字块
- 页脚：多列链接网格

### 留白哲学
- **等宽节奏**：Berkeley Mono 的固定宽度特性创造了自然的垂直网格。1.50 和 2.00 的行高保持一致的节奏。
- **窄而专注**：内容被限制在窄列中，创造宽大的侧边距，将注意力集中在文字上。
- **通过间距划分章节**：无装饰性分隔线。章节通过宽大的垂直间距（48-96px）而非边框或背景变化来分隔。

### 边框圆角尺度
- 微圆（4px）：所有元素的默认值——按钮、容器、徽章
- 输入框（6px）：表单输入获得稍大的圆度
- 整个系统仅使用两个圆角值，强化实用主义美学

## 6. 深度与层级

| 级别 | 处理方式 | 用途 |
|------|----------|------|
| Flat（Level 0） | 无阴影、无边框 | 大多数元素的默认状态 |
| Border Subtle（Level 1） | `1px solid rgba(15, 0, 0, 0.12)` | 章节分隔线、输入边框、水平线 |
| Border Tab（Level 2） | `2px solid #9a9898` 仅底部 | 活动 Tab 指示器 |
| Border Outline（Level 3） | `1px solid #646262` | 容器轮廓、提升元素 |

**阴影哲学**：OpenCode 的深度系统特意保持扁平。提取的 Token 中没有 box-shadow——零阴影值被检测到。深度仅通过边框处理和背景颜色变化来传达。这种扁平化与终端美学一致：终端没有阴影，OpenCode 也一样。三个边框级别（透明暖色、Tab 指示器、实线轮廓）创造了足够的视觉层次，无需任何层级错觉。

### 装饰性深度
- 背景颜色在 `#201d1d` 和 `#302c2c` 之间变化，创造微妙的表面区分
- 12% 不透明度的透明边框提供几乎不可见的结构
- 边框颜色中的温暖红色调（`rgba(15, 0, 0, 0.12)`）将边框与整体温暖深色调色板联系起来
- 无渐变、无模糊、无环境效果——纯扁平终端美学

## 7. 交互与动效

### 悬停状态
- 链接：颜色从默认变为强调蓝（`#007aff`）或下划线样式变化
- 按钮：微妙的背景变亮或边框强调
- 强调蓝提供三阶段悬停序列：`#007aff` → `#0056b3` → `#004085`（默认 → 悬停 → 活动）
- Danger Red：`#ff3b30` → `#d70015` → `#a50011`
- Warning Orange：`#ff9f0a` → `#cc7f08` → `#995f06`

### 焦点状态
- 基于边框的焦点：增加边框不透明度或实线边框颜色
- 无基于阴影的焦点环——与扁平、无阴影美学一致
- 键盘焦点可能使用轮廓或边框颜色变化为强调蓝

### 过渡
- 预期最小化过渡——终端启发的界面倾向于即时状态变化
- 颜色过渡：100-150ms 用于微妙状态反馈
- 无缩放、旋转或复杂变换动画

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 关键变化 |
|------|------|----------|
| Mobile | <640px | 单列、减少内边距、标题缩小 |
| Tablet | 640-1024px | 内容宽度扩展、略微增加内边距 |
| Desktop | >1024px | 完整内容宽度（约 800-900px 居中）、最大留白 |

### 触摸目标
- 4px 20px 内边距的按钮提供足够的水平触摸区域
- 20px 内边距的输入框确保舒适的移动端输入
- 16px 紧凑行高的 Tab 项可能需要移动端适配

### 折叠策略
- Hero 标题：38px → 28px → 24px 在较小屏幕上
- 导航：水平链接 → 移动端汉堡菜单/抽屉
- 功能列表：保持单列、减少水平内边距
- Terminal Hero：保持全宽、减少内部内边距
- 页脚列：多列 → 堆叠单列
- 章节间距：96px → 64px → 48px 在移动端

### 图片行为
- 终端截图保持宽高比和边框处理
- 全宽元素按比例缩放
- 等宽字体因固定宽度特性在所有尺寸下保持可读性

### 迭代指南
1. Berkeley Mono 是唯一字体——永远不要引入第二种字体。大小和字重创造所有层次。
2. 保持表面扁平：无阴影、无渐变、无模糊效果。仅使用边框和背景变化。
3. 温暖底色很重要：使用 `#201d1d` 而非 `#000000`，使用 `#fdfcfc` 而非 `#ffffff`。红色温暖微妙但至关重要。
4. 边框圆角到处都是 4px，除了输入框（6px）。永远不要使用圆角药丸或大圆角。
5. 语义颜色遵循 Apple HIG：`#007aff` blue、`#ff3b30` red、`#30d158` green、`#ff9f0a` orange。每个都有悬停和活动的加深变体。
6. 三阶段交互：默认 → 悬停（加深）→ 活动（深度加深）适用于所有语义颜色。
7. 边框使用 `rgba(15, 0, 0, 0.12)`——一种温暖的透明深色，而非中性灰色。这将边框与温暖调色板联系起来。
8. 间距遵循 8px 网格：8、16、24、32、40、48、64、80、96px。仅使用 4px 进行精细调整。
