# 设计系统：Resend

## 1. 视觉主题与氛围

Resend 的网站是一个深色、电影感的画布，将邮件基础设施像奢侈品一样呈现。整个页面笼罩在纯黑色（`#000000`）中，文字以近乎白色（`#f0f0f0`）发光，创造出一种剧院般的体验，内容在虚空舞台上表演。这不是典型的开发者工具的黑暗——这是摄影画廊中受控的黑暗，每个元素都有意为之地点亮，没有任何东西争夺注意力。

字体系统是主角。三种精心挑选的字体创造了一种既具有编辑感又具有技术感的层次结构：Domaine Display（Klim Type Foundry 的衬线字体）以巨大的 96px 出现在主标题中，行高几乎不存在（1.00）且字距为负值（-0.96px），创造出像杂志封面一样的展示文字。ABC Favorit（由 Dinamo 设计）处理章节标题，使用更激进的字间距（56px 时为 -2.8px），赋予中等层级文字一种压缩的、工程感的质量。Inter 接管正文和 UI，提供干净的易读性，让展示字体闪耀。Commit Mono 作为代码块字体完成了这个字体家族。

让 Resend 独特的是其冰冷的蓝色调边框系统。Resend 不使用中性灰色边框，而是使用 `rgba(214, 235, 253, 0.19)`——一种霜状的、略带蓝色调的线条，19% 不透明度，赋予每个容器和分隔线在黑色背景上一种冷冽、晶莹的质感。结合药丸形按钮（9999px 圆角）、多色强调系统（orange、green、blue、yellow、red——每种都有自己的 CSS 变量色阶）和 OpenType 特殊样式集（`"ss01"`、`"ss03"`、`"ss04"`、`"ss11"`），结果是一个感觉高端、精确且低调自信的设计系统。

**关键特征：**
- 纯黑背景配近乎白色（`#f0f0f0`）文字——戏剧性的、画廊般的黑暗
- 三字体层次：Domaine Display（衬线主标题）、ABC Favorit（几何章节标题）、Inter（正文/UI）
- 冰冷蓝色调边框：`rgba(214, 235, 253, 0.19)`——每个边框都有冷冽、晶莹的微光
- 多色强调系统：orange、green、blue、yellow、red——每种都有编号的 CSS 变量色阶
- 药丸形按钮和标签（9999px 圆角），透明背景
- 展示字体上的 OpenType 特殊样式集（`"ss01"`、`"ss03"`、`"ss04"`、`"ss11"`）
- Commit Mono 用于代码——等宽字体作为设计元素，而非事后补充
- 使用蓝色调光环的微弱阴影：`rgba(176, 199, 217, 0.145) 0px 0px 0px 1px`

## 2. 色彩调色板与角色

### 主色
- **Void Black**（`#000000`）：页面背景，定义性的画布颜色（通过 `--color-black-12` 实现 95% 不透明度）
- **Near White**（`#f0f0f0`）：主要文字、按钮文字、高对比度元素
- **Pure White**（`#ffffff`）：`--color-white`，最大强调文字、链接高亮

### 强调色阶 — Orange
- **Orange 4**（`#ff5900`）：`--color-orange-4`，22% 不透明度——微妙的暖色光晕
- **Orange 10**（`#ff801f`）：`--color-orange-10`，主要 orange 强调色——温暖、充满活力
- **Orange 11**（`#ffa057`）：`--color-orange-11`，较浅的 orange 用于次要用途

### 强调色阶 — Green
- **Green 3**（`#22ff99`）：`--color-green-3`，12% 不透明度——微弱的翡翠色冲洗
- **Green 4**（`#11ff99`）：`--color-green-4`，18% 不透明度——成功指示器光晕

### 强调色阶 — Blue
- **Blue 4**（`#0075ff`）：`--color-blue-4`，34% 不透明度——中等 blue 强调色
- **Blue 5**（`#0081fd`）：`--color-blue-5`，42% 不透明度——更强的 blue
- **Blue 10**（`#3b9eff`）：`--color-blue-10`，明亮的 blue——链接、交互元素

### 强调色阶 — 其他
- **Yellow 9**（`#ffc53d`）：`--color-yellow-9`，温暖的金色用于警告或高亮
- **Red 5**（`#ff2047`）：`--color-red-5`，34% 不透明度——错误状态、破坏性操作

### 中性色阶
- **Silver**（`#a1a4a5`）：次要文字、柔和链接、描述
- **Dark Gray**（`#464a4d`）：三级文字、弱化内容
- **Mid Gray**（`#5c5c5c`）：悬停状态、微妙强调
- **Medium Gray**（`#494949`）：四级文字
- **Light Gray**（`#f8f8f8`）：浅色模式表面（如适用）
- **Border Gray**（`#eaeaea`）：浅色上下文边框
- **Edge Gray**（`#ececec`）：浅色表面上的微妙边框
- **Mist Gray**（`#dedfdf`）：浅色分隔线
- **Soft Gray**（`#e5e6e6`）：替代浅色边框

### 表面与覆盖
- **Frost Primary**（`#fcfdff`）：主要颜色标记（略带蓝色调，94% 不透明度）
- **White Hover**（`rgba(255, 255, 255, 0.28)`）：深色背景上的按钮悬停状态
- **White 60%**（`oklab(0.999994 ... / 0.577)`）：半透明白色用于柔和文字
- **White 64%**（`oklab(0.999994 ... / 0.642)`）：略亮的半透明白色

### 边框与阴影
- **Frost Border**（`rgba(214, 235, 253, 0.19)`）：标志性元素——冰冷蓝色调边框，19% 不透明度
- **Frost Border Alt**（`rgba(217, 237, 254, 0.145)`）：略浅的变体用于列表项
- **Ring Shadow**（`rgba(176, 199, 217, 0.145) 0px 0px 0px 1px`）：蓝色调阴影作为边框
- **Focus Ring**（`rgb(0, 0, 0) 0px 0px 0px 8px`）：重型黑色聚焦环
- **Subtle Shadow**（`rgba(0, 0, 0, 0.1) 0px 1px 3px, rgba(0, 0, 0, 0.1) 0px 1px 2px -1px`）：最小卡片抬升

## 3. 字体规则

### 字体家族
- **展示衬线体**：`domaine`（Domaine Display，Klim Type Foundry 出品）——主标题
- **展示无衬线体**：`aBCFavorit`（ABC Favorit，Dinamo 出品），备选：`ui-sans-serif, system-ui`——章节标题
- **正文 / UI**：`inter`，备选：`ui-sans-serif, system-ui`——正文、按钮、导航
- **等宽字体**：`commitMono`，备选：`ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas`
- **次要字体**：`Helvetica`——特定 UI 上下文的备选
- **系统字体**：`-apple-system, system-ui, Segoe UI, Roboto`——嵌入内容

### 原则
- **三字体编辑层次**：Domaine Display（衬线、主标题）、ABC Favorit（几何无衬线、章节）、Inter（易读正文）。每种字体都有严格的角色——它们从不越界。
- **展示文字的激进负字间距**：Domaine 为 -0.96px，ABC Favorit 为 -2.8px。展示字体感觉压缩、紧迫、精心设计——就像杂志刊头。
- **导航的正字间距**：ABC Favorit 导航链接使用 +0.35px 字间距——系统中唯一的正字间距。这创造了通风、间隔开的导航文字，与压缩的标题形成对比。
- **OpenType 作为身份**：`"ss01"`、`"ss03"`、`"ss04"`、`"ss11"` 特殊样式集在所有 ABC Favorit 和 Domaine 文字上启用，激活替代字形，赋予 Resend 字体独特的个性。
- **Commit Mono 作为设计元素**：等宽字体不只隐藏在代码块中——它被突出用于代码示例和技术内容，被视为一等视觉元素。

## 4. 组件样式

### 按钮

**主要透明药丸按钮**
- 背景：透明
- 文字：`#f0f0f0`
- 内边距：5px 12px
- 圆角：9999px（完整药丸形）
- 边框：`1px solid rgba(214, 235, 253, 0.19)`（霜冻边框）
- 悬停：背景 `rgba(255, 255, 255, 0.28)`（白色玻璃）
- 用途：深色背景上的主要 CTA

**白色实心药丸按钮**
- 背景：`#ffffff`
- 文字：`#000000`
- 内边距：5px 12px
- 圆角：9999px
- 用途：高对比度 CTA（"Get started"）

**幽灵按钮**
- 背景：透明
- 文字：`#f0f0f0`
- 圆角：4px
- 无边框
- 悬停：微妙的背景色调
- 用途：次要操作、标签项

### 卡片与容器
- 背景：透明或非常微妙的深色调
- 边框：`1px solid rgba(214, 235, 253, 0.19)`（霜冻边框）
- 圆角：16px（标准卡片）、24px（大型区块/面板）
- 阴影：`rgba(176, 199, 217, 0.145) 0px 0px 0px 1px`（光环阴影）
- 深色产品截图和代码演示作为卡片内容
- 无传统盒阴影抬升

### 输入与表单
- 文字：深色背景上 `#f0f0f0`，浅色背景上 `#000000`
- 圆角：4px
- 聚焦：基于阴影的光环
- 最小化样式——继承深色主题

### 导航
- 固定深色头部，底部霜冻边框：`1px solid rgba(214, 235, 253, 0.19)`
- "Resend" 文字标记左对齐
- ABC Favorit 14px 字重 500，导航链接 +0.35px 字间距
- 药丸形 CTA 右对齐
- 移动端：汉堡菜单折叠

### 图片处理
- 产品截图和代码演示主导内容区域
- 深色主题截图在深色背景上——无缝融合
- 圆角：图片上 12px–16px
- 全宽区块配合微妙的渐变覆盖

### 特色组件

**标签导航**
- 水平标签配合微妙的选择指示器
- 标签项：8px 圆角
- 激活状态配合微妙的背景区分

**代码预览面板**
- 使用 Commit Mono 的深色代码块
- 霜冻边框（`rgba(214, 235, 253, 0.19)`）
- 语法高亮使用多色强调标记（orange、blue、green、yellow）

**多色强调徽章**
- 每个产品功能都有自己的强调色，来自 CSS 变量色阶
- 徽章使用强调色作为低不透明度（12–42%）背景，全不透明度文字

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 色阶：1px, 2px, 4px, 5px, 6px, 7px, 8px, 10px, 12px, 16px, 20px, 24px, 30px, 32px, 40px

### 网格与容器
- 居中内容，慷慨的最大宽度
- 全宽黑色区块，内部内容受限
- 单列主区块，下方展开为功能网格
- 代码预览面板作为全宽或受限展示

### 留白哲学
- **电影感的黑色空间**：黑色背景就是留白。章节间慷慨的垂直间距（80px–120px+）创造了一种穿越黑暗滚动的体验，每个章节像场景一样浮现。
- **紧凑内容，广阔周围**：文字块和卡片内部紧凑，但漂浮在广阔的深色空间中——创造出孤立的"岛屿"内容。
- **字体主导的节奏**：巨大的展示字体（96px）创造自己的垂直节奏——每个标题都是一个视觉事件，锚定周围空间。

### 边框圆角色阶
- 尖锐（4px）：按钮（幽灵）、输入、小型交互元素
- 微妙（6px）：菜单面板、导航项
- 标准（8px）：标签、内容块
- 舒适（10px）：强调元素
- 卡片（12px）：剪贴板按钮、中型容器
- 大（16px）：功能卡片、图片、主按钮
- 区块（24px）：大型面板、区块容器
- 药丸（9999px）：主要 CTA、标签、徽章

## 6. 深度与抬升

| 层级 | 处理方式 | 用途 |
|------|----------|------|
| 平坦（层级 0） | 无阴影，透明背景 | 默认——深色虚空上的大多数元素 |
| 光环（层级 1） | `rgba(176, 199, 217, 0.145) 0px 0px 0px 1px` | 卡片、容器的阴影作为边框 |
| 霜冻边框（层级 1b） | `1px solid rgba(214, 235, 253, 0.19)` | 显式边框——按钮、分隔线、标签 |
| 微妙（层级 2） | `rgba(0, 0, 0, 0.1) 0px 1px 3px, rgba(0, 0, 0, 0.1) 0px 1px 2px -1px` | 轻微卡片抬升 |
| 聚焦（层级 3） | `rgb(0, 0, 0) 0px 0px 0px 8px` | 重型黑色聚焦环——无障碍 |

**阴影哲学**：Resend 几乎不使用阴影。在纯黑背景上，传统阴影是看不见的——你无法向虚空投射阴影。相反，Resend 通过其标志性的霜冻边框（`rgba(214, 235, 253, 0.19)`）创造深度——细薄、冰冷蓝色调的线条在黑暗中捕捉光线。这创造了一种"漂浮在太空中的玻璃面板"美学，边框是主要的深度机制。

### 装饰性深度
- 主内容背后微妙的暖色渐变光晕（orange/amber 色调）
- 产品截图通过自身内部 UI 创造视觉深度
- 无渐变背景——深度来自边框亮度和内容对比

## 7. 做与不做

### 做
- 使用纯黑（`#000000`）作为页面背景——虚空是画布
- 对所有结构线条应用霜冻边框（`rgba(214, 235, 253, 0.19)`）——它们是蓝色调的标志
- Domaine Display 仅用于主标题（96px），ABC Favorit 用于章节标题，Inter 用于其他所有内容
- 在 Domaine 和 ABC Favorit 文字上启用 OpenType `"ss01"`、`"ss04"`、`"ss11"`
- 对主要 CTA 和标签应用药丸圆角（9999px）
- 使用多色强调色阶（orange/green/blue/yellow/red）配合不透明度变体进行上下文特定高亮
- 将阴影保持在光环级别（`0px 0px 0px 1px`）——在黑色上，传统阴影不起作用
- 在 ABC Favorit 导航链接上使用 +0.35px 字间距——唯一的正字间距

### 不做
- 不要将背景提亮超过 `#000000`——纯黑虚空是不可协商的
- 不要使用中性灰色边框——所有边框必须有霜冻蓝色调
- 不要将 Domaine Display 应用于正文——它是仅用于展示的衬线字体
- 不要在同一个组件中混合强调色——每个功能有一个强调色
- 不要在深色背景上使用盒阴影进行抬升——改用霜冻边框
- 不要跳过 OpenType 特殊样式集——它们定义了字体的个性
- 不要在导航链接上使用负字间距——ABC Favorit 导航使用正的 +0.35px
- 不要使深色上的按钮不透明——透明配霜冻边框是模式

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 关键变化 |
|------|------|----------|
| 小型移动端 | <480px | 单列，紧凑内边距，76.8px 主标题 |
| 移动端 | 480–600px | 标准移动端，堆叠布局 |
| 桌面端 | >600px | 完整布局，96px 主标题，展开的章节 |

*注意：Resend 使用最小断点系统——仅检测到 480px 和 600px。设计以桌面优先，有干净的移动端折叠。*

### 触摸目标
- 药丸按钮：足够的内边距（最小 5px 12px）
- 标签项：8px 圆角，舒适的点击区域
- 导航链接使用 0.35px 字间距进行视觉分隔

### 折叠策略
- 主区块：Domaine 96px → 移动端 76.8px
- 导航：水平 → 汉堡菜单
- 功能区块：并排 → 堆叠
- 代码面板：保持宽度，需要时水平滚动
- 间距按比例压缩

### 图片行为
- 产品截图保持宽高比
- 深色截图在所有尺寸下与深色背景无缝融合
- 圆角（12px–16px）在所有断点保持

### 迭代指南
1. 从纯黑开始——一切漂浮在虚空中
2. 霜冻边框（`rgba(214, 235, 253, 0.19)`）是通用的结构元素——不是灰色、不是中性
3. 三种字体，三种角色：Domaine（主标题）、ABC Favorit（章节）、Inter（正文）——从不交叉
4. OpenType 特殊样式集在展示字体上是强制的——它们定义了个性
5. 多色强调以低不透明度（12–42%）用于背景，全不透明度用于文字
6. CTA 和徽章使用药丸形状（9999px），容器使用标准圆角（4px–16px）
7. 无阴影——使用霜冻边框在虚空中创造深度
