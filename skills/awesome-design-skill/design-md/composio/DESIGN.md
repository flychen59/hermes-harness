# 设计系统：Composio

## 1. 视觉主题与氛围

Composio 的界面是一个夜间指挥中心——一个密集的、面向开发者的黑暗世界，点缀着电光 cyan 和深 cobalt 信号。整个体验建立在近乎纯黑的画布（`#0f0f0f`）上，内容漂浮在几乎不可见的容器边框内，营造出高科技控制面板的感觉，而非传统的营销页面。这是一个向生活在黑暗终端中的开发者传递权威感的网站。

视觉语言深受代码编辑器和终端窗口美学的影响。JetBrains Mono 与 abcDiatype 的几何精确性并存，强化了这是一个由开发者*为*开发者构建的工具这一信息。装饰元素克制但有力——微妙的 cyan-blue 渐变光晕从卡片和区块散发出来，如同深海中的生物发光生物，而精选元素上的硬偏移阴影（`4px 4px`）增添了原始的、野兽派风格的边缘，使设计不会显得过于冷峻。

Composio 的独特之处在于其极端极简主义与策略性发光色彩爆发之间的张力。网站从不喧哗——标题使用紧凑的行高（0.87），将文字压缩成密集、权威的区块。颜色像稀有资源一样被配给：白色文字用于主要内容，半透明白色（`rgba(255,255,255,0.5-0.6)`）用于次要内容，品牌蓝（`#0007cd`）或电光 cyan（`#00ffff`）仅保留用于交互时刻和强调光晕。

**核心特征：**
- 漆黑画布配以近乎不可见的白色边框容器（4-12% 不透明度）
- 双字体身份：几何无衬线字体（abcDiatype）用于内容，等宽字体（JetBrains Mono）用于技术可信度
- 超紧凑标题行高（0.87-1.0），创造压缩、有力的文字区块
- 生物发光强调策略——cyan 和蓝色光晕，仿佛从内部发出光芒
- 精选交互元素上的硬偏移野兽派阴影（`4px 4px`）
- 单色层次结构，颜色仅用于最高信号时刻
- 开发者终端美学，连接营销与文档

## 2. 色彩系统与角色

### 主色
- **Composio Cobalt**（`#0007cd`）：核心品牌色——一种深沉、饱和的蓝色，谨慎用于高优先级交互元素和品牌时刻。它以安静的力量锚定身份。

### 辅助色与强调色
- **Electric Cyan**（`#00ffff`）：引人注目的强调色——以低不透明度（`rgba(0,255,255,0.12)`）用于发光按钮背景和卡片高亮。在全饱和时，它是黑暗画布上的活力对照点。
- **Signal Blue**（`#0089ff` / `rgb(0,137,255)`）：用于精选按钮边框和交互焦点状态，连接 Cobalt 和 Cyan 之间的差距。
- **Ocean Blue**（`#0096ff` / `rgb(0,150,255)`）：CTA 按钮上的强调边框色，比 Signal Blue 稍暖。

### 表面与背景
- **Void Black**（`#0f0f0f`）：主页面背景——不是纯黑，而是稍暖一点，减少暗色显示器上的眼睛疲劳。
- **Pure Black**（`#000000`）：用于卡片内部和深层嵌套容器，与页面背景形成微妙的深度区分。
- **Charcoal**（`#2c2c2c` / `rgb(44,44,44)`）：用于次要按钮边框和暗色表面上的分隔线。

### 中性色与文字
- **Pure White**（`#ffffff`）：暗色表面上的主标题和高强调文字颜色。
- **Muted Smoke**（`#444444`）：弱化的正文文字、元数据和三级内容。
- **Ghost White**（`rgba(255,255,255,0.6)`）：次要正文文字和链接标签——可见但有意后退。
- **Whisper White**（`rgba(255,255,255,0.5)`）：三级按钮文字和占位符内容。
- **Phantom White**（`rgba(255,255,255,0.2)`）：微妙的按钮背景和深层后退的 UI 边框。

### 语义与强调
- **Border Mist 12**（`rgba(255,255,255,0.12)`）：最高不透明度边框处理——用于突出的卡片边缘和内容分隔符。
- **Border Mist 10**（`rgba(255,255,255,0.10)`）：暗色表面上的标准容器边框。
- **Border Mist 08**（`rgba(255,255,255,0.08)`）：微妙的区块分隔符和次要卡片边缘。
- **Border Mist 06**（`rgba(255,255,255,0.06)`）：背景分组的近乎不可见的容器边框。
- **Border Mist 04**（`rgba(255,255,255,0.04)`）：最淡的边框——仅用于氛围分隔。
- **Light Border**（`#e0e0e0` / `rgb(224,224,224)`）：保留用于浅色表面场景（本网站罕见）。

### 渐变系统
- **Cyan Glow**：使用极低不透明度的 `#00ffff` 径向渐变，在卡片和功能区块后创造生物发光光晕。
- **Blue-to-Black Fade**：从 Composio Cobalt（`#0007cd`）渐变到 Void Black（`#0f0f0f`）的线性渐变，用于 hero 背景和区块过渡。
- **White Fog**：页面底部从暗色渐变到漫射白色/灰色的渐变，在 footer 附近创造氛围性的"地平线"效果。

## 3. 排版规则

### 字体家族
- **主字体**：`abcDiatype`，后备字体：`abcDiatype Fallback, ui-sans-serif, system-ui, Apple Color Emoji, Segoe UI Emoji, Segoe UI Symbol, Noto Color Emoji`
- **等宽字体**：`JetBrains Mono`，后备字体：`JetBrains Mono Fallback, ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, Liberation Mono, Courier New`
- **系统等宽字体**（后备）：`Menlo`、`monospace` 用于最小的行内代码

### 原则
- **压缩创造权威**：标题行高极度紧凑（0.87-1.0），使大文字感觉密集而有威严，而非轻盈装饰。
- **双重个性**：abcDiatype 承载营销声音——几何、精确、友好。JetBrains Mono 承载技术声音——可信、功能性强、开发者熟悉。
- **字重克制**：几乎所有内容都使用字重 400（regular）。字重 500（medium）保留用于小标签、徽章和精选卡片标题。字重 700（bold）仅出现在微型系统等宽场景。
- **代码使用负字间距**：JetBrains Mono 使用负字间距（-0.28px 到 -0.98px），创造密集、紧凑的代码块，感觉像真正的 IDE。
- **大写需要赢得**：`uppercase` + `letter-spacing` 处理仅保留用于极小的 overline 标签和技术标签——从不用于标题。

## 4. 组件样式

### 按钮

**Primary CTA（白色填充）**
- 背景：Pure White（`#ffffff`）
- 文字：Near Black（`oklch(0.145 0 0)`）
- 内边距：舒适（8px 24px）
- 边框：无
- 圆角：微圆（根据 token 尺度可能为 4px）
- 悬停：可能有微妙的透明度降低或轻微灰色偏移

**Cyan Accent CTA**
- 背景：Electric Cyan 12% 不透明度（`rgba(0,255,255,0.12)`）
- 文字：Near Black（`oklch(0.145 0 0)`）
- 内边距：舒适（8px 24px）
- 边框：细实线 Ocean Blue（`1px solid rgb(0,150,255)`）
- 圆角：微圆（4px）
- 在暗色背景上创造"内部发光"效果

**Ghost / Outline（Signal Blue）**
- 背景：透明
- 文字：Near Black（`oklch(0.145 0 0)`）
- 内边距：平衡（10px）
- 边框：细实线 Signal Blue（`1px solid rgb(0,137,255)`）
- 悬停：可能有填充或边框颜色变化

**Ghost / Outline（Charcoal）**
- 背景：透明
- 文字：Near Black（`oklch(0.145 0 0)`）
- 内边距：平衡（10px）
- 边框：细实线 Charcoal（`1px solid rgb(44,44,44)`）
- 用于暗色表面上的次要/三级操作

**Phantom Button**
- 背景：Phantom White（`rgba(255,255,255,0.2)`）
- 文字：Whisper White（`rgba(255,255,255,0.5)`）
- 无可见边框
- 用于深度弱化的操作

### 卡片与容器
- 背景：Pure Black（`#000000`）或透明
- 边框：极低不透明度的白色，范围从 Border Mist 04（`rgba(255,255,255,0.04)`）到 Border Mist 12（`rgba(255,255,255,0.12)`），取决于突出程度
- 圆角：几乎直角（行内元素 2px，内容卡片 4px）
- 阴影：精选卡片使用硬偏移野兽派阴影（`rgba(0,0,0,0.15) 4px 4px 0px 0px`）——一个独特的设计选择，增添原始深度
- 升起阴影：深层容器使用柔和漫射阴影（`rgba(0,0,0,0.5) 0px 8px 32px`）
- 悬停行为：可能有微妙的边框透明度增加或微弱发光效果

### 输入框与表单
- 无明确的输入 token 数据——输入框可能遵循暗色表面模式：
  - 背景：透明或 Pure Black
  - 边框：Border Mist 10（`rgba(255,255,255,0.10)`）
  - 焦点：边框变为 Signal Blue（`#0089ff`）或 Electric Cyan
  - 文字：Pure White 配 Ghost White 占位符

### 导航
- 暗色/黑色背景上的粘性顶部导航栏
- Logo（白色 SVG）：左侧 Composio 字标
- 导航链接：Pure White（`#ffffff`），标准正文大小（16px，abcDiatype）
- 导航中的 CTA 按钮：White Fill Primary 样式
- 移动端：折叠为汉堡菜单，单列布局
- 导航上有微妙的底边框（Border Mist 06-08）

### 图片处理
- 暗色主题的产品截图和 UI 模拟图占主导
- 图片位于匹配卡片系统的带边框容器内
- 功能图片后或下方有 blue/cyan 渐变光晕
- 图片无可见圆角（除容器圆角 4px 外）
- 在其卡片容器内全出血

### 独特组件

**统计数据/指标展示**
- 大型等宽数字（JetBrains Mono）——"10k+" 风格
- 紧凑布局，下方有微妙的标签文字

**代码块/终端预览**
- 使用 JetBrains Mono 的暗色容器
- 语法高亮内容
- 微妙的带边框容器（Border Mist 10）

**集成/合作伙伴 Logo 网格**
- 暗色表面上的工具 logo 网格布局
- 包含在带边框卡片内
- 展示生态系统广度

**"COMPOSIO" 品牌展示**
- 超大品牌排版——可能是页面上最大的文字
- 用作区块分隔/品牌声明
- 黑底白字，鲜明对比

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 尺度：1px, 2px, 4px, 6px, 8px, 10px, 12px, 14px, 16px, 18px, 20px, 24px, 30px, 32px, 40px
- 组件内边距：通常 10px（按钮）到 24px（CTA 按钮水平）
- 区块内边距：充裕的垂直间距（估计主要区块间 80-120px）
- 卡片内部内边距：约 24-32px

### 网格与容器
- 最大容器宽度：约 1200px，居中
- 内容区块使用单列或 2-3 列网格展示功能卡片
- Hero：居中单列，最大冲击力
- 功能区块：混合文字区块和产品截图的非对称布局

### 留白哲学
- **区块间呼吸空间**：大型垂直间隙在页面滚动中创造明显的"章节"。
- **组件内部密集**：卡片和文字区块内部紧凑（紧凑行高、最小内部内边距），创造聚焦的信息节点。
- **对比驱动的分隔**：Composio 不单纯依赖留白，而是使用边框透明度差异和微妙的背景变化来划分内容区域。

### 圆角尺度
- 近方形（2px）：行内代码 span、小标签、pre 块——最锐利的处理，传达技术精确性
- 微圆（4px）：内容卡片、图片、标准容器——通用圆角
- 药丸形（37px）：精选按钮和徽章——为关键 CTA 创造更柔和、更亲和的感觉
- 全圆（9999px+）：圆形元素、头像类容器、装饰点

## 6. 深度与升起

| 层级 | 处理方式 | 用途 |
|------|----------|------|
| 扁平（Level 0） | 无阴影，无边框 | 页面背景、行内文字 |
| 容器（Level 1） | Border Mist 04-08，无阴影 | 背景分组、微妙区块 |
| 卡片（Level 2） | Border Mist 10-12，无阴影 | 标准内容卡片、代码块 |
| 野兽派（Level 3） | 硬偏移阴影（`4px 4px`，15% 黑色） | 精选交互卡片、独特功能高亮 |
| 浮动（Level 4） | 柔和漫射阴影（`0px 8px 32px`，50% 黑色） | 模态框、覆盖层、深层升起内容 |

**阴影哲学**：Composio 谨慎而有对比地使用阴影。硬偏移野兽派阴影是标志性的——它以原始的、几乎复古计算的感觉打破流畅的暗色。柔和漫射阴影保留给真正浮动的元素。大多数深度通过边框透明度层次而非阴影来传达。

### 装饰性深度
- **Cyan Glow Halos**：在功能卡片和图片后使用低不透明度 Electric Cyan 的径向渐变光晕。创造"屏幕发光"效果，仿佛 UI 元素正在发出光芒。
- **Blue-Black Gradient Washes**：从 Composio Cobalt 到 Void Black 的线性渐变用作区块背景，增添微妙的色温变化。
- **White Fog Horizon**：页面底部从暗色到漫射白色/灰色的渐变，在 footer 前创造氛围性的"黎明"效果。

## 7. 设计准则

### 应该做的
- 使用 Void Black（`#0f0f0f`）作为主页面背景——主表面绝不使用纯白
- 保持标题行高超紧凑（0.87-1.0），创造压缩、权威的文字区块
- 使用白色透明度边框（4-12%）作为容器——这里它们比阴影更重要
- 仅将 Electric Cyan（`#00ffff`）用于高信号时刻——CTA、光晕、交互强调
- 将 abcDiatype 与 JetBrains Mono 配对，强化开发者工具身份
- 在精选元素上有意使用硬偏移阴影（`4px 4px`），展现野兽派个性
- 保持按钮文字深色（`oklch(0.145 0 0)`），即使在最暗的背景上——按钮自带表面
- 分层使用透明度边框，创造无需阴影的微妙深度
- 仅在极小的 overline 标签（12px 或更小）上使用大写 + 字间距

### 不应该做的
- 不要使用明亮背景或浅色表面作为主容器
- 不要到处应用厚重阴影——深度来自边框透明度，而非 box-shadow
- 不要将 Composio Cobalt（`#0007cd`）用作文字颜色——在暗色上太暗，在亮色上太饱和
- 不要将标题行高增加到 1.2 以上——压缩感是身份的核心
- 不要在正文或标题文字中使用粗体（700）字重——400-500 是上限
- 不要混合暖色——调色板严格冷色（蓝、cyan、白、黑）
- 不要在内容卡片上使用大于 4px 的圆角——近直角的精确性是有意的
- 不要在大表面上使用全不透明度的 Electric Cyan——它是强调色，背景最多使用 12%
- 不要使用装饰性衬线或手写字体——整个身份是几何无衬线 + 等宽
- 不要在技术内容上跳过等宽字体——JetBrains Mono 不是装饰，它是可信度信号

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 关键变化 |
|------|------|----------|
| 移动端 | <768px | 单列、汉堡导航、全宽卡片、减少区块内边距、hero 文字缩小至 ~28-40px |
| 平板 | 768-1024px | 卡片 2 列网格、精简导航、略微缩小的 hero 文字 |
| 桌面 | 1024-1440px | 完整多列布局、展开导航显示所有链接、大型 hero 排版（64px） |
| 大桌面 | >1440px | 最大宽度容器居中、充裕的水平边距 |

### 触摸目标
- 最小触摸目标：所有交互元素 44x44px
- 按钮使用舒适的内边距（最小 8px 24px），确保足够的触摸区域
- 导航链接间距足够，适合拇指导航

### 折叠策略
- **导航**：桌面端完整水平导航在移动端折叠为汉堡菜单
- **功能网格**：3 列 → 2 列 → 单列堆叠
- **Hero 文字**：64px → 40px → 28px 渐进缩放
- **区块内边距**：按比例减少，但保持充裕的垂直节奏
- **卡片**：移动端垂直堆叠，全宽处理
- **代码块**：较小视口上水平滚动而非换行

### 图片行为
- 产品截图在其容器内按比例缩放
- 暗色主题图片在所有尺寸的暗色背景上保持对比度
- 渐变光晕效果随容器大小缩放
- 断点间无可见的艺术指导变化——相同裁剪，按比例缩放
