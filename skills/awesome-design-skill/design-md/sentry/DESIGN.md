# 设计系统：Sentry

## 1. 视觉主题与氛围

Sentry 的网站是一个以深色模式优先的开发者工具界面，使用代码编辑器和终端窗口的语言风格。整个美学植根于深紫黑色背景（`#1f1633`、`#150f23`），唤起 Sentry 专为深夜调试会话而生的氛围。在这片墨黑的画布上，精心策划的紫色、粉色系列以及独特的青柠绿强调色（`#c2ef4e`）共同构建了一套既专业又充满活力的视觉系统。

字体搭配经过精心设计："Dammit Sans" 以大标题尺寸（88px，字重 700）呈现，作为带有个性与态度的展示字体，与 Sentry 不拘一格的品牌调性（"Code breaks. Fix it faster."）相得益彰；而 Rubik 则作为主力 UI 字体，服务于所有功能性文本——标题、正文、按钮、说明文字和导航。Monaco 为代码片段和技术内容提供等宽字体层，完善了开发者工具的三位一体。

Sentry 的独特之处在于拥抱"深色 IDE"美学，却不显得冷冰冰或单调乏味。温暖的紫色调取代了开发者工具典型的冷灰色，大胆的插画元素（3D 角色、色彩丰富的产品截图）点缀着深色画布。按钮系统使用标志性的柔和紫色（`#79628c`）配合内阴影，营造出一种触感，几乎有物理质感——按钮仿佛可以被按入表面。

**关键特征：**
- 深紫黑色背景（`#1f1633`、`#150f23`）——从不使用纯黑
- 温暖的紫色强调光谱：从深色（`#362d59`）到中色（`#79628c`、`#6a5fc1`）再到鲜艳（`#422082`）
- 青柠绿强调色（`#c2ef4e`）用于高可见性 CTA 和高亮
- 粉色/珊瑚色强调（`#ffb287`、`#fa7faa`）用于焦点状态和次要高亮
- "Dammit Sans" 展示字体，用于大标题尺寸的品牌个性表达
- Rubik 作为主要 UI 字体，配合大写字母间距标签
- Monaco 等宽字体用于代码元素
- 按钮上的内阴影营造触感深度
- 毛玻璃效果，使用 `blur(18px) saturate(180%)`

## 2. 色彩调色板与角色

### 主要品牌色
- **Deep Purple**（`#1f1633`）：主要背景色，品牌的标志性颜色
- **Darker Purple**（`#150f23`）：更深区域、页脚、次要背景
- **Border Purple**（`#362d59`）：边框、分隔线、细微的结构线条

### 强调色
- **Sentry Purple**（`#6a5fc1`）：主要交互色——链接、悬停状态、焦点环
- **Muted Purple**（`#79628c`）：按钮背景、次要交互元素
- **Deep Violet**（`#422082`）：选择下拉框、激活状态、高强调表面
- **Lime Green**（`#c2ef4e`）：高可见性强调、特殊链接、徽章高亮
- **Coral**（`#ffb287`）：焦点状态背景、温暖强调
- **Pink**（`#fa7faa`）：焦点轮廓、装饰性强调

### 文字颜色
- **Pure White**（`#ffffff`）：深色背景上的主要文字
- **Light Gray**（`#e5e7eb`）：次要文字、弱化内容
- **Code Yellow**（`#dcdcaa`）：语法高亮、代码标记

### 表面与叠加层
- **Glass White**（`rgba(255, 255, 255, 0.18)`）：毛玻璃按钮背景
- **Glass Dark**（`rgba(54, 22, 107, 0.14)`）：毛玻璃元素上的悬停叠加
- **Input White**（`#ffffff`）：表单输入背景（浅色上下文）
- **Input Border**（`#cfcfdb`）：表单字段边框

### 阴影
- **Ambient Glow**（`rgba(22, 15, 36, 0.9) 0px 4px 4px 9px`）：深紫色环境阴影
- **Button Hover**（`rgba(0, 0, 0, 0.18) 0px 0.5rem 1.5rem`）：提升的悬停状态
- **Card Shadow**（`rgba(0, 0, 0, 0.1) 0px 10px 15px -3px`）：标准卡片层级
- **Inset Button**（`rgba(0, 0, 0, 0.1) 0px 1px 3px 0px inset`）：触感按压效果

## 3. 排版规则

### 字体家族
- **Display**：`Dammit Sans`——用于大标题的品牌个性字体
- **Primary UI**：`Rubik`，备用字体：`-apple-system, system-ui, Segoe UI, Helvetica, Arial`
- **Monospace**：`Monaco`，备用字体：`Menlo, Ubuntu Mono`

### 层级结构

| 角色 | 字体 | 尺寸 | 字重 | 行高 | 字母间距 | 备注 |
|------|------|------|------|------|----------|------|
| Display Hero | Dammit Sans | 88px (5.50rem) | 700 | 1.20（紧凑） | normal | 最大冲击力，品牌调性 |
| Display Secondary | Dammit Sans | 60px (3.75rem) | 500 | 1.10（紧凑） | normal | 次要大标题文字 |
| Section Heading | Rubik | 30px (1.88rem) | 400 | 1.20（紧凑） | normal | 主要区块标题 |
| Sub-heading | Rubik | 27px (1.69rem) | 500 | 1.25（紧凑） | normal | 功能区块标题头 |
| Card Title | Rubik | 24px (1.50rem) | 500 | 1.25（紧凑） | normal | 卡片和区块标题 |
| Feature Title | Rubik | 20px (1.25rem) | 600 | 1.25（紧凑） | normal | 强调功能名称 |
| Body | Rubik | 16px (1.00rem) | 400 | 1.50 | normal | 标准正文文本 |
| Body Emphasis | Rubik | 16px (1.00rem) | 500–600 | 1.50 | normal | 加粗正文、导航项 |
| Nav Label | Rubik | 15px (0.94rem) | 500 | 1.40 | normal | 导航链接 |
| Uppercase Label | Rubik | 15px (0.94rem) | 500 | 1.25（紧凑） | normal | `text-transform: uppercase` |
| Button Text | Rubik | 14px (0.88rem) | 500–700 | 1.14–1.29（紧凑） | 0.2px | `text-transform: uppercase` |
| Caption | Rubik | 14px (0.88rem) | 500–700 | 1.00–1.43 | 0.2px | 通常为大写 |
| Small Caption | Rubik | 12px (0.75rem) | 600 | 2.00（宽松） | normal | 细微注释 |
| Micro Label | Rubik | 10px (0.63rem) | 600 | 1.80（宽松） | 0.25px | `text-transform: uppercase` |
| Code | Monaco | 16px (1.00rem) | 400–700 | 1.50 | normal | 代码块、技术文本 |

### 原则
- **双重个性**：Dammit Sans 在展示尺寸上带来不拘一格的品牌个性；Rubik 为所有功能性元素提供干净的专业感。
- **大写作为系统**：按钮、说明文字、标签和微型文本都使用 `text-transform: uppercase` 配合细微的字母间距（0.2px–0.25px），在整个系统中创建"技术标签"模式。
- **字重分层**：Rubik 使用 400（正文）、500（强调/导航）、600（标题/强强调）、700（按钮/CTA）——一个清晰的四层字重系统。
- **紧凑标题，宽松正文**：所有标题使用 1.10–1.25 行高；正文使用 1.50；小注释在微小尺寸下扩展到 2.00 以提高可读性。

## 4. 组件样式

### 按钮

**Primary Muted Purple（主要柔和紫色）**
- 背景：`#79628c`（rgb(121, 98, 140)）
- 文字：`#ffffff`，大写，14px，字重 500–700，字母间距 0.2px
- 边框：`1px solid #584674`
- 圆角：13px
- 阴影：`rgba(0, 0, 0, 0.1) 0px 1px 3px 0px inset`（触感内嵌）
- 悬停：提升阴影 `rgba(0, 0, 0, 0.18) 0px 0.5rem 1.5rem`

**Glass White（毛玻璃白）**
- 背景：`rgba(255, 255, 255, 0.18)`（毛玻璃效果）
- 文字：`#ffffff`
- 内边距：8px
- 圆角：12px（左对齐变体：`12px 0px 0px 12px`）
- 阴影：`rgba(0, 0, 0, 0.08) 0px 2px 8px`
- 悬停背景：`rgba(54, 22, 107, 0.14)`
- 用途：深色表面上的次要操作

**White Solid（纯白实心）**
- 背景：`#ffffff`
- 文字：`#1f1633`
- 内边距：12px 16px
- 圆角：8px
- 悬停：背景过渡为 `#6a5fc1`，文字变为白色
- 焦点：背景 `#ffb287`（coral），轮廓 `rgb(106, 95, 193) solid 0.125rem`
- 用途：深色背景上的高可见性 CTA

**Deep Violet（深紫罗兰，用于选择/下拉）**
- 背景：`#422082`
- 文字：`#ffffff`
- 内边距：8px 16px
- 圆角：8px

### 输入框

**文本输入框**
- 背景：`#ffffff`
- 文字：`#1f1633`
- 边框：`1px solid #cfcfdb`
- 内边距：8px 12px
- 圆角：6px
- 焦点：边框颜色保持 `#cfcfdb`，阴影 `rgba(0, 0, 0, 0.15) 0px 2px 10px inset`

### 链接
- **深色背景默认**：`#ffffff`，下划线装饰
- **悬停**：颜色过渡为 `#6a5fc1`（Sentry Purple）
- **紫色链接**：`#6a5fc1` 默认，悬停时下划线
- **青柠强调链接**：`#c2ef4e` 默认，悬停过渡为 `#6a5fc1`
- **深色上下文链接**：`#362d59`，悬停过渡为 `#ffffff`

### 卡片与容器
- 背景：半透明或深紫色表面
- 圆角：8px–12px
- 阴影：`rgba(0, 0, 0, 0.1) 0px 10px 15px -3px`
- 背景滤镜：`blur(18px) saturate(180%)` 用于毛玻璃效果

### 导航
- 深色透明头部，覆盖在大标题内容之上
- Rubik 15px 字重 500 用于导航链接
- 白色文字，悬停变为 Sentry Purple（`#6a5fc1`）
- 大写标签配合 0.2px 字母间距用于分类
- 移动端：汉堡菜单，全宽展开

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 尺度：1px, 2px, 4px, 5px, 6px, 8px, 12px, 16px, 24px, 32px, 40px, 44px, 45px, 47px

### 网格与容器
- 最大内容宽度：1152px（XL 断点）
- 响应式内边距：2rem（移动端）→ 4rem（平板及以上）
- 内容在容器内居中
- 全宽深色区块，内部内容居中

### 断点
| 名称 | 宽度 | 关键变化 |
|------|------|----------|
| Mobile | < 576px | 单列，堆叠布局 |
| Small Tablet | 576–640px | 微调宽度 |
| Tablet | 640–768px | 开始两列布局 |
| Small Desktop | 768–992px | 完整导航可见 |
| Desktop | 992–1152px | 标准布局 |
| Large Desktop | 1152–1440px | 最大宽度内容 |

### 留白哲学
- **深色呼吸空间**：区块之间宽裕的垂直间距（64px–80px+）让深色背景作为视觉休憩
- **内容岛屿**：功能区块是漂浮在深紫色海洋中的独立模块，每个都有自己的内部间距节奏
- **非对称内边距**：按钮使用非对称内边距模式（12px 16px、8px 12px），感觉有机而非僵化

### 圆角尺度
- 最小（6px）：表单输入、小型交互元素
- 标准（8px）：按钮、卡片、容器
- 舒适（10px–12px）：较大容器、毛玻璃面板
- 圆润（13px）：主要柔和按钮
- 胶囊（18px）：图片容器、徽章

## 6. 深度与层级

| 层级 | 处理方式 | 用途 |
|------|----------|------|
| 下陷（Level -1） | 内阴影 `rgba(0, 0, 0, 0.1) 0px 1px 3px inset` | 主要按钮（触感按压效果） |
| 平坦（Level 0） | 无阴影 | 默认表面、深色背景 |
| 表面（Level 1） | `rgba(0, 0, 0, 0.08) 0px 2px 8px` | 毛玻璃按钮、细微卡片 |
| 提升层（Level 2） | `rgba(0, 0, 0, 0.1) 0px 10px 15px -3px` | 卡片、浮动面板 |
| 突出层（Level 3） | `rgba(0, 0, 0, 0.18) 0px 0.5rem 1.5rem` | 悬停状态、模态框 |
| 环境光（Level 4） | `rgba(22, 15, 36, 0.9) 0px 4px 4px 9px` | 大标题周围的深紫色环境光晕 |

**阴影哲学**：Sentry 使用内阴影（按钮感觉被按入表面）和环境光晕（内容从深色背景中散发）的独特组合。深紫色环境阴影（`rgba(22, 15, 36, 0.9)`）是标志性特征——它创造出一种生物发光的质感，内容似乎在散发自己的紫色调光芒。

## 7. 最佳实践

### 应该做的
- 使用深紫色背景（`#1f1633`、`#150f23`）——从不使用纯黑（`#000000`）
- 在主要按钮上应用内阴影以获得触感按压效果
- 只在大标题/展示标题使用 Dammit Sans——其他所有内容使用 Rubik
- 在按钮和标签上应用 `text-transform: uppercase` 配合 `letter-spacing: 0.2px`
- 谨慎使用青柠绿强调色（`#c2ef4e`）以获得最大冲击力
- 使用毛玻璃效果（`blur(18px) saturate(180%)`）创建分层表面
- 保持温暖的紫色阴影调——阴影应该感觉带有紫色调，而非中性灰色
- 使用 Rubik 的四层字重系统：400（正文）、500（导航/强调）、600（标题）、700（CTA）

### 不应该做的
- 不要使用纯黑（`#000000`）作为背景——始终使用温暖的紫黑色
- 不要将 Dammit Sans 应用于正文文本或 UI 元素——它仅用于展示
- 不要使用标准灰色（`#666`、`#999`）作为边框——使用带紫调的灰色（`#362d59`、`#584674`）
- 不要在按钮上省略大写处理——这是系统级模式
- 不要使用尖角（0px 圆角）——所有交互元素至少 6px 圆角
- 不要在同一组件中混合青柠绿强调色与珊瑚色/粉色强调色
- 不要在主要按钮上使用平面（非内嵌）阴影——触感质量是标志性特征
- 不要忘记大写文本的字母间距——至少 0.2px

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 关键变化 |
|------|------|----------|
| Mobile | <576px | 单列，汉堡导航，堆叠 CTA |
| Tablet | 576–768px | 开始两列功能网格 |
| Small Desktop | 768–992px | 完整导航，并排布局 |
| Desktop | 992–1152px | 最大宽度容器，完整布局 |
| Large | >1152px | 内容最大宽度保持，宽裕边距 |

### 折叠策略
- 大标题文字：88px Dammit Sans → 60px → 移动端缩放
- 导航：水平 → 汉堡菜单滑出
- 功能区块：并排 → 堆叠卡片
- 按钮：行内 → 移动端全宽堆叠
- 容器内边距：4rem → 2rem

### 迭代指南
1. 始终从深紫色背景开始——调色板是为深色模式构建的

5. 叠加面板使用毛玻璃，主要表面使用纯紫色
6. Rubik 处理 90% 的排版——Dammit Sans 仅用于大标题
