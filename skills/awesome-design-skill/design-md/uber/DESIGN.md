# 设计系统：Uber

## 1. 视觉主题与氛围

Uber 的设计语言是自信极简主义的典范——一个黑白世界，每个像素都有其目的，没有任何装饰是无意义的。整个体验建立在一个鲜明的二元对立之上：漆黑（`#000000`）和纯白（`#ffffff`），几乎没有中调灰色来稀释信息。这不是一个尚未完成设计的初创公司的贫乏极简主义——而是一个已经如此成熟、可以轻声细语的品牌所表现出的刻意克制。

标志性字体 UberMove 是一种专有的几何无衬线字体，具有鲜明的方形、工程化特质。UberMove Bold 在 52px 大小下的标题承载着广告牌的份量——权威、直接、不容置疑。配套字体 UberMoveText 以中等字重（500）处理正文和按钮，具有略微柔和、更易读的特点。它们共同创建了一个感觉像交通地图的排版系统：清晰、高效、专为快速浏览而设计。

让 Uber 设计真正独特的是其全出血摄影和插图的使用，配合药丸形状的交互元素（999px border-radius）。导航标签、CTA 按钮和类别选择器都共享这种胶囊形状，创造了一种触感友好、易于拇指操作的界面语言，这无疑是 Uber 的标志。插图——温暖、略带风格化的司机、乘客和城市场景——为原本可能冷冰冰的单色系统注入了人情味。网站在白色内容区域和全黑页脚之间交替，基于卡片的布局使用最轻柔的阴影（rgba(0,0,0,0.12-0.16)）来创造微妙的提升感，而不破坏扁平的美学。

**核心特征：**
- 纯黑白基础，UI 界面中几乎没有中调灰色
- UberMove（标题）+ UberMoveText（正文/UI）——专有几何无衬线字体家族
- 所有元素皆为药丸形状：按钮、标签、导航项都使用 999px border-radius
- 温暖、人性化的插图与鲜明的单色界面形成对比
- 基于卡片的布局，配以极淡的阴影（0.12-0.16 不透明度）
- 8px 间距网格，紧凑、信息密集的布局
- 大胆的摄影作品作为全出血 Hero 背景
- 黑色页脚锚定页面，营造深沉、高对比度的环境

## 2. 色彩调色板与角色

### 主色
- **Uber Black**（`#000000`）：定义性的品牌颜色——用于主要按钮、标题、导航文字和页脚。不是"近黑色"或"偏黑色"，而是真实、不妥协的纯黑。
- **Pure White**（`#ffffff`）：主要表面颜色和反向文字。用于页面背景、卡片表面和黑色元素上的文字。

### 交互与按钮状态
- **Hover Gray**（`#e2e2e2`）：白色按钮悬停状态——一种干净、冷调的浅灰色，提供清晰的反馈，不带温度感。
- **Hover Light**（`#f3f3f3`）：凸起的白色按钮的微妙悬停——几乎不可见的灰色，用于轻柔的交互反馈。
- **Chip Gray**（`#efefef`）：次要/筛选按钮和导航标签的背景——一种中性的、超浅的灰色。

### 文字与内容
- **Body Gray**（`#4b4b4b`）：次要文字和页脚链接——一种真正的中灰色，没有暖调或冷调偏向。
- **Muted Gray**（`#afafaf`）：三级文字、弱化的页脚链接和占位内容。

### 边框与分隔
- **Border Black**（`#000000`）：用于结构性包围的细边框——少量用于分隔线和表单容器。

### 阴影与深度
- **Shadow Light**（`rgba(0, 0, 0, 0.12)`）：标准卡片提升——为内容卡片提供羽毛般的轻微浮起感。
- **Shadow Medium**（`rgba(0, 0, 0, 0.16)`）：浮动操作按钮和覆盖层的稍强提升效果。
- **Button Press**（`rgba(0, 0, 0, 0.08)`）：次要按钮活动/按下状态的内阴影。

### 链接状态
- **Default Link Blue**（`#0000ee`）：带下划线的文字链接的标准浏览器蓝色——用于正文内容。
- **Link White**（`#ffffff`）：深色表面上的链接——用于页脚和深色区域。
- **Link Black**（`#000000`）：浅色表面上的链接，带下划线装饰。

### 渐变系统
- Uber 的设计**完全没有渐变**。黑白二元对立和平面色块创造了所有视觉层次。系统中没有任何地方出现渐变——每个表面都是纯色，每个过渡都是硬边或阴影。

## 3. 排版规则

### 字体家族
- **标题 / 展示**：`UberMove`，回退字体：`UberMoveText, system-ui, Helvetica Neue, Helvetica, Arial, sans-serif`
- **正文 / UI**：`UberMoveText`，回退字体：`system-ui, Helvetica Neue, Helvetica, Arial, sans-serif`

*注意：UberMove 和 UberMoveText 是专有字体。对于外部实现，使用 `system-ui` 或 Inter 作为最接近的可用替代。UberMove 的几何、方形比例特征可以用 Inter 或 DM Sans 来近似。*

### 层级结构

| 角色 | 字体 | 大小 | 字重 | 行高 | 备注 |
|------|------|------|--------|-------------|-------|
| 展示 / Hero | UberMove | 52px (3.25rem) | 700 | 1.23（紧凑） | 最大冲击力，广告牌级存在感 |
| 区块标题 | UberMove | 36px (2.25rem) | 700 | 1.22（紧凑） | 主要区块锚点 |
| 卡片标题 | UberMove | 32px (2rem) | 700 | 1.25（紧凑） | 卡片和功能标题 |
| 副标题 | UberMove | 24px (1.5rem) | 700 | 1.33 | 次要区块标题 |
| 小标题 | UberMove | 20px (1.25rem) | 700 | 1.40 | 紧凑标题，列表标题 |
| 导航 / UI 大号 | UberMoveText | 18px (1.13rem) | 500 | 1.33 | 导航链接，突出 UI 文字 |
| 正文 / 按钮 | UberMoveText | 16px (1rem) | 400-500 | 1.25-1.50 | 标准正文，按钮标签 |
| 说明文字 | UberMoveText | 14px (0.88rem) | 400-500 | 1.14-1.43 | 元数据，描述，小链接 |
| 微型文字 | UberMoveText | 12px (0.75rem) | 400 | 1.67（宽松） | 细则，法律文本 |

### 原则
- **粗体标题，中等正文**：UberMove 标题独占 700（粗体）字重——每个标题都以广告牌级别的力量呈现。UberMoveText 正文和 UI 文字使用 400-500，通过字重对比创造清晰的视觉层次。
- **紧凑的标题行高**：所有标题使用 1.22-1.40 之间的行高——紧凑有力，专为浏览而非阅读设计。
- **功能性排版**：没有任何装饰性的文字处理。没有 letter-spacing，没有 text-transform，没有装饰性大小。每个文字元素都服务于直接的沟通目的。
- **两种字体，严格分工**：UberMove 专用于标题。UberMoveText 专用于正文、按钮、链接和 UI。界限从不逾越。

## 4. 组件样式

### 按钮

**主要黑色按钮（CTA）**
- 背景：Uber Black（`#000000`）
- 文字：Pure White（`#ffffff`）
- 内边距：10px 12px
- 圆角：999px（完整药丸形状）
- 轮廓：无
- 焦点：内嵌环 `rgb(255,255,255) 0px 0px 0px 2px`
- 主要操作按钮——大胆、高对比度、无法忽视

**次要白色按钮**
- 背景：Pure White（`#ffffff`）
- 文字：Uber Black（`#000000`）
- 内边距：10px 12px
- 圆角：999px（完整药丸形状）
- 悬停：背景变为 Hover Gray（`#e2e2e2`）
- 焦点：背景变为 Hover Gray，出现内嵌环
- 用于深色表面或作为主要黑色按钮旁边的次要操作

**标签 / 筛选按钮**
- 背景：Chip Gray（`#efefef`）
- 文字：Uber Black（`#000000`）
- 内边距：14px 16px
- 圆角：999px（完整药丸形状）
- 激活：内阴影 `rgba(0,0,0,0.08)`
- 导航标签，类别选择器，筛选切换

**浮动操作按钮**
- 背景：Pure White（`#ffffff`）
- 文字：Uber Black（`#000000`）
- 内边距：14px
- 圆角：999px（完整药丸形状）
- 阴影：`rgba(0,0,0,0.16) 0px 2px 8px 0px`
- 变换：`translateY(2px)` 轻微偏移
- 悬停：背景变为 `#f3f3f3`
- 地图控件，返回顶部，浮动 CTA

### 卡片与容器
- 背景：白色页面上的 Pure White（`#ffffff`）；没有明显的卡片背景区分
- 边框：默认无——卡片由阴影定义，而非描边
- 圆角：标准内容卡片使用 8px；特色/推广卡片使用 12px
- 阴影：`rgba(0,0,0,0.12) 0px 4px 16px 0px` 用于标准提升
- 卡片内容密集，内部内边距最小
- 图片主导的卡片使用全出血图像，文字覆盖或在下方

### 输入框与表单
- 文字：Uber Black（`#000000`）
- 背景：Pure White（`#ffffff`）
- 边框：1px 实线黑色（`#000000`）——唯一显眼出现边框的地方
- 圆角：8px
- 内边距：标准舒适间距
- 焦点：没有提取的自定义焦点状态——依赖标准浏览器焦点环

### 导航
- 粘性顶部导航，白色背景
- Logo：Uber 文字标记/图标，24x24px，黑色
- 链接：UberMoveText，14-18px，字重 500，Uber Black 颜色
- 药丸形状的导航标签，Chip Gray（`#efefef`）背景用于类别导航（"Ride"、"Drive"、"Business"、"Uber Eats"）
- 菜单切换：圆形按钮，50% border-radius
- 移动端：汉堡菜单模式

### 图片处理
- 温暖、手绘风格场景（特色区域使用插图而非照片）
- 插图风格：略带风格化的人物，插图内使用温暖色调，现代感
- Hero 区域使用大胆的摄影或插图作为全宽背景
- 用于应用下载 CTA 的二维码
- 所有图像在卡片中时使用标准的 8px 或 12px border-radius

### 独特组件

**类别药丸导航**
- 顶层导航的水平药丸按钮行（"Ride"、"Drive"、"Business"、"Uber Eats"、"About"）
- 每个药丸：Chip Gray 背景，黑色文字，999px 圆角
- 激活状态由黑色背景和白色文字表示（反转）

**双操作 Hero**
- 分割 Hero：左侧文字/CTA，右侧地图/插图
- 并排两个输入框用于上车/目的地
- "See prices" CTA 按钮使用黑色药丸

**提前规划卡片**
- 推广"Uber Reserve"和行程规划等功能的卡片
- 插图为主，温暖、以人为本的图像
- 底部有白色文字的黑色 CTA 按钮

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 比例：4px, 6px, 8px, 10px, 12px, 14px, 16px, 18px, 20px, 24px, 32px
- 按钮内边距：10px 12px（紧凑）或 14px 16px（舒适）
- 卡片内部内边距：约 24-32px
- 区块垂直间距：宽裕但高效——主要区块之间约 64-96px

### 网格与容器
- 最大容器宽度：约 1136px，居中
- Hero：分割布局，左侧文字，右侧视觉
- 功能区块：2 列卡片网格或全宽单列
- 页脚：黑色背景上的多列链接网格
- 全宽区块延伸至视口边缘

### 留白哲学
- **高效，而非空旷**：Uber 的留白是功能性的——足以分隔，但永远不会让人感觉空荡。这是交通系统的间距：紧凑、清晰、目标导向。
- **内容密集的卡片**：卡片紧密地打包信息，内部间距最小，依靠阴影和圆角来定义边界。
- **区块呼吸空间**：主要区块获得宽裕的垂直间距，但在区块内部，元素紧密排列。

### 圆角比例
- 尖角（0px）：交互元素中不使用方角
- 标准（8px）：内容卡片，输入框，列表框
- 舒适（12px）：特色卡片，较大容器，链接卡片
- 完整药丸（999px）：所有按钮、标签、导航项、药丸
- 圆形（50%）：头像图片，图标容器，圆形控件

## 6. 深度与提升

| 层级 | 处理方式 | 用途 |
|-------|-----------|-----|
| 扁平（Level 0） | 无阴影，纯色背景 | 页面背景，内联内容，文字区块 |
| 微妙（Level 1） | `rgba(0,0,0,0.12) 0px 4px 16px` | 标准内容卡片，功能块 |
| 中等（Level 2） | `rgba(0,0,0,0.16) 0px 4px 16px` | 提升卡片，覆盖元素 |
| 浮动（Level 3） | `rgba(0,0,0,0.16) 0px 2px 8px` + translateY(2px) | 浮动操作按钮，地图控件 |
| 按下（Level 4） | `rgba(0,0,0,0.08) inset`（999px 扩散） | 活动/按下按钮状态 |
| 焦点环 | `rgb(255,255,255) 0px 0px 0px 2px inset` | 键盘焦点指示器 |

**阴影哲学**：Uber 将阴影纯粹作为结构工具使用，从不作装饰。阴影始终是极低不透明度（0.08-0.16）的黑色，创造分隔内容层所需的最小提升。模糊半径适中（8-16px）——足以感觉自然，但从不夸张。没有彩色阴影，没有层叠阴影堆栈，没有环境光晕效果。深度更多通过黑白区块对比来传达，而非阴影提升。

## 7. 应做与不应做

### 应做
- 使用纯黑（`#000000`）和纯白（`#ffffff`）作为主要调色板——鲜明对比即是 Uber
- 所有按钮、标签和药丸形导航元素使用 999px border-radius
- 所有标题使用 UberMove Bold（700）以获得广告牌级别的冲击力
- 使用极淡阴影（0.12-0.16 不透明度）进行卡片提升——几乎不可见
- 保持紧凑、信息密集的布局风格——Uber 优先考虑效率而非空旷
- 使用温暖、以人为本的插图来柔化单色界面
- 内容卡片使用 8px 圆角，特色容器使用 12px
- 导航和突出 UI 文字使用 UberMoveText 字重 500
- 黑色主要按钮与白色次要按钮配对用于双操作布局

### 不应做
- 不要在 UI 界面中引入颜色——Uber 界面严格使用黑、白、灰
- 按钮上不要使用小于 999px 的圆角——完整药丸形状是核心标识元素
- 不要使用厚重阴影或高不透明度的投影——深度是微妙的
- 任何地方都不要使用衬线字体——Uber 排版完全是几何无衬线字体
- 不要创建过度留白的空旷布局——Uber 的密度是刻意的
- 不要使用渐变或颜色覆盖——每个表面都是扁平纯色
- 不要将 UberMove 混入正文或将 UberMoveText 混入标题——层次严格
- 不要使用装饰性边框——边框是功能性的（输入框、分隔线）或完全不存在
- 不要用灰白或近黑色柔化黑白对比——二元对立是刻意的

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 关键变化 |
|------|-------|-------------|
| 移动端小屏 | 320px | 最小布局，单列，堆叠输入，紧凑排版 |
| 移动端 | 600px | 标准移动端，堆叠布局，汉堡导航 |
| 平板小屏 | 768px | 开始出现两列网格，扩展卡片布局 |
| 平板 | 1119px | 完整平板布局，并排 Hero 内容 |
| 桌面小屏 | 1120px | 激活桌面网格，水平导航药丸 |
| 桌面 | 1136px | 完整桌面布局，最大容器宽度，分割 Hero |

### 触摸目标
- 所有药丸按钮：最小 44px 高度（10-14px 垂直内边距 + 行高）
- 导航标签：宽裕的 14px 16px 内边距，舒适的拇指点击
- 圆形控件（菜单、关闭）：50% 圆角确保大型、易于点击的目标
- 卡片表面在移动端作为全区域触摸目标

### 折叠策略
- **导航**：水平药丸导航折叠为带圆形切换的汉堡菜单
- **Hero**：分割布局（文字 + 地图/视觉）堆叠为单列——文字在上，视觉在下
- **输入框**：并排的上车/目的地输入垂直堆叠
- **功能卡片**：2 列网格折叠为全宽堆叠卡片
- **标题**：52px 展示文字依次缩小至 36px、32px、24px、20px
- **页脚**：多列链接网格折叠为手风琴或堆叠单列
- **类别药丸**：小屏幕上水平滚动溢出

### 图片行为
- 插图在其容器内按比例缩放
- Hero 图像保持宽高比，小屏幕上可能裁剪
- 二维码区域在移动端隐藏（应用下载转为直接商店链接）
- 卡片图像在所有尺寸保持 8-12px 圆角

### 迭代指南
1. 每次专注于一个组件
2. 参考严格的黑白调色板——"使用 Uber Black (#000000)"，而非"让它变暗"
3. 始终为按钮和药丸指定 999px 圆角——这对 Uber 标识不可协商
4. 明确描述字体家族——"标题使用 UberMove Bold，标签使用 UberMoveText Medium"
5. 对于阴影，使用"轻柔阴影 (rgba(0,0,0,0.12) 0px 4px 16px)"——从不用厚重投影
6. 保持布局紧凑和信息密集——Uber 追求效率，而非空旷
7. 插图应温暖且人性化——描述"温暖色调中的风格化人物"而非抽象形状
8. 将黑色 CTA 与白色次要按钮配对，实现平衡的双操作布局
