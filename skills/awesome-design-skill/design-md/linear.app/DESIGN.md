# 设计系统：Linear

## 1. 视觉主题与氛围

Linear 的网站是深色优先产品设计的典范 —— 一个近乎纯黑的画布（`#08090a`），内容如同星光般从黑暗中浮现。整体印象是极致精密的工程感：每个元素都存在于精心校准的亮度层级中，从几乎不可见的边框（`rgba(255,255,255,0.05)`）到柔和明亮的文字（`#f7f8f8`）。这不是在浅色设计上套用深色主题 —— 黑暗本身就是原生媒介，信息密度通过白色透明度的微妙渐变来管理，而非色彩变化。

字体系统完全基于 Inter Variable，全局启用了 OpenType 特性 `"cv01"` 和 `"ss03"`，赋予字体更简洁、更具几何感的特征。Inter 的字重范围非常广泛 —— 从 300（轻量正文）到 510（中等，Linear 的标志性字重）再到 590（半粗强调）。510 字重尤其独特：它介于常规和中等之间，创造出一种不张扬的微妙强调感。在展示尺寸（72px、64px、48px）下，Inter 使用激进的负字间距（-1.584px 到 -1.056px），形成紧凑、权威的标题，给人工程化而非设计感的印象。Berkeley Mono 作为等宽字体伴侣，用于代码和技术标签，备选字体包括 ui-monospace、SF Mono 和 Menlo。

色彩系统几乎完全是非彩色的 —— 深色背景配白色/灰色文字 —— 点缀着唯一的品牌强调色：Linear 标志性的靛紫色调（背景用 `#5e6ad2`，交互强调用 `#7170ff`）。这个强调色使用得克制而有目的性，仅出现在 CTA、激活状态和品牌元素上。边框系统使用超细、半透明的白色边框（`rgba(255,255,255,0.05)` 到 `rgba(255,255,255,0.08)`），在不产生视觉噪点的情况下构建结构，如同月光下绘制的线框图。

**核心特征：**
- 深色原生模式：`#08090a` 营销背景，`#0f1011` 面板背景，`#191a1b` 抬升表面
- Inter Variable 全局启用 `"cv01", "ss03"` —— 几何变体带来更简洁的美学
- 标志性字重 510（介于常规和中等之间）用于大多数 UI 文字
- 展示尺寸使用激进的负字间距（72px 时 -1.584px，48px 时 -1.056px）
- 品牌靛紫色调：`#5e6ad2`（背景）/ `#7170ff`（强调）/ `#828fff`（悬停）—— 系统中唯一的彩色
- 全局使用半透明白色边框：`rgba(255,255,255,0.05)` 到 `rgba(255,255,255,0.08)`
- 按钮背景接近零透明度：`rgba(255,255,255,0.02)` 到 `rgba(255,255,255,0.05)`
- 多层阴影配合内嵌变体，在深色表面营造深度感
- Radix UI 原语作为组件基础（检测到 6 个原语）
- 成功绿色（`#27a644`、`#10b981`）仅用于状态指示器

## 2. 色彩调色板与角色

### 背景表面
- **Marketing Black**（`#010102` / `#08090a`）：最深的背景 —— 主页区块和营销页面的画布。近乎纯黑，带有难以察觉的蓝冷色调底色。
- **Panel Dark**（`#0f1011`）：侧边栏和面板背景。比营销黑高一个层次。
- **Level 3 Surface**（`#191a1b`）：抬升的表面区域、卡片背景、下拉菜单。
- **Secondary Surface**（`#28282c`）：最浅的深色表面 —— 用于悬停状态和略微抬升的组件。

### 文字与内容
- **Primary Text**（`#f7f8f8`）：近乎白色，带有微暖色调。默认文字颜色 —— 不是纯白，避免在深色背景上造成眼睛疲劳。
- **Secondary Text**（`#d0d6e0`）：冷银灰色，用于正文、描述和次要内容。
- **Tertiary Text**（`#8a8f98`）：柔和灰色，用于占位符、元数据和弱化内容。
- **Quaternary Text**（`#62666d`）：最弱的文字 —— 时间戳、禁用状态、微妙标签。

### 品牌与强调色
- **Brand Indigo**（`#5e6ad2`）：主要品牌色 —— 用于 CTA 按钮背景、品牌标志和关键交互表面。
- **Accent Violet**（`#7170ff`）：更亮的变体，用于交互元素 —— 链接、激活状态、选中项。
- **Accent Hover**（`#828fff`）：更浅、更饱和的变体，用于强调元素的悬停状态。
- **Security Lavender**（`#7a7fad`）：柔和的靛蓝色，专门用于安全相关的 UI 元素。

### 状态色
- **Green**（`#27a644`）：主要成功/激活状态。用于"进行中"指示器。
- **Emerald**（`#10b981`）：次要成功色 —— 胶囊徽章、完成状态。

### 边框与分隔线
- **Border Primary**（`#23252a`）：实色深色边框，用于明显的分隔。
- **Border Secondary**（`#34343a`）：稍浅的实色边框。
- **Border Tertiary**（`#3e3e44`）：最浅的实色边框变体。
- **Border Subtle**（`rgba(255,255,255,0.05)`）：超微妙的半透明边框 —— 默认选择。
- **Border Standard**（`rgba(255,255,255,0.08)`）：标准半透明边框，用于卡片、输入框、代码块。
- **Line Tint**（`#141516`）：几乎不可见的线条，用于最细微的分隔。
- **Line Tertiary**（`#18191a`）：稍明显一点的分隔线。

### 浅色模式中性色（用于浅色主题场景）
- **Light Background**（`#f7f8f8`）：浅色模式下的页面背景。
- **Light Surface**（`#f3f4f5` / `#f5f6f7`）：微妙的表面着色。
- **Light Border**（`#d0d6e0`）：浅色场景下的可见边框。
- **Light Border Alt**（`#e6e6e6`）：备选的更浅边框。
- **Pure White**（`#ffffff`）：卡片表面、高亮。

### 遮罩层
- **Overlay Primary**（`rgba(0,0,0,0.85)`）：模态框/对话框背景 —— 极深以实现焦点隔离。

## 3. 字体规则

### 字体族
- **主要字体**：`Inter Variable`，备选字体：`SF Pro Display, -apple-system, system-ui, Segoe UI, Roboto, Oxygen, Ubuntu, Cantarell, Open Sans, Helvetica Neue`
- **等宽字体**：`Berkeley Mono`，备选字体：`ui-monospace, SF Mono, Menlo`
- **OpenType 特性**：全局启用 `"cv01", "ss03"` —— cv01 提供替代的小写字母 'a'（单层结构），ss03 调整特定字母形状以获得更简洁的几何外观。

### 原则
- **510 是标志性字重**：Linear 使用 Inter Variable 的 510 字重（介于常规 400 和中等 500 之间）作为默认强调字重。这创造了一种微妙的加粗感，没有传统中等或半粗字重的厚重感。
- **大尺寸压缩**：展示尺寸使用逐渐收紧的字间距 —— 72px 时 -1.584px，64px 时 -1.408px，48px 时 -1.056px，32px 时 -0.704px。低于 24px 时，字间距趋于正常。
- **OpenType 即身份**：`"cv01", "ss03"` 不是装饰 —— 它们将 Inter 转变为 Linear 独特的字体，赋予其更几何、更有目的性的特征。
- **三层字重系统**：400（阅读）、510（强调/UI）、590（强烈强调）。300 字重仅出现在刻意弱化的场景中。

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 尺度：1px, 4px, 7px, 8px, 11px, 12px, 16px, 19px, 20px, 22px, 24px, 28px, 32px, 35px
- 7px 和 11px 数值暗示光学对齐的微调
- 主要节奏：8px、16px、24px、32px（标准 8px 网格）

### 网格与容器
- 最大内容宽度：约 1200px
- 主页：居中单列，充裕的垂直内边距
- 功能区：2–3 列网格用于功能卡片
- 全宽深色区块，内部最大宽度约束
- 更新日志：单列时间线布局

### 留白哲学
- **黑暗即空间**：在 Linear 的深色画布上，空白不是白色 —— 而是缺失。近黑色背景本身就是留白，内容从中浮现。
- **压缩的标题，扩展的周围**：72px 展示文字配合 -1.584px 字间距是密集压缩的，但位于宽阔的深色内边距中。排版密度与空间慷慨之间的对比创造了张力。
- **区块隔离**：每个功能区之间有充裕的垂直内边距（80px+），无需可见分隔线 —— 深色背景提供了天然分隔。

### 边框圆角尺度
- 微型（2px）：内联徽章、工具栏按钮、微妙标签
- 标准（4px）：小型容器、列表项
- 舒适（6px）：按钮、输入框、功能元素
- 卡片（8px）：卡片、下拉菜单、弹出框
- 面板（12px）：面板、特色卡片、区块容器
- 大型（22px）：大型面板元素
- 全胶囊（9999px）：标签、筛选胶囊、状态标签
- 圆形（50%）：图标按钮、头像、状态点

## 6. 深度与层级

| 级别 | 处理方式 | 用途 |
|-------|-----------|-----|
| 扁平（Level 0） | 无阴影，`#010102` 背景 | 页面背景、最深的画布 |
| 微妙（Level 1） | `rgba(0,0,0,0.03) 0px 1.2px 0px` | 工具栏按钮、微抬升 |
| 表面（Level 2） | `rgba(255,255,255,0.05)` 背景 + `1px solid rgba(255,255,255,0.08)` 边框 | 卡片、输入框、容器 |
| 内嵌（Level 2b） | `rgba(0,0,0,0.2) 0px 0px 12px 0px inset` | 凹陷面板、内阴影 |
| 环绕（Level 3） | `rgba(0,0,0,0.2) 0px 0px 0px 1px` | 边框即阴影技术 |
| 抬升（Level 4） | `rgba(0,0,0,0.4) 0px 2px 4px` | 浮动元素、下拉菜单 |
| 对话框（Level 5） | 多层堆叠：`rgba(0,0,0,0) 0px 8px 2px, rgba(0,0,0,0.01) 0px 5px 2px, rgba(0,0,0,0.04) 0px 3px 2px, rgba(0,0,0,0.07) 0px 1px 1px, rgba(0,0,0,0.08) 0px 0px 1px` | 弹出框、命令面板、模态框 |
| 焦点 | `rgba(0,0,0,0.1) 0px 4px 12px` + 额外层 | 交互元素的键盘焦点 |

**阴影哲学**：在深色表面上，传统阴影（深色上的深色）几乎不可见。Linear 通过使用半透明白色边框作为主要深度指示器来解决这个问题。层级不是通过阴影暗度传达，而是通过背景亮度阶梯 —— 每个层级略微增加表面背景的白色透明度（`0.02` → `0.04` → `0.05`），创造微妙的堆叠效果。内阴影技术（`rgba(0,0,0,0.2) 0px 0px 12px 0px inset`）为凹陷面板创造了独特的"下沉"效果，增加了传统深色主题所缺乏的维度深度。

## 7. 宜与忌

### 宜
- 在所有文字上使用启用了 `"cv01", "ss03"` 的 Inter Variable —— 这些特性对 Linear 的字体身份至关重要
- 使用字重 510 作为默认强调字重 —— 它是 Linear 的标志性中间字重
- 在展示尺寸应用激进的负字间距（72px 时 -1.584px，48px 时 -1.056px）
- 在近黑色背景上构建：`#08090a` 用于营销，`#0f1011` 用于面板，`#191a1b` 用于抬升表面
- 使用半透明白色边框（`rgba(255,255,255,0.05)` 到 `rgba(255,255,255,0.08)`）而非实色深色边框
- 保持按钮背景几乎透明：`rgba(255,255,255,0.02)` 到 `rgba(255,255,255,0.05)`
- 将品牌靛蓝（`#5e6ad2` / `#7170ff`）仅保留用于主要 CTA 和交互强调
- 使用 `#f7f8f8` 作为主要文字 —— 不要用纯 `#ffffff`，那会太刺眼
- 应用亮度堆叠模型：更深 = 更深背景，抬升 = 稍浅背景

### 忌
- 不要使用纯白（`#ffffff`）作为主要文字 —— `#f7f8f8` 可防止眼睛疲劳
- 不要为按钮使用实色背景 —— 透明度是系统核心（rgba 白色 0.02–0.05）
- 不要装饰性地使用品牌靛蓝 —— 它仅保留用于交互/CTA 元素
- 不要在展示文字上使用正字间距 —— 大尺寸 Inter 始终使用负值
- 不要在深色背景上使用可见/不透明边框 —— 边框应该是极细的半透明白色
- 不要跳过 OpenType 特性（`"cv01", "ss03"`）—— 没有它们，就是普通 Inter，不是 Linear 的 Inter
- 不要使用字重 700（粗体）—— Linear 的最大字重是 590，510 是主力
- 不要在 UI 框架中引入暖色调 —— 调色板只有冷灰色和蓝紫强调色
- 不要在深色表面使用投影来表现层级 —— 改用背景亮度阶梯

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 关键变化 |
|------|-------|-------------|
| Mobile Small | <600px | 单列、紧凑内边距 |
| Mobile | 600–640px | 标准移动端布局 |
| Tablet | 640–768px | 开始双列网格 |
| Desktop Small | 768–1024px | 完整卡片网格、扩展内边距 |
| Desktop | 1024–1280px | 标准桌面、完整导航 |
| Large Desktop | >1280px | 完整布局、充裕边距 |

### 触控目标
- 按钮使用舒适的内边距，最小 6px 圆角
- 导航链接 13–14px，有足够间距
- 胶囊标签有 10px 水平内边距以保证触控可访问性
- 图标按钮 50% 圆角确保圆形、易点击的目标
- 搜索触发器位置突出，点击区域充裕

### 折叠策略
- 主页：72px → 48px → 32px 展示文字，字间距按比例调整
- 导航：水平链接 + CTA → 768px 时折叠为汉堡菜单
- 功能卡片：3 列 → 2 列 → 单列堆叠
- 产品截图：保持宽高比，可能减少内边距
- 更新日志：时间线在所有尺寸保持单列
- 页脚：多列 → 堆叠单列
- 区块间距：80px+ → 移动端 48px

### 图片行为
- 仪表板截图在所有尺寸保持边框处理
- 主页视觉在移动端简化（更少浮动 UI 元素）
- 产品截图使用响应式尺寸和一致的圆角
- 深色背景确保截图在任何视口自然融合

### 迭代指南
1. 始终在所有 Inter 文字上设置 font-feature-settings `"cv01", "ss03"` —— 这对 Linear 的外观是不可妥协的
2. 字间距随字体大小缩放：72px 时 -1.584px，48px 时 -1.056px，32px 时 -0.704px，16px 以下正常
3. 三种字重：400（阅读）、510（强调/导航）、590（宣告）

6. 边框始终是半透明白色，深色背景上从不用实色深色
7. 任何代码或技术内容使用 Berkeley Mono，其他所有内容使用 Inter Variable
