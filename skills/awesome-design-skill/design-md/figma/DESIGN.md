# 设计系统：Figma

## 1. 视觉主题与氛围

Figma 的界面是自我设计的工具——排版精妙的杰作，自定义可变字体（figmaSans）在刀锋细（字重 320）和粗体（字重 700）之间调制，还有大多数字重系统从未探索的不寻常中间值（330、340、450、480、540）。这种精细的字重控制使每个文字元素都有精确校准的视觉重量，通过微差而非"常规对粗体"的粗暴工具创造层级。

页面呈现迷人的双重性：界面装饰严格黑白（实际上只检测到 `#000000` 和 `#ffffff` 作为颜色），而英雄区和产品展示爆发出充满活力的多色渐变——电光绿、明亮黄、深紫、热粉。这种分离意味着设计系统本身是无色的，将产品多彩的输出视为英雄内容。Figma 的营销页面本质上是一面白色画廊墙，展示多彩艺术。

让 Figma 在可变字体之外独特的是其圆形和药丸几何。按钮使用 50px 圆角（药丸）或 50%（图标按钮的完美圆形），创造有机、工具调色板般的感觉。虚线轮廓聚焦指示器（`dashed 2px`）是一个深思熟虑的设计选择，呼应 Figma 编辑器中的选择手柄——网站的 UI 语言引用产品的 UI 语言。

**核心特征：**
- 自定义可变字体（figmaSans）配不寻常字重站点：320、330、340、450、480、540、700
- 严格黑白界面装饰——颜色只存在于产品内容中
- figmaMono 用于大写技术标签，配宽字间距
- 药丸（50px）和圆形（50%）按钮几何
- 虚线聚焦轮廓，呼应 Figma 的编辑器选择手柄
- 充满活力的多色英雄渐变（绿、黄、紫、粉）
- 全局启用 OpenType `"kern"` 特性
- 全程负字间距——甚至正文在 -0.14px 到 -0.26px

## 2. 色彩调色板与角色

### 主色
- **Pure Black**（`#000000`）：所有文字、所有实心按钮、所有边框。界面唯一的"颜色"。
- **Pure White**（`#ffffff`）：所有背景、白色按钮、深色表面上的文字。二元性的另一半。

*注意：Figma 的营销网站界面层只使用这两种颜色。所有鲜艳颜色只出现在产品截图、英雄渐变和嵌入内容中。*

### 表面与背景
- **Pure White**（`#ffffff`）：主要页面背景和卡片表面。
- **Glass Black**（`rgba(0, 0, 0, 0.08)`）：次要圆形按钮和玻璃效果的微妙深色覆盖。
- **Glass White**（`rgba(255, 255, 255, 0.16)`）：深色/彩色表面上按钮的毛玻璃覆盖。

### 渐变系统
- **Hero Gradient**：使用电光绿、明亮黄、深紫和热粉的充满活力多站渐变。这个渐变是英雄区的视觉签名——代表工具的创意可能性。
- **Product Section Gradients**：各个产品区域（Design、Dev Mode、Prototyping）可能在其展示中使用独特的颜色主题。

## 3. 排版规则

### 字体家族
- **主要**：`figmaSans`，回退：`figmaSans Fallback, SF Pro Display, system-ui, helvetica`
- **等宽 / 标签**：`figmaMono`，回退：`figmaMono Fallback, SF Mono, menlo`

### 层级

| 角色 | 字体 | 大小 | 字重 | 行高 | 字间距 | 备注 |
|------|------|------|--------|-------------|----------------|-------|
| 展示 / 英雄 | figmaSans | 86px (5.38rem) | 400 | 1.00 (紧凑) | -1.72px | 最大冲击，极端字距 |
| 章节标题 | figmaSans | 64px (4rem) | 400 | 1.10 (紧凑) | -0.96px | 功能区块标题 |
| 副标题 | figmaSans | 26px (1.63rem) | 540 | 1.35 | -0.26px | 强调区块文字 |
| 轻副标题 | figmaSans | 26px (1.63rem) | 340 | 1.35 | -0.26px | 轻字重区块文字 |
| 功能标题 | figmaSans | 24px (1.5rem) | 700 | 1.45 | normal | 粗体卡片标题 |
| 大正文 | figmaSans | 20px (1.25rem) | 330–450 | 1.30–1.40 | -0.1px 到 -0.14px | 描述，介绍 |
| 正文 / 按钮 | figmaSans | 16px (1rem) | 330–400 | 1.40–1.45 | -0.14px 到 normal | 标准正文，导航，按钮 |
| 轻正文 | figmaSans | 18px (1.13rem) | 320 | 1.45 | -0.26px 到 normal | 轻字重正文 |
| 等宽标签 | figmaMono | 18px (1.13rem) | 400 | 1.30 (紧凑) | 0.54px | 大写区块标签 |
| 等宽小字 | figmaMono | 12px (0.75rem) | 400 | 1.00 (紧凑) | 0.6px | 大写小标签 |

### 原则
- **可变字体精确性**：figmaSans 使用大多数系统从未触及的字重——320、330、340、450、480、540。这通过微妙字重差异而非戏剧性跳跃创造层级。330 和 340 的差异几乎无法察觉，但在结构上很重要。
- **轻字重为基础**：大多数正文使用 320–340（比典型 400"常规"更轻），创造空灵、轻盈的阅读体验，符合设计工具美学。
- **全面启用 Kern**：每个文字元素启用 OpenType `"kern"` 特性——字距调整不是可选的，是结构性的。
- **默认负字距**：甚至正文也使用 -0.1px 到 -0.26px 字间距，创造普遍紧凑的文字。展示文字进一步压缩到 -0.96px 和 -1.72px。
- **等宽用于结构**：figmaMono 大写配正字间距（0.54px–0.6px）创造技术路标标签。

## 4. 组件样式

### 按钮

**黑色实心（药丸）**
- 背景：Pure Black（`#000000`）
- 文字：Pure White（`#ffffff`）
- 圆角：圆形（50%）用于图标按钮
- 聚焦：虚线 2px 轮廓
- 最大强调

**白色药丸**
- 背景：Pure White（`#ffffff`）
- 文字：Pure Black（`#000000`）
- 内边距：8px 18px 10px（垂直不对称）
- 圆角：药丸（50px）
- 聚焦：虚线 2px 轮廓
- 深色/彩色表面的标准 CTA

**玻璃深色**
- 背景：`rgba(0, 0, 0, 0.08)`（微妙深色覆盖）
- 文字：Pure Black
- 圆角：圆形（50%）
- 聚焦：虚线 2px 轮廓
- 浅色表面的次要操作

**玻璃浅色**
- 背景：`rgba(255, 255, 255, 0.16)`（毛玻璃）
- 文字：Pure White
- 圆角：圆形（50%）
- 聚焦：虚线 2px 轮廓
- 深色/彩色表面的次要操作

### 卡片与容器
- 背景：Pure White
- 边框：无或最小
- 圆角：6px（小容器），8px（图片，卡片，对话框）
- 阴影：微妙到中等高度效果
- 产品截图作为卡片内容

### 导航
- 白色上的干净水平导航
- 标志：黑色 Figma 字标
- 产品标签：药丸形状（50px）标签导航
- 链接：黑色文字，下划线 1px 装饰
- CTA：黑色药丸按钮
- 悬停：通过 CSS 变量的文字颜色

### 独特组件

**产品标签栏**
- 水平药丸形状标签（50px 圆角）
- 每个标签代表一个 Figma 产品区域（Design、Dev Mode、Prototyping 等）
- 活动标签高亮

**英雄渐变区块**
- 全宽充满活力多色渐变背景
- 白色文字覆盖，86px 展示标题
- 产品截图在渐变内浮动

**虚线聚焦指示器**
- 所有交互元素在聚焦时使用 `dashed 2px` 轮廓
- 引用 Figma 编辑器中的选择手柄
- 连接网站和产品的元设计选择

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 刻度：1px、2px、4px、4.5px、8px、10px、12px、16px、18px、24px、32px、40px、46px、48px、50px

### 网格与容器
- 最大容器宽度：最大 1920px
- 英雄区：全宽渐变配居中内容
- 产品区块：交替展示
- 页脚：深色全宽区块
- 响应式从 559px 到 1920px

### 留白哲学
- **画廊般的节奏**：慷慨间距让每个产品区块作为自己的展览呼吸。
- **颜色区块作为视觉呼吸**：渐变英雄和产品展示在单色界面区块之间提供色彩缓解。

### 边框圆角刻度
- 最小（2px）：小链接元素
- 微妙（6px）：小容器，分隔符
- 舒适（8px）：卡片，图片，对话框
- 药丸（50px）：标签按钮，CTA
- 圆形（50%）：图标按钮，圆形元素

## 6. 深度与高度

| 级别 | 处理 | 用途 |
|-------|-----------|-----|
| 扁平（Level 0） | 无阴影 | 页面背景，大多数文字 |
| 表面（Level 1） | 渐变/深色区块上的白色卡片 | 卡片，产品展示 |
| 悬浮（Level 2） | 微妙阴影 | 浮动卡片，悬停状态 |

**阴影哲学**：Figma 谨慎使用阴影。主要深度机制是**背景对比**（彩色/深色区块上的白色内容）和产品截图本身的固有维度。

## 7. 做与不做

### 做
- 使用 figmaSans 配精确可变字重（320–540）——精细字重控制就是设计
- 保持界面严格黑白——颜色只来自产品内容
- 所有交互元素使用药丸（50px）和圆形（50%）几何
- 应用虚线 2px 聚焦轮廓——标志性无障碍模式
- 所有文字启用 `"kern"` 特性
- 标签使用 figmaMono 大写配正字间距
- 全程应用负字间距（-0.1px 到 -1.72px）

### 不做
- 不要添加界面颜色——单色调色板是绝对的
- 不要使用标准字重（400、500、600、700）——使用可变字体的独特站点（320、330、340、450、480、540）
- 按钮不要使用尖角——只用药丸和圆形几何
- 不要使用实线聚焦轮廓——虚线是签名
- 正文字体字重不要超过 450——轻字重美学是核心
- 正文文字不要使用正字间距——总是负的

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 关键变化 |
|------|-------|-------------|
| 小移动端 | <560px | 紧凑布局，堆叠 |
| 平板 | 560–768px | 微调 |
| 小桌面 | 768–960px | 2 列布局 |
| 桌面 | 960–1280px | 标准布局 |
| 大桌面 | 1280–1440px | 扩展 |
| 超宽 | 1440–1920px | 最大宽度 |

### 折叠策略
- 英雄文字：86px → 64px → 48px
- 产品标签：移动端水平滚动
- 功能区块：堆叠单列
- 页脚：多列 → 堆叠

## 9. Agent 提示指南

### 快速颜色参考
- 一切："Pure Black (#000000)" 和 "Pure White (#ffffff)"
- 玻璃深色："rgba(0, 0, 0, 0.08)"
- 玻璃浅色："rgba(255, 255, 255, 0.16)"

### 示例组件提示
- "在充满活力多色渐变（绿、黄、紫、粉）上创建英雄区。标题 86px figmaSans 字重 400，行高 1.0，字间距 -1.72px。白色文字。白色药丸 CTA 按钮（50px 圆角，8px 18px 内边距）。"
- "设计产品标签栏配药丸形状按钮（50px 圆角）。活动：黑色背景，白色文字。非活动：透明，黑色文字。figmaSans 20px 字重 480。"
- "构建区块标签：figmaMono 18px，大写，字间距 0.54px，黑色文字。启用 Kern。"
- "创建正文 20px figmaSans 字重 330，行高 1.40，字间距 -0.14px。白色上 Pure Black。"

### 迭代指南
1. 精确使用可变字重站点：320、330、340、450、480、540、700
2. 界面总是黑 + 白——永不向装饰添加颜色
3. 虚线聚焦轮廓，非实线
4. 正文字间距总是负的，等宽标签总是正的
5. 按钮/标签用药丸（50px），图标按钮用圆形（50%）
