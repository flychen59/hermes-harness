# 设计系统：BMW

## 1. 视觉主题与氛围

BMW 的网站是汽车工程的视觉化——一个传达精准、性能和德国工业信心的设计系统。页面在深邃的深色英雄章节（展示全出血汽车摄影）和干净的白色内容区域之间交替，创造出类似豪华汽车展厅的电影节奏，车辆在黑暗中被灯光照亮。BMW CI2020 设计语言（其企业形象刷新）定义了每个元素。

排版建立在 BMWTypeNextLatin 之上——一种专有字体，有两个变体：BMWTypeNextLatin Light（字重 300）用于巨大的大写展示标题，BMWTypeNextLatin Regular 用于正文和 UI 文本。60px 大写标题字重 300 是定义性的排版姿态——轻字重类型低语权威而非喊叫。后备堆栈包括 Helvetica 和日文字体（Hiragino、Meiryo），反映 BMW 的全球存在。

使 BMW 独特的是其 CSS 变量驱动的主题系统。上下文感知变量（`--site-context-highlight-color: #1c69d4`、`--site-context-focus-color: #0653b6`、`--site-context-metainfo-color: #757575`）表明设计系统为多品牌、多上下文部署而构建，颜色可以全局交换。蓝色强调色（`#1c69d4`）是 BMW 的标志性蓝色——仅用于交互元素和聚焦状态，从不装饰性使用。检测到零 border-radius——BMW 的设计是棱角分明、尖角、毫不妥协地几何化。

**关键特征：**
- BMWTypeNextLatin Light（字重 300）大写用于展示——低语的权威
- BMW Blue（`#1c69d4`）作为唯一强调色——仅用于交互元素
- 检测到零 border-radius——棱角分明、尖角、工业几何
- 深色英雄摄影 + 白色内容章节——展厅照明节奏
- CSS 变量驱动主题：`--site-context-*` 令牌用于品牌灵活性
- 导航强调使用字重 900——与 300 展示形成极端对比
- 全程紧凑行高（1.15–1.30）——压缩、高效、德国工程
- 全出血汽车摄影作为主要视觉内容

## 2. 色彩调色板与角色

### 主品牌色
- **Pure White**（`#ffffff`）：`--site-context-theme-color`，主表面，卡片背景
- **BMW Blue**（`#1c69d4`）：`--site-context-highlight-color`，主要交互强调
- **BMW Focus Blue**（`#0653b6`）：`--site-context-focus-color`，键盘聚焦和激活状态

### 中性刻度
- **Near Black**（`#262626`）：浅色表面上的主文本，深色链接文本
- **Meta Gray**（`#757575`）：`--site-context-metainfo-color`，次要文本，元数据
- **Silver**（`#bbbbbb`）：三级文本，柔和链接，页脚元素

### 交互状态
- 所有链接悬停变白（`#ffffff`）——表明主要是深色表面导航
- 文本链接使用下划线：悬停时无——干净的交互

### 阴影
- 最小阴影系统——深度通过摄影和明/暗章节对比实现

## 3. 排版规则

### 字体家族
- **Display Light**：`BMWTypeNextLatin Light`，后备字体：`Helvetica, Arial, Hiragino Kaku Gothic ProN, Hiragino Sans, Meiryo`
- **正文 / UI**：`BMWTypeNextLatin`，相同后备堆栈

### 层级

| 角色 | 字体 | 大小 | 字重 | 行高 | 备注 |
|------|------|------|--------|-------------|-------|
| Display Hero | BMWTypeNextLatin Light | 60px (3.75rem) | 300 | 1.30 (紧凑) | `text-transform: uppercase` |
| Section Heading | BMWTypeNextLatin | 32px (2.00rem) | 400 | 1.30 (紧凑) | 主要章节标题 |
| Nav Emphasis | BMWTypeNextLatin | 18px (1.13rem) | 900 | 1.30 (紧凑) | 导航粗体项 |
| Body | BMWTypeNextLatin | 16px (1.00rem) | 400 | 1.15 (紧凑) | 标准正文 |
| Button Bold | BMWTypeNextLatin | 16px (1.00rem) | 700 | 1.20–2.88 | CTA 按钮 |
| Button | BMWTypeNextLatin | 16px (1.00rem) | 400 | 1.15 (紧凑) | 标准按钮 |

### 原则
- **轻展示，重导航**：字重 300 用于英雄标题创造低语的优雅；字重 900 用于导航创造鲜明的权威。这种极端字重对比（300 vs 900）是标志性排版张力。
- **通用大写展示**：60px 英雄始终大写——创造纪念碑式、建筑般的品质。
- **全程紧凑**：行高 1.15 到 1.30 贯穿整个系统。没有任何东西呼吸——每行都压缩、高效、德国工程。
- **单一字体家族**：BMWTypeNextLatin 处理从 60px 展示到 16px 正文的所有内容——通过一种字体的不同字重实现统一。

## 4. 组件样式

### 按钮
- 文本：16px BMWTypeNextLatin，主要字重 700，次要字重 400
- 行高：1.15–2.88（大变化表明内边距驱动的尺寸）
- 边框：深色表面上的白色底边框（`1px solid #ffffff`）
- 无 border-radius——尖锐的矩形按钮

### 卡片与容器
- 无 border-radius——所有容器都是尖角矩形
- 浅色章节使用白色背景
- 英雄/特性章节使用深色背景
- 大多数元素无可见边框

### 导航
- BMWTypeNextLatin 18px 字重 900 用于主导航链接
- 深色头部上白色文本
- BMW logo 54x54px
- 悬停：保持白色，text-decoration 无
- 头部中"首页"文本链接

### 图片处理
- 全出血汽车摄影
- 深色电影照明
- 边缘到边缘英雄图片
- 汽车摄影作为主要视觉内容

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 刻度：1px, 5px, 8px, 10px, 12px, 15px, 16px, 20px, 24px, 30px, 32px, 40px, 45px, 56px, 60px

### 网格与容器
- 全宽英雄摄影
- 居中内容章节
- 页脚：多列链接网格

### 留白哲学
- **展厅节奏**：深色英雄章节配慷慨内边距创造穿行展厅的感觉，每辆车都在自己的空间被聚光灯照亮。
- **压缩内容**：正文区域使用紧凑行高和紧凑间距——信息密集，无浪费。

### Border Radius 刻度
- **未检测到。** BMW 完全使用尖角——每个元素都是精准的矩形。这是分析过的最棱角分明的设计系统。

## 6. 深度与提升

| 级别 | 处理方式 | 用途 |
|-------|-----------|-----|
| 摄影 (Level 0) | 全出血深色图像 | 英雄背景 |
| 平面 (Level 1) | 白色表面，无阴影 | 内容章节 |
| 聚焦 (无障碍) | BMW Focus Blue（`#0653b6`） | 聚焦状态 |

**阴影哲学**：BMW 几乎不使用阴影。深度完全通过深色摄影章节和白色内容章节之间的对比创造——汽车照明完成提升工作。

## 7. 宜与忌

### 宜
- 所有展示标题使用 BMWTypeNextLatin Light（300）大写
- 保持所有角落尖锐（0px 圆角）——棱角几何不可协商
- 仅将 BMW Blue（`#1c69d4`）用于交互元素——从不装饰性
- 导航强调应用字重 900——极端字重对比是有意为之
- 英雄章节使用全出血汽车摄影
- 全程保持紧凑行高（1.15–1.30）
- 使用 `--site-context-*` CSS 变量进行主题化

### 忌
- 不要圆角——零半径是 BMW 身份
- 不要将 BMW Blue 用于背景或大表面——它仅作强调
- 不要使用中等字重（500–600）——系统使用 300、400、700、900 极端值
- 不要添加装饰元素——摄影和排版承载一切
- 不要使用宽松行高——BMW 文本始终压缩
- 不要减轻深色英雄章节——与白色的对比即是设计

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 关键变化 |
|------|-------|-------------|
| Mobile Small | <375px | 支持的最小尺寸 |
| Mobile | 375–480px | 单列 |
| Mobile Large | 480–640px | 轻微调整 |
| Tablet Small | 640–768px | 2 列开始 |
| Tablet | 768–920px | 标准平板 |
| Desktop Small | 920–1024px | 桌面布局开始 |
| Desktop | 1024–1280px | 标准桌面 |
| Large Desktop | 1280–1440px | 扩展 |
| Ultra-wide | 1440–1600px | 最大布局 |

### 折叠策略
- 英雄：60px → 缩小，保持大写
- 导航：水平 → 汉堡
- 摄影：所有尺寸保持全出血
- 内容章节：垂直堆叠
- 页脚：多列 → 堆叠
