# 设计系统：Ollama

## 1. 视觉主题与氛围

Ollama 的界面是将极简主义发挥到极致的体现——一个纯白的虚空，内容在其中浮动，没有装饰、阴影或色彩。设计哲学映射了产品本身：剥离一切不必要的元素，直到只剩下核心工具。这是 Dieter Rams 设计理念的数字化呈现——每个像素都有其存在的理由，而设计的缺失本身就是设计。

整个页面完全采用灰度呈现。界面中没有彩色——没有品牌蓝色，没有强调绿色，没有语义红色。唯一存在的颜色是纯黑（`#000000`和纯白（`#ffffff`之间的灰色层次，创造出一个单色环境，让用户对"开放模型"的心智模型不被品牌色彩所影响。Ollama 的羊驼吉祥物以简单的黑色线条艺术呈现，是唯一的插图——甚至它也是单色的。

让 Ollama 与众不同的是 SF Pro Rounded（Apple 的圆角系统字体）与独特的胶囊形状几何元素（所有交互元素使用 9999px 圆角）的结合。圆润的字母形态 + 圆润的按钮 + 圆润的容器创造了一种连贯的"柔和语言"，让开发者 CLI 工具感觉亲切友好而非令人望而生畏。这是带有温度的极简主义——不是冰冷的瑞士风格网格极简主义，而是边缘真正被软化的一种风格。

**核心特征：**
- 纯白画布，零彩色——完全灰度
- SF Pro Rounded 标题创造出独特的 Apple 式柔和感
- 二元圆角系统：12px（容器）或 9999px（所有交互元素）
- 零阴影——深度完全来自背景色变化和边框
- 所有交互元素使用胶囊形状（按钮、标签页、输入框、标签）
- Ollama 羊驼作为唯一的插图——黑色线条艺术，无色彩
- 极端的内容克制——首页短小精悍、聚焦清晰、整洁有序

## 2. 色彩系统与角色

### 主色
- **Pure Black**（`#000000`）：主标题、主链接和最深的文字。唯一能吸引注意力的"颜色"。
- **Near Black**（`#262626`）：浅色表面上的按钮文字，次要标题字重。
- **Darkest Surface**（`#090909`）：尽可能最深的表面——几乎与纯黑无法区分，用于页脚或深色容器。

### 表面与背景
- **Pure White**（`#ffffff`）：主页面背景——不是米白，不是奶油色，是纯白。次要操作的按钮表面。
- **Snow**（`#fafafa`）：与白色最微妙的表面区分——用于区块背景和略微抬升的容器。
- **Light Gray**（`#e5e5e5`）：按钮背景、边框和主要的包围色。最常用的中性色。

### 中性色与文字
- **Stone**（`#737373`）：次要正文、页脚链接和弱化内容。主要的"柔和"色调。
- **Mid Gray**（`#525252`）：强调的次要文字，比 Stone 略深。
- **Silver**（`#a3a3a3`）：三级文字、占位符和深度弱化的元数据。
- **Button Text Dark**（`#404040`）：专门用于白色表面按钮文字。

### 语义与强调
- **Ring Blue**（`#3b82f6` 50% 透明度）：整个系统中唯一的非灰色——Tailwind 的默认焦点环，仅用于键盘可访问性。在正常交互流程中永远不可见。
- **Border Light**（`#d4d4d4`）：白色表面按钮边框的略深灰色。

### 渐变系统
- **无。** Ollama 完全不使用渐变。视觉分离来自扁平色块和单像素边框。这是一个深思熟虑、近乎哲学性的设计选择。

## 3. 字体排版规则

### 字体家族
- **Display**：`SF Pro Rounded`，回退字体：`system-ui, -apple-system, system-ui`
- **Body / UI**：`ui-sans-serif`，回退字体：`system-ui, Apple Color Emoji, Segoe UI Emoji, Segoe UI Symbol, Noto Color Emoji`
- **Monospace**：`ui-monospace`，回退字体：`SFMono-Regular, Menlo, Monaco, Consolas, Liberation Mono, Courier New`

*注意：SF Pro Rounded 是 Apple 的系统字体——在 macOS/iOS 上呈现圆角终端，在其他平台上回退到系统无衬线字体。*

### 层级

| 角色 | 字体 | 大小 | 字重 | 行高 | 字间距 | 备注 |
|------|------|------|------|------|--------|------|
| Display / Hero | SF Pro Rounded | 48px (3rem) | 500 | 1.00 (紧凑) | normal | 最大视觉冲击，圆角字母形态 |
| Section Heading | SF Pro Rounded | 36px (2.25rem) | 500 | 1.11 (紧凑) | normal | 功能区块标题 |
| Sub-heading | SF Pro Rounded / ui-sans-serif | 30px (1.88rem) | 400–500 | 1.20 (紧凑) | normal | 卡片标题、功能名称 |
| Card Title | ui-sans-serif | 24px (1.5rem) | 400 | 1.33 | normal | 中等强调标题 |
| Body Large | ui-sans-serif | 18px (1.13rem) | 400–500 | 1.56 | normal | Hero 描述、按钮文字 |
| Body / Link | ui-sans-serif | 16px (1rem) | 400–500 | 1.50 | normal | 标准正文、导航 |
| Caption | ui-sans-serif | 14px (0.88rem) | 400 | 1.43 | normal | 元数据、描述 |
| Small | ui-sans-serif | 12px (0.75rem) | 400 | 1.33 | normal | 最小的无衬线文字 |
| Code Body | ui-monospace | 16px (1rem) | 400 | 1.50 | normal | 行内代码、命令 |
| Code Caption | ui-monospace | 14px (0.88rem) | 400 | 1.43 | normal | 代码片段、次要内容 |
| Code Small | ui-monospace | 12px (0.75rem) | 400–700 | 1.63 | normal | 标签、标注 |

### 原则
- **圆角标题，标准正文**：SF Pro Rounded 以其独特的圆角终端承载展示标题，而标准系统无衬线字体处理所有正文。圆角字体本身就是品牌表达。
- **字重克制**：只有两个字重重要——400（常规）用于正文，500（中等）用于标题。没有粗体、没有细体、没有黑色字重。这种极端的克制强化了极简哲学。
- **紧凑标题，舒适正文**：标题压缩至 1.0 行高，而正文放宽至 1.43–1.56。这种对比创造了清晰的层级，无需字重对比。
- **等宽字体彰显开发者身份**：代码块和终端命令作为主要内容贯穿始终，使用系统等宽字体栈。

## 4. 组件样式

### 按钮

**Gray Pill（主要）**
- 背景：Light Gray（`#e5e5e5`）
- 文字：Near Black（`#262626`）
- 内边距：10px 24px
- 边框：细实线 Light Gray（`1px solid #e5e5e5`）
- 圆角：胶囊形状（9999px）
- 主要操作按钮——低调、灰度、始终保持胶囊形状

**White Pill（次要）**
- 背景：Pure White（`#ffffff`）
- 文字：Button Text Dark（`#404040`）
- 内边距：10px 24px
- 边框：细实线 Border Light（`1px solid #d4d4d4`）
- 圆角：胶囊形状（9999px）
- 次要操作——视觉上比 Gray Pill 更轻

**Black Pill（CTA）**
- 背景：Pure Black（`#000000`）
- 文字：Pure White（`#ffffff`）
- 圆角：胶囊形状（9999px）
- 推断自"Create account"和"Explore"按钮
- 最大强调——黑底白字

### 卡片与容器
- 背景：Pure White 或 Snow（`#fafafa`）
- 边框：细实线 Light Gray（`1px solid #e5e5e5`），需要时使用
- 圆角：舒适圆润（12px）——系统中唯一的非胶囊圆角
- 阴影：**无**——任何元素都没有阴影
- 悬停：可能是微妙的背景变化或边框加深

### 输入框与表单
- 背景：Pure White
- 边框：`1px solid #e5e5e5`
- 圆角：胶囊形状（9999px）——搜索输入框和表单字段都是胶囊形状
- 焦点：Ring Blue（`#3b82f6` 50% 透明度）环
- 占位符：Silver（`#a3a3a3`）

### 导航
- 简洁的水平导航，元素极少
- Logo：Ollama 羊驼图标 + 黑色文字标识
- 链接："Models"、"Docs"、"Pricing"，黑色，16px，字重 400
- 搜索栏：胶囊形状，带占位文字
- 右侧："Sign in"链接 + "Download" 黑色胶囊 CTA 按钮
- 无边框、无背景——白色页面上的透明导航

### 图片处理
- Ollama 羊驼吉祥物是唯一的插图——白底黑色线条艺术
- 代码截图/终端输出显示在有边框的容器中（12px 圆角）
- 集成 Logo 以简单图标形式在网格中展示
- 无照片、无渐变、无装饰性图像

### 独特组件

**Tab Pills**
- 胶囊形状的标签选择器（如"Coding" | "OpenClaw"）
- 激活态：Light Gray 背景；非激活态：透明
- 全部胶囊形状（9999px）

**Model Tags**
- 小型胶囊形状标签（如"ollama"、"launch"、"claude"）
- Light Gray 背景，深色文字
- 浏览模型的主要方式

**Terminal Command Block**
- 显示 `ollama run` 命令的等宽代码
- 极简样式——仅带边框的 12px 圆角容器
- 集成复制按钮

**Integration Grid**
- 集成 Logo 网格（Codex、Claude Code、OpenCode、LangChain 等）
- 每个在有边框的胶囊或卡片中，含图标 + 名称
- 按类别分标签（Coding、Documents & RAG、Automation、Chat）

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 刻度：4px, 6px, 8px, 9px, 10px, 12px, 14px, 16px, 20px, 24px, 32px, 40px, 48px, 88px, 112px
- 按钮内边距：10px 24px（所有按钮一致）
- 卡片内部内边距：约 24–32px
- 区块垂直间距：非常充裕（88px–112px）

### 网格与容器
- 最大容器宽度：约 1024–1280px，居中
- Hero：居中单列，带羊驼插图
- 功能区块：双列布局（左侧文字，右侧代码）
- 集成网格：响应式多列
- 页脚：简洁单行

### 留白哲学
- **空即奢华**：页面出奇地短小稀疏——没有任何功能区块喧宾夺主。每个概念获得最小但足够的空间。
- **内容密度设计上就很低**：其他 AI 公司堆砌一个又一个功能，而 Ollama 只呈现三个想法（运行模型、与应用配合使用、集成）然后停止。
- **留白本身就是品牌**：零装饰的纯白空间传达着"这个工具不会碍事"。

### 圆角刻度
- 舒适圆润（12px）：唯一的容器圆角——代码块、卡片、面板
- 胶囊形状（9999px）：所有交互元素——按钮、标签页、输入框、标签、徽章

*这个二元系统极端而独特。没有 4px、没有 8px、没有圆角梯度。元素要么是容器（12px），要么是交互元素（胶囊）。*

## 6. 深度与层级

| 层级 | 处理方式 | 用途 |
|------|----------|------|
| 扁平（Level 0） | 无阴影、无边框 | 页面背景、大多数内容 |
| 有边框（Level 1） | `1px solid #e5e5e5` | 卡片、代码块、按钮 |

**阴影哲学**：Ollama 使用**零阴影**。这不是疏忽——而是深思熟虑的设计决策。其他所有主要 AI 产品网站都至少使用微妙的阴影。Ollama 扁平、无阴影的方法创造了纸张般的体验，元素仅通过背景色和单像素边框区分。深度通过**内容层级和字重**传达，而非视觉分层。

## 7. 做与不做

### 做
- 使用纯白（`#ffffff`）作为页面背景——绝不用米白或奶油色
- 在所有交互元素上使用胶囊形状（9999px）圆角——按钮、标签页、输入框、标签
- 在所有非交互容器上使用 12px 圆角——代码块、卡片、面板
- 严格保持灰度调色板——除蓝色焦点环外无彩色
- 在展示标题使用字重 500 的 SF Pro Rounded——圆角终端是品牌表达
- 保持零阴影——深度仅来自边框和背景变化
- 保持低内容密度——每个区块应呈现一个清晰的想法
- 对终端命令和代码使用等宽字体——这是主要内容，不是装饰
- 所有按钮保持 10px 24px 内边距和胶囊形状——一致性是绝对的

### 不做
- 不要引入任何彩色——没有品牌蓝、没有强调绿、没有暖色调
- 不要使用 12px 到 9999px 之间的圆角——系统是二元的
- 不要给任何元素添加阴影——扁平美学是有意为之
- 不要使用超过 500 的字重——没有粗体、没有黑色字重
- 不要添加羊驼吉祥物以外的装饰性插图
- 不要在任何地方使用渐变——只用扁平色块和边框
- 不要过度复杂化布局——最多两列，无复杂网格
- 不要使用超过 1px 的边框——包围总是尽可能轻盈的触感
- 不要添加悬停动画或过渡——交互应该感觉即时和直接

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 关键变化 |
|------|------|----------|
| Mobile | <640px | 单列，所有内容堆叠，汉堡导航 |
| Small Tablet | 640–768px | 间距的细微调整 |
| Tablet | 768–850px | 双列布局开始 |
| Desktop | 850–1024px | 标准布局，扩展功能 |
| Large Desktop | 1024–1280px | 最大内容宽度 |

### 触摸目标
- 所有按钮都是胶囊形状，内边距充裕（10px 24px）
- 导航链接舒适的 16px 大小
- 最小触摸区域轻松超过 44x44px

### 折叠策略
- **导航**：移动端折叠为汉堡菜单
- **功能区块**：双列 → 堆叠单列
- **Hero 文字**：48px → 36px → 30px 渐进缩放
- **集成网格**：多列 → 双列 → 单列
- **代码块**：保持水平滚动

### 图片行为
- 羊驼吉祥物按比例缩放
- 代码块保持等宽格式
- 集成图标重排为更少列数
- 无艺术指导变化

### 迭代指南
1. 一次专注于一个组件
2. 所有值保持灰度——使用"Stone (#737373)"而非"使用浅色"
3. 始终指定胶囊（9999px）或容器（12px）圆角——没有中间值
4. 阴影始终为零——永远不要添加
5. 字重始终是 400 或 500——从不使用粗体
6. 如果感觉某处过度装饰，移除它——对 Ollama 来说少即是多
